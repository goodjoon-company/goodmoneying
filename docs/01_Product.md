# 제품 기준

Status: Draft
Last Updated: 2026-06-16

## 목적

이 문서는 코인 투자와 주식 투자 자동화를 위한 대상 종목과 코인 발굴, 장기간의 과거 데이터(Back Data)를 기반으로 한 패턴 분석과 매매전략 개발, 개발된 전략을 실행하는 봇(Bot)의 설계와 동작 시뮬레이션(Simulation), 이후 모의매매(Paper Trading)와 실제 자동매매(Live Trading)까지 확장 가능한 개인용 투자 데이터 플랫폼을 만드는 데 목적을 둔다.

초기 제품은 실제 주문 실행보다 먼저 신뢰 가능한 시장 데이터 기반을 구축한다. 국내 주식, 해외 주식, 업비트(Upbit) 거래소 코인, 호가(Orderbook), 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시를 지속적으로 수집하고, 대규모 언어 모델(LLM, Large Language Model) 분석 결과를 전략이 사용할 수 있는 구조화 신호(Signal)로 축적한다.

## 제품 범위

### 전체 제품 범위

- 국내 주식, 해외 주식, 업비트(Upbit) 거래소 코인을 대상으로 투자 분석 데이터를 수집한다.
- 업비트 코인은 분석 실행 시점 기준 거래소 내 시가총액(Market Cap) 상위 50개로 제한한다.
- 가격, 거래량, 시가총액, 외국인/기관 보유와 매매 동향, 공매도(Short Selling), 재무지표를 수집한다.
- 24시간 365일 주기적으로 국내/해외 시장 호가(Orderbook)를 수집한다.
- 호가 데이터는 원천 스냅샷(Snapshot)과 전략용 요약 데이터를 병행 저장한다.
- 증권사 종목 리포트, 뉴스, 공시를 주기적으로 수집하고 원천 데이터를 보존한다.
- 대규모 언어 모델(LLM, Large Language Model)은 리포트, 뉴스, 공시를 분석해 사람이 읽는 요약과 전략이 소비할 수 있는 구조화 신호(Signal)를 모두 생성한다.
- LLM 신호는 주가 영향 카테고리(Category), 긍정/부정 방향, 강도(Strength), 신뢰도(Confidence), 기준 시점, 관련 종목을 포함한다.
- 가격 차트의 시계열(Time Series) 위에서 LLM 신호와 이후 가격/거래량 변화를 비교 분석할 수 있어야 한다.
- 사용자는 수집 데이터를 기반으로 시장 스캐닝(Scanning), 관심 대상 심층 분석, 전략 설계, 백테스트(Backtest)를 수행할 수 있어야 한다.
- 사용자는 봇(Bot)별 투자 가능 금액, 대상 시장, 대상 종목/코인, 리스크 한도, 실행 주기, 주문 방식, 전략 가중치(Weight)를 설정할 수 있어야 한다.
- 봇은 여러 전략을 파이프라인(Pipeline)으로 조합하고, 개별 전략 신호, 차단 조건, 리스크 조건, 가중치를 반영해 최종 매수/매도/보류 판단을 만들 수 있어야 한다.
- 봇 동작은 실제 주문 없이 시뮬레이션(Simulation)할 수 있어야 하며, 충분히 검증된 뒤 모의매매(Paper Trading)와 실제 자동매매(Live Trading)로 확장한다.

### 1차 MVP 범위

1차 최소 기능 제품(MVP, Minimum Viable Product)은 자동매매나 실거래가 아니라 데이터 수집, 저장, 조회, 품질 확인, LLM 분석 신호 생성까지로 제한한다.

- 국내 주식, 해외 주식, 업비트(Upbit) 코인 수집 대상을 구성한다.
- 가격/거래량, 시가총액, 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시, 호가(Orderbook)를 주기적으로 수집한다.
- 수집한 원천 데이터와 전략용 요약 데이터를 저장하고 조회할 수 있게 한다.
- 결측, 중복, 지연, 원천 신뢰도 같은 데이터 품질(Data Quality)을 확인할 수 있게 한다.
- 리포트, 뉴스, 공시를 LLM으로 분석해 요약과 구조화 신호(Signal)를 저장한다.
- 가격, 거래량, 수급, 공매도, 호가 요약, LLM 신호를 같은 시계열(Time Series) 기준으로 정렬한다.

### 제외 범위

- 1차 MVP에서는 실제 주문 실행, 모의매매(Paper Trading), 자동매매(Live Trading)를 제외한다.
- 1차 MVP에서는 다중 사용자, 권한(Authorization), 과금(Billing), 팀 공유 기능을 제외한다.
- 호가(Orderbook) 데이터가 없는 과거 구간에서는 호가 기반 전략 검증을 완전하게 지원하지 않는다.
- 이 제품은 투자 수익 보장이나 법적 의미의 매수/매도 추천을 목표로 하지 않는다.

## 사용자와 주요 시나리오

### 초기 사용자

초기 사용자는 본인 투자 분석과 자동화 기반 구축을 목표로 하는 개인 단일 사용자다.

### 1차 MVP 시나리오

1. **데이터 수집 기반 구축**
   - 사용자는 국내 주식, 해외 주식, 업비트(Upbit) 코인을 수집 대상으로 설정한다.
   - 시스템은 가격, 거래량, 시가총액(Market Cap), 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시, 호가(Orderbook)를 주기적으로 수집한다.
   - 시스템은 수집 상태, 실패, 지연, 중복, 결측을 데이터 품질(Data Quality) 상태로 기록한다.

2. **LLM 분석 신호 생성**
   - 시스템은 리포트, 뉴스, 공시를 대규모 언어 모델(LLM, Large Language Model)로 분석한다.
   - 시스템은 매출증대, 원가절감, 경쟁력상승, 공급계약, 비용증가, 규제리스크, 실적악화, 수요감소 같은 주가 영향 카테고리(Category)를 관리한다.
   - 시스템은 각 문서에서 카테고리, 긍정/부정 방향, 강도(Strength), 신뢰도(Confidence), 기준 시점, 관련 종목을 구조화 신호(Signal)로 저장한다.
   - 시스템은 사용자가 읽을 수 있는 요약도 함께 저장한다.

3. **시간축 정렬**
   - 시스템은 가격, 거래량, 수급, 공매도, 호가 요약, LLM 신호를 같은 시계열(Time Series) 기준으로 정렬한다.
   - 이후 시장 스캐닝(Scanning), 전략 검증, 봇(Bot) 실행 판단이 같은 입력 데이터를 사용할 수 있게 한다.

### 전체 제품 시나리오

1. **시장 스캐닝**
   - 사용자는 전체 시장에서 관심 후보를 찾기 위한 조건을 설정한다.
   - 시스템은 가격/거래량, 수급, 공매도, 재무지표, 호가 요약, LLM 신호를 기반으로 후보 종목과 코인을 추린다.
   - 업비트 코인은 분석 실행 시점의 시가총액(Market Cap) 상위 50개만 스캐닝 대상에 포함한다.

2. **관심 대상 심층 분석**
   - 사용자는 특정 국내 주식, 해외 주식, 코인을 선택한다.
   - 시스템은 장기 시계열(Time Series), 수급 변화, 공매도 변화, 호가 요약, 재무지표, 뉴스/공시/리포트 요약과 LLM 신호를 함께 보여준다.

3. **신호와 가격 영향 분석**
   - 사용자는 특정 카테고리(Category)의 긍정/부정 신호와 강도가 이후 가격, 수익률, 변동성, 거래량에 어떤 영향을 줬는지 확인한다.
   - 시스템은 신호 발생 시점을 가격 차트의 시계열(Time Series)에 정렬하고, 이후 구간의 가격/거래량 변화를 분석할 수 있어야 한다.

4. **전략 설계와 백테스트**
   - 사용자는 가격/거래량, 수급, 공매도, 재무지표, 호가 요약, LLM 신호를 조합해 전략 조건을 정의한다.
   - 시스템은 과거 데이터로 전략 성과를 검증하는 백테스트(Backtest)를 제공한다.
   - 호가 기반 전략은 호가 데이터 수집이 시작된 이후 기간에만 완전하게 검증할 수 있다는 제약을 명시한다.

5. **봇 설계**
   - 사용자는 봇(Bot)별 투자 가능 금액, 대상 시장, 대상 종목/코인, 리스크 한도, 실행 주기, 주문 방식, 전략 가중치(Weight)를 설정한다.
   - 같은 전략이라도 봇별로 파라미터(Parameter), 가중치, 투자 비중, 실행 방식을 다르게 적용할 수 있다.

6. **전략 파이프라인 구성**
   - 사용자는 여러 전략의 결과를 순차 또는 병렬 파이프라인(Pipeline)으로 조합한다.
   - 시스템은 개별 전략 신호, 차단 조건, 리스크 조건, 가중치를 반영해 최종 매수/매도/보류 판단을 만든다.

7. **봇 동작 시뮬레이션**
   - 사용자는 실제 주문 없이 봇이 시장 데이터나 과거 재생 데이터(Replay Data)를 입력받았을 때 어떻게 판단하고 행동하는지 시뮬레이션(Simulation)한다.
   - 시스템은 투자 가능 금액, 포지션(Position), 주문 후보, 체결 가정, 손익, 전략별 판단 근거, 최종 결정을 추적한다.
   - 시뮬레이션은 실제 주문 전 봇의 의사결정 흐름과 리스크를 검증하는 단계다.

8. **모의매매와 실거래**
   - 사용자는 검증된 봇을 모의매매(Paper Trading)로 실행한다.
   - 충분히 검증된 봇은 실제 자동매매(Live Trading)로 전환할 수 있다.

## 정책

- 1차 MVP(Minimum Viable Product)는 데이터 수집, 저장, 조회, 데이터 품질(Data Quality) 확인, LLM 분석 신호 생성까지로 제한한다.
- 실제 주문 실행, 모의매매(Paper Trading), 자동매매(Live Trading)는 MVP에서 제외한다.
- 원천 데이터는 재처리와 감사(Audit)를 위해 가능한 한 보존한다.
- 호가(Orderbook) 데이터가 없는 과거 구간에서는 호가 기반 전략 검증을 완전하게 지원하지 않는다.
- LLM 분석 결과는 투자 판단의 보조 입력이며, 사실 데이터와 동일한 확정 정보로 취급하지 않는다.
- LLM 신호 카테고리(Category)는 버전 관리해야 하며, 카테고리 변경 시 과거 신호와 비교 가능성이 깨질 수 있음을 기록한다.
- 제품은 개인용 단일 사용자 플랫폼으로 시작하며, 다중 사용자 권한(Authorization), 과금(Billing), 팀 공유는 초기 범위에서 제외한다.
- 제품은 투자 수익을 보장하지 않으며, 매수/매도 추천의 법적 책임을 제품 목표로 두지 않는다.

## 로드맵

| 단계 | 목표 | 완료 기준 |
|---|---|---|
| M0 | 문서와 계약(Contract) 기반 정리 | 제품 범위, 아키텍처(Architecture) 색인, 데이터 계약 위치, Task/Test/History 구조가 정리된다. |
| M1 | 시장 데이터 수집 MVP(Minimum Viable Product) | 국내 주식, 해외 주식, 업비트(Upbit) 코인에 대해 가격/거래량, 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시, 호가(Orderbook)를 수집, 저장, 조회하고 데이터 품질(Data Quality)을 확인할 수 있다. |
| M2 | LLM 분석 신호 기반 | 뉴스, 공시, 리포트를 요약하고, 주가 영향 카테고리(Category), 긍정/부정 방향, 강도(Strength), 신뢰도(Confidence)를 구조화 신호(Signal)로 저장한다. |
| M3 | 시계열(Time Series) 영향 분석 | 가격/거래량, 수급, 공매도, 호가 요약, LLM 신호를 같은 시간축에 정렬하고, 특정 신호가 이후 수익률, 변동성, 거래량에 미친 영향을 분석한다. |
| M4 | 시장 스캐닝(Scanning)과 심층 분석 | 수집 데이터와 LLM 신호를 기반으로 후보 종목/코인을 발굴하고, 관심 대상의 장기 분석 화면 또는 리포트를 제공한다. |
| M5 | 전략 설계와 백테스트(Backtest) | 가격/거래량, 수급, 공매도, 재무지표, 호가, LLM 신호를 조합한 전략을 정의하고 과거 데이터로 검증한다. |
| M6 | 봇 설계와 동작 시뮬레이션(Simulation) | 봇별 투자 가능 금액, 시장, 리스크 한도, 전략 가중치(Weight), 실행 주기를 설정하고, 실제 주문 없이 봇 판단과 손익을 시뮬레이션한다. |
| M7 | 모의매매(Paper Trading)와 실거래(Live Trading) 준비 | 검증된 봇을 모의매매로 실행하고, 실제 자동매매 전환에 필요한 주문 계약, 리스크 제한, 감사(Audit) 기록을 갖춘다. |

## 변경 규칙

- 제품 범위, 요구사항, 정책, 로드맵이 바뀌면 이 문서를 먼저 갱신한다.
- 구현 로그와 검증 결과는 이 문서에 장황하게 남기지 않는다. 해당 내용은 `Task/`, `Test/`, `History/`에 기록한다.
