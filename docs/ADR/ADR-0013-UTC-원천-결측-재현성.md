# ADR-0013: UTC 원천 시각, 결측 의미와 재현성

- 상태: Accepted
- 날짜: 2026-07-17
- 대체: ADR-0002의 KST 저장·계산 결정 중 시간대 부분

## 맥락

기존 DB 세션과 계산은 KST를 기본으로 사용했다. 다중 원천, 거래소 사건, 수신·저장 지연, 백테스트 인과성을 분리하려면 지역 시간대에 의존하지 않는 내부 표준과 획득 불가·무거래를 구분하는 계약이 필요하다.

## 결정

모든 내부 `timestamptz`는 UTC 의미로 저장·계산하고 UI에서만 KST를 명시해 변환한다. 원천 사건은 거래소 시각, 수신 시각, 저장 시각, 백테스트 사용 가능 시각을 분리한다.

커버리지 상태는 `available`, `no_trade`, `missing`, `unavailable`, `unverified`로 고정한다. 무거래 캔들을 원천에 합성하지 않는다. 채움 정책은 파생 분석 입력이며 데이터셋·전략·백테스트 결과에 버전과 영향을 기록한다. 가격·수량·금액 계산은 Decimal만 사용한다.

## 대안

1. **KST 저장 유지**: 단일 사용자 표시는 편하지만 API UTC, DST가 있는 미래 원천, 사건 인과성 비교에서 혼동이 커진다.
2. **모든 빈 슬롯을 0 거래 캔들로 생성**: 연속 시계열은 쉽지만 무거래와 획득 실패를 구분하지 못해 연구 결과가 왜곡된다.
3. **부동소수점 계산**: 빠르고 편하지만 주문 금액·수수료·결정론적 재실행에 오차가 누적된다.

## 트레이드오프

- 기존 KST 기반 테스트와 쿼리를 이행해야 한다.
- UI·CSV·로그가 표시 시간대를 명시해야 한다.
- 결측 계약과 manifest 저장량이 늘지만 결과의 설명 가능성과 재현성이 높아진다.

## 결과와 후속 작업

- 기존 행의 절대 시각 보존을 검증하는 확장-수축 migration을 작성한다.
- 운영 적용 전 ADR-0015의 백업·복원 gate를 통과한다.
- 시간대·미래 참조·Decimal 속성 테스트를 추가한다.
- 모든 API 날짜 필드에 UTC wire format과 KST display 정책을 문서화한다.
