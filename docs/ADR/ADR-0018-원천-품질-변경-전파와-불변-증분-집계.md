# ADR-0018: 원천·품질 변경 전파와 불변 증분 집계

- 상태: Accepted
- 날짜: 2026-07-17
- 관련 Issue: [#29](https://github.com/goodjoon-company/goodmoneying/issues/29)
- 대체·확장: ADR-0009의 최대 시각 워터마크·집계 UPSERT, ADR-0017의 집계 저장 결정

## 맥락

최신 원천 시각만 비교하면 과거 캔들 정정과 커버리지 품질 전이를 놓친다. 집계 행을 제자리 갱신하면 과거 시점에 알 수 있었던 결과와 데이터셋 입력을 재현할 수 없다.

## 결정

- 신규 원천 개정과 커버리지 품질 이벤트는 생성 트랜잭션 안에서 영향 파생 버킷의 `candle_rollup_invalidations`와 내구성 작업을 추가한다.
- 원천·품질 전파는 시장 단위 트랜잭션 자문 잠금(advisory lock)으로 직렬화해 최신 원천 상한과 품질 상한을 함께 가진 결합 프런티어(frontier)를 반드시 남긴다.
- 커버리지 스냅샷은 작업 범위와 겹치는 구간만 반개방 범위로 자르고 정렬해 해시한다. 범위 밖 상태는 입력 문맥에 포함하지 않는다.
- 품질 전이 뒤 버킷에 원천 캔들이 하나라도 있으면 품질 집계 개정판을 만든다. 원천이 완전히 빈 구간은 가짜 OHLCV나 재계산 작업을 만들지 않고 `coverage_intervals`와 `data_quality_events`를 권위 있는 품질 원장으로 유지한다.
- 한 작업은 출력 버킷 512개 이하이고 인접 범위만 합친다. 재계산 원천 조회는 범위와 고정된 원천 개정 상한으로 제한한다.
- 원천 현재 투영은 상한 안에서 `source_as_of DESC, revision_number DESC, id DESC`로 선택한다. 늦게 도착한 과거 사실은 원장에 남지만 현재 결과를 퇴행시키지 않는다.
- `candle_rollups`는 대리 ID를 가진 추가 전용 개정 원장이다. 입력 내용 해시·커버리지 스냅샷 해시·원천 개정 상한·품질 이벤트 상한이 모두 같은 재실행만 멱등 흡수한다. 따라서 값이 `A → B → A`로 돌아와도 전진한 프런티어의 새 개정은 보존하며 UPDATE·DELETE는 DB가 거부한다.
- 결과 내용 해시는 OHLCV, 품질, 완전성, 계산 버전의 의미 값만 포함한다. 대리 ID와 `materialized_at`은 제외한다.
- 현재 조회는 원천 개정 상한·지식 시각 순으로 최신 개정을 선택하고, 선택적 지식 시각 또는 원천 개정 상한으로 과거 투영을 재현한다.
- 전체 물질화도 범위별 품질 이벤트 상한과 지식 시각을 고정한다. 런타임은 품질 이벤트 원장을 직접 읽지 않고 PUBLIC 실행 권한을 회수한 최소권한 보안 정의자(SECURITY DEFINER) 함수만 실행한다.
- 작업 상태는 `pending|running|retry_wait|succeeded|dead_letter|cancelled`이며 임대 소유자·만료 시각 fencing, 지수 백오프, 최대 시도, 안전 재시작을 적용한다.

## 결과

- 과거 원천·집계 개정은 보존되고 영향 밖 버킷의 해시와 물질화 시각은 변하지 않는다.
- P2-3 지표와 P2-5 데이터셋 버전·실행 입력은 지식 시각·원천 개정 상한으로 같은 집계를 다시 선택할 수 있다.
- 운영 화면은 최신 증분 작업 상태, 범위, 시도 횟수, 기록 행 수와 오류 코드를 표시한다.
