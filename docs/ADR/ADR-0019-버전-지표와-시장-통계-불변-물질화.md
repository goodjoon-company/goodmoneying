# ADR-0019: 버전 지표와 시장 통계 불변 물질화

- 상태: Accepted
- 날짜: 2026-07-17
- 관련 Issue: [#29](https://github.com/goodjoon-company/goodmoneying/issues/29)
- 확장: ADR-0008의 WebSocket 증분 메시지, ADR-0017의 개정 계보, ADR-0018의 무효화·임대·fencing

## 맥락

요청 시점 계산은 같은 URL이 저장 상태와 실행 시각에 따라 달라지고, 과거 원천 정정과 품질 변화가 어떤 지표를 만들었는지 설명할 수 없다. 지표 정의를 이름만으로 참조하면 알고리즘·파라미터·정밀도 변화도 재현할 수 없다. 시장 통계를 임의 지표 정의에 묶으면 연구 데이터의 독립 버전 경계가 사라진다.

## 결정

- SMA20·SMA60·EMA20·볼린저 밴드20·RSI14는 버전 1의 알고리즘, 파라미터, SHA-256 정의 해시, Decimal 정밀도 50과 `ROUND_HALF_EVEN`을 고정한다.
- 지표 물질화와 값, 시장 통계는 추가 전용이다. 각 행은 현재 원천 개정 또는 현재 집계 중 정확히 하나, 원천·품질 프런티어, `source_as_of`, `knowledge_at`, 부모 물질화/값 계보와 내용 해시를 가진다.
- 지표 정의 루트와 정의 버전, 값-집계 연결도 생성 후 수정·삭제하지 않는다. 표시 이름 변경이나 계산 의미 변경은 새 정의 버전으로 표현한다.
- 시장 통계 버전 1은 1구간 종가 수익률, 20개 수익률의 모집단 실현 변동성, 현재 구간 거래량·거래대금을 독립 원장에 저장한다. 0 채움이나 전방 채움은 하지 않는다.
- 결측·불완전·미확인 품질과 시간 간격은 계산 상태를 초기화한다. EMA는 최초 20개 단순 평균을 seed로 사용하고 RSI는 Wilder 평활을 사용한다.
- 원천 1분 개정, 집계 개정, 품질만 바뀐 집계 무효화 완료는 내구성 `indicator_invalidations`를 만든다. 작업은 DB 시계 임대, 세대(generation) fencing과 상품·주기 자문 잠금을 사용한다.
- 1분 품질 전이는 집계 완료를 기다리지 않고 품질 이벤트 자체로 지표 무효화를 만들며, 같은 영향 시각의 `A → B → A` 프런티어는 ID 순서로 직렬 처리한다.
- 각 지표·시장 통계 행은 다음 구간 계산에 필요한 최근 값, EMA·RSI 평활 상태와 `sourceAsOf`를 체크포인트(checkpoint)로 저장한다. 워커는 한 번에 최대 512구간을 계산하고 `progress_at`으로 이어서 처리하되 모든 청크가 동일한 원천·품질·지식 프런티어를 사용한다. 저장 체크포인트가 바로 이전 구간인 정상 정정·추가는 영향 범위만 물질화한다. 체크포인트가 없거나 떨어진 복구·정의 초기화는 실제 재생한 예열 구간도 같은 프런티어로 512개씩 물질화해 부모·값·현재 입력 계보를 잃지 않는다. 무효화 행의 일시 체크포인트는 청크 재시작을 보조하고 성공 또는 dead letter 전이 시 제거한다.
- 상품·주기의 프런티어는 불변 프런티어와 무효화 ID 순서로 직렬화한다. 선점 트랜잭션과 계산 자문 잠금 뒤에 선행 미완료 프런티어가 없음을 각각 확인해, 선행 청크가 `progress_at`으로 재개 대기 중일 때 후속 프런티어가 먼저 계산되지 않게 한다.
- 물질화 삽입과 무효화 완료는 같은 트랜잭션에서 임대 세대·소유자·만료를 다시 확인한다. 계산 또는 완료 fencing이 실패하면 해당 청크의 지표·값·시장 통계를 모두 롤백한다.
- 업그레이드 시 트리거 이전의 기존 이력은 상품·주기별 현재 프런티어 하나만 시드해 큐 크기를 제한한다.
- PostgreSQL REST GET과 분석 WebSocket은 저장된 물질화만 읽고 계산·쓰기를 수행하지 않는다. 페이지 커서는 최초 조회의 물질화 ID 상한과 `asOf`·정의 집합 해시뿐 아니라 상품, 주기, 조회 시작·끝을 고정해 다른 요청에 재사용할 수 없다.
- 운영 계약은 PostgreSQL에 한정한다. 테스트용 SQLite 저장소는 같은 계산·해시·상태 의미를 검증하는 프로세스 메모리 호환 폴백(fallback)이며 재시작 내구성이나 저장 전용 GET을 보장하지 않는다.
- WebSocket은 마지막 append/교체가 최신 캔들과 같은 시각일 때만 단일 upsert를 보내고, 과거 지점 변경만 전체 지표 스냅숏으로 갱신한다. 분·시간 구독은 반개방 종료 경계를 적용한 최신 1,000개 캔들과 같은 시각의 저장 지표만 같은 순서로 보낸다.

## 결과

- 값이 `A → B → A`로 돌아가도 세 개 프런티어와 부모 계보가 보존되고 `asOf`로 중간 B를 재현한다.
- 늦게 생긴 물질화는 진행 중인 cursor 페이지에 섞이지 않는다.
- 긴 정정 suffix도 512개 청크 체크포인트로 제한된 메모리에서 처리하며 마지막 청크까지 정정 `sourceAsOf`를 잃지 않는다.
- 새 지표 정의나 계산 의미 변경은 기존 행 수정이 아니라 새 정의 버전·계산 버전을 요구한다.
- 별도 지표 워커 프로세스 분리는 처리량과 장애 격리 측정 뒤 결정하며 현재 집계 워커는 집계 작업을 우선하고 유휴 시 지표 큐를 처리한다.
