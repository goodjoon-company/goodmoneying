## 변경 요약

- 백필 계획 생성 액션을 운영 상태 코인 행 상세에서 제거했다.
- 수집 대상/설정 화면에 백필 계획 생성 버튼, 수집 범위 설정 레이어 팝업, 백필 승인 패널의 계획 목록과 승인 버튼을 추가했다.
- 백필 API 클라이언트가 화면에서 지정한 목표 시작/종료 시각을 전송할 수 있게 했다.
- 워커가 승인된 백필 작업을 실행할 때 이미 저장된 시작 구간을 중복 요청하지 않고 첫 빈 1분봉부터 요청하도록 변경했다.
- 제품(Product)과 아키텍처(Architecture) 문서에 백필 후보 선택부터 워커 지속 수집까지의 흐름을 반영했다.

## 리스크

- 승인 전 백필 계획 목록은 현재 화면 세션 상태다. 새로고침 뒤 복구가 필요하면 서버 측 임시 계획 목록 조회 계약이 필요하다.
- 워커의 첫 빈 구간 탐색은 현재 목표 기간 전체의 기존 캔들을 조회한다. 장기간·대량 대상 최적화는 후속 개선 여지가 있다.

## 후속 작업

- 백필 계획 세트를 서버에 보존해야 하는지 결정한다.
- 결측 구간 단위로 API 요청을 더 세분화할지 검토한다.
