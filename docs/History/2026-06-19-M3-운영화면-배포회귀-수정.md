# 2026-06-19 M3 운영화면 배포회귀 수정

## 요약

운영 배포 후 수집 대상 저장과 백필 계획 생성이 401로 실패하던 문제를 수정했다. 기존 web 번들은 `local-dev-token`을 브라우저에 굽고 있었고, 운영 API는 별도 `GOODMONEYING_OPERATOR_TOKEN`을 요구했다. web Nginx 프록시(proxy)가 `/api` 요청을 API 서버로 전달하면서 운영 토큰을 서버 측 헤더(header)로 붙이도록 변경했다.

첫 화면 로딩도 대시보드 중심으로 줄였다. 후보 유니버스, 시장 리스트, 상세, 캔들은 탭 또는 상세 레이어 진입 시 지연 로딩한다.

## 변경

- `apps/web/nginx.conf.template` 추가
- `apps/web/Dockerfile`의 기본 `VITE_API_BASE_URL`을 `/api`로 변경
- `apps/web/src/api.ts` 첫 화면 로딩 경량화와 브라우저 토큰 제거
- `apps/web/src/App.tsx`, `apps/web/src/styles.css` 운영 상태 테이블/품질 tooltip/표시값 개선
- `docs/contracts/api/openapi.yaml`에 `qualityDetail`, `storedRowCount` 추가
- `deploy/profiles/prod-home/README.md`의 `web.env` 예시와 Nginx 프록시 운영 방식을 갱신
- `.github/workflows/deploy.yml` web build arg를 `/api`로 변경

- 운영 web 서버의 `/home/goodjoon/applications/goodmoneying/env/web.env`에 `GOODMONEYING_API_INTERNAL_URL=http://app-server01:8000`과 운영 `GOODMONEYING_OPERATOR_TOKEN`이 필요하다.
