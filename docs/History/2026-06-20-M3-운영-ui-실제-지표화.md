## 변경 요약

- 일반 API 실행에서 fixture 데이터가 자동으로 seed되지 않도록 변경했다.
- E2E(End-to-End)와 demo 목적의 fixture는 `GOODMONEYING_DEMO_DATA=1`을 명시해야만 사용하도록 했다.
- SQLite/PostgreSQL 저장소의 `source_candle` 커버리지(Coverage)를 실제 수집 계획 범위와 저장된 1분 버킷(bucket) 기준으로 계산하도록 변경했다.
- 결측 세그먼트(Missing Segment)는 고정 비율이 아니라 실제 비어 있는 연속 구간으로 만든다.
- 후보 유니버스(Candidate Universe)의 품질 상태와 설명은 거래대금 순위 휴리스틱(heuristic)이 아니라 활성 수집 대상의 실제 수집 상태를 요약한다.
- 시장 리스트의 품질 상태도 실제 커버리지 상태에서 산출한다.
- 실제 계산 근거가 없는 rate limit 여유율과 중복 행 수는 API/UI 표시에서 제거했다.
- Playwright 설정은 E2E 포트를 환경변수로 바꾸면 서버 command도 같은 포트를 쓰도록 수정했다.
