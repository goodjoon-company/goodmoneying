## 변경 요약

- 운영 콘솔 디자인을 프론트엔드에 더 가깝게 반영하면서, 백엔드 계약과 맞지 않는 합성 지표를 제거했다.
- `Rate limit 여유` 패널은 최근 1분 수집 행 수 패널로 바꾸고, 실시간 수집과 백필 수집을 분리했다.
- 코인 상세의 `중복 행` 표시는 제거하고 24시간 변동 금액, 등락율, 거래량, 거래량 등락율을 추가했다.
- 헤더의 `CSV 내보내기`, `운영 변경 저장` 버튼과 사이드바 `코인 상세` 메뉴를 제거했다.
- 저장 단위의 주요 표시를 용량보다 Row Count 중심의 저장 행으로 조정했다.
- 기본 `npm run dev:web` 실행에서 `/api` 요청이 로컬 API로 전달되도록 Vite 개발 서버 프록시를 추가했다.
- 구버전 API 응답에 신규 운영 콘솔 필드가 없어도 첫 화면이 검은 화면으로 비지 않도록 프론트엔드 응답 정규화를 추가했다.
- 운영 콘솔 첫 화면을 `design/design_mod_v0.1/운영 콘솔.dc.html`의 사이드바, 상단바, 히어로, KPI 그리드, 히트맵, 저장 Row Count, 구간형 차트, 운영 헬스, 코인별 수집 상태 구조에 맞춰 재정렬했다.
- 실제 `localhost:5173`에서 시장 리스트 진입 시 빈 화면이 되던 원인은 이전 API 프로세스가 `MarketListRow.storageRowCount` 없는 응답을 반환하던 상태였으며, 개발 프로세스 재시작 후 최신 API 계약으로 정상 렌더링됨을 확인했다.

## 리스크

- 24시간 거래량 등락율은 저장된 캔들 데이터 범위에 의존한다.
- 실제 rate limit 헤더나 중복 저장 시도 관측 저장소가 생기기 전까지 해당 지표는 계속 미노출로 유지한다.

## 후속 작업

- 업비트 rate limit 헤더 영속화가 추가되면 별도 운영 지표로 재검토한다.
- 업서트 충돌 또는 중복 저장 시도 추적이 추가되면 중복 관련 관측 지표를 별도 계약으로 재검토한다.
