# 2026-06-21 Backfill 관리 후보 목록 정리

Related Task: `docs/Task/M3-T15-2026-06-21-001-Backfill-관리-후보-목록-정리.md`
Related Test: `docs/Test/2026-06-21-Backfill-관리-후보-목록-정리-검증.md`

## 변경 요약

- 좌측 메뉴의 `수집 대상/설정`을 `Backfill 관리`로 변경했다.
- 후보 목록의 활성 컬럼에서 보이는 `수집` 텍스트를 제거하고 체크박스 접근성 라벨만 유지했다.
- 거래대금 컬럼명을 `24시간 거래대금`으로 변경했다.
- 후보 목록의 품질 컬럼과 품질순 정렬을 제거했다.
- 수집된 범위를 `수집 시작일`과 `수집 최종일` 컬럼으로 분리하고, 실시간 수집 대상에는 `실시간` 라벨을 표시했다.
- 후보 유니버스(Candidate Universe) API 응답과 OpenAPI 계약에 `collectedStartAt`, `collectedEndAt`, `isRealtimeTarget` 필드를 추가했다.

## 리스크와 후속 작업

- `collectionRangeDisplay`는 하위 호환을 위해 유지했다. 모든 클라이언트가 새 필드로 전환되면 제거 여부를 별도 검토한다.
- 수집 시작일과 수집 최종일은 원천 캔들(Source Candle)의 실제 최소/최대 `candle_start_at` 기준이다. 캔들 외 데이터 유형까지 범위를 확장할 필요가 생기면 데이터 유형별 범위 계약을 별도 추가한다.
