# 2026-06-21 worker 현황판 계약과 화면

## 변경 요약

- 운영 상태 첫 카드의 기존 수집 현황 KPI를 worker 현황판으로 교체했다.
- DB 계약에 `collection_worker_heartbeats`를 추가했다.
- API `DashboardSummary`에 `workerStatus`를 추가했다.
- 실시간 수집 워커와 백필 수집 워커가 heartbeat를 기록하도록 했다.
- 오류 수 클릭 시 최근 오류를 보여주는 레이어 팝업을 추가했다.

## 영향

- 운영 화면은 realtime/backfill worker의 동작 상태를 아이콘과 텍스트로 표시한다.
- 실시간 worker는 마지막 수집 시각, 24시간 오류 수, 24시간 실패율을 표시한다.
- 백필 worker는 마지막 수집 시각, 전체 오류 수, 전체 실패율, 현재 백필 동작 코인 수를 표시한다.

## 리스크

- M1에서는 worker별 단일 heartbeat만 유지한다. 다중 worker 인스턴스별 상태는 M3.5 이후 별도 설계가 필요하다.
- heartbeat가 없는 기존 운영 DB에서는 첫 worker 실행 전까지 상태가 `stale`로 표시된다.
