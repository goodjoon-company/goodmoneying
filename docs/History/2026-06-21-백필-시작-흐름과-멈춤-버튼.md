# 2026-06-21 백필 시작 흐름과 멈춤 버튼

Related Task: `docs/Task/M3.md`
Related Test: `docs/Test/2026-06-21-백필-시작-흐름과-멈춤-버튼-검증.md`

## 변경 요약

- 백필 계획 생성 팝업의 실행 버튼을 `확인`에서 `백필 시작`으로 변경했다.
- 백필 시작 시 별도 승인 단계 없이 `POST /v1/backfill/jobs`로 `pending` 백필 작업을 바로 생성한다.
- 수집 대상/설정 화면의 백필 계획 승인 버튼과 로컬 승인 대기 목록을 제거했다.
- 백필 작업 패널에 멈춤(Pause) 버튼을 추가하고 기존 중지(Stop), 삭제(Delete) 제어와 함께 표시한다.
- 제품(Product), 아키텍처(Architecture), API 계약(Contract), 사용설명서를 새 백필 시작 흐름으로 갱신했다.

## 운영 영향

- 운영자가 백필 범위를 입력하고 `백필 시작`을 누르면 작업은 DB에 저장된 순서대로 백필 수집 워커(Backfill Collection Worker)에 의해 실행된다.
- `pending` 작업도 멈춤(Pause)할 수 있어 아직 실행 전인 작업을 대기열에서 일시정지할 수 있다.

## 리스크와 후속 작업

- `POST /v1/backfill/plans`는 예상 요청 수 산정용 계약으로 남아 있지만 현재 화면에서는 직접 사용하지 않는다. 후속으로 별도 견적(Estimate) UI가 필요 없다고 결정되면 계약 축소를 검토한다.
- 전체 검증 결과는 관련 Test 문서에 추가한다.
