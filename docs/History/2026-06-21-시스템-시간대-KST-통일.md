# 2026-06-21 시스템 시간대 KST 통일

## 변경 요약

- 공유 시간 유틸리티(shared time utility)를 KST(Korea Standard Time) 기준으로 전환했다.
- PostgreSQL(PostgreSQL) 연결 세션, DB 기본 timezone 계약, Docker Compose와 Dockerfile 런타임 시간대를 `Asia/Seoul`로 고정했다.
- slim/alpine 기반 Docker 이미지에서 `Asia/Seoul` 이름을 안정적으로 해석하도록 `tzdata` 설치를 명시했다.
- `dev.sh`로 실행하는 로컬 API, 웹, 실시간 수집 워커(Realtime Collection Worker), 백필 수집 워커(Backfill Collection Worker)에 `TZ=Asia/Seoul`, Python/DB 프로세스에는 `PGTZ=Asia/Seoul`을 주입한다.
- 백필 계획 생성 화면은 KST 입력을 명시하고 API payload를 `+09:00` 오프셋 ISO 문자열로 보낸다.
- API 계약(contract), 제품 문서, 아키텍처 문서의 시간대 정책을 KST 기준으로 갱신했다.

## 결정

- DB 타입은 `TIMESTAMPTZ`를 유지한다. 절대 시점 보존과 PostgreSQL 시간대 표시를 분리하기 위해 타입을 바꾸지 않고 세션/DB timezone을 KST로 고정한다.
- 업비트(Upbit) 외부 API 요청 경계에서는 provider protocol에 맞춰 UTC(Coordinated Universal Time) 변환을 유지한다. 내부 row와 저장소(repository) 모델은 KST로 변환한다.
