# 2026-06-22 M3-T19 실시간 체결 빈도 히트맵 전환

## 변경 요약

- 운영 대시보드의 실시간 히트맵(Heat Map)을 수집 row 충실도 기준에서 체결 이벤트(Trade Event) 빈도 기준으로 전환했다.
- 업비트 웹소켓(WebSocket) 구독에 공개 시세 `trade` 타입을 추가했다.
- `TradeEvent` 모델과 `trade_events` DB 테이블을 추가하고, SQLite/Postgres 저장소에 체결 이벤트 기록 경로를 추가했다.
- 최근 24시간 관심 코인별 시간당 체결 이벤트를 집계해 `tradeCount`, `averageTradesPerMinute`, `tradeStrength`, `tradeVolume`, `tradeAmount`를 계산한다.
- 분당 평균 체결 빈도 기준 색상 단계를 `red`, `orange`, `yellow`, `blue`, `green`으로 변경했다.
- API 계약(OpenAPI)의 `RealtimeCollectionHeatmapCell` 스키마를 체결 빈도 지표 구조로 갱신했다.
- 운영 화면 히트맵 제목, 범례, 셀 색상, 마우스 오버 상세를 체결 빈도 기준으로 변경했다.
- 관련 아키텍처 문서와 업비트 수집 파이프라인 문서를 체결 이벤트 저장과 히트맵 집계 기준에 맞게 갱신했다.

## 리스크와 후속 작업

- 실제 업비트 웹소켓 live 연결은 외부 API 호출 없이 계약과 변환 로직 중심으로 검증했다.
- `trade_events` 보존 기간, 파티셔닝(Partitioning), 시간별 사전 집계 테이블은 데이터 증가량을 관측한 뒤 별도 Task로 결정한다.
- 체결강도 계산은 공개 체결 이벤트의 `ask_bid` 기준이며, 개인 주문 체결(MyOrder)이나 계정별 체결 상태를 의미하지 않는다.
