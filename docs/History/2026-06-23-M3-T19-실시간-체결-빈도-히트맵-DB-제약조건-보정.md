# 2026-06-23 M3-T19 실시간 체결 빈도 히트맵 DB 제약조건 보정

## 변경 요약

- 기존 PostgreSQL 개발 DB에서 `trade_event` data type을 허용하지 않는 check constraint 때문에 realtime worker가 실패하던 문제를 보정했다.
- `docs/contracts/db/schema.sql`에 idempotent migration 블록을 추가해 `collection_runs_data_type_ck`, `target_collection_results_data_type_ck`가 `trade_event`를 포함하지 않으면 drop/add로 갱신하도록 했다.
- DB 계약 테스트를 추가해 향후 체결 이벤트(Trade Event) data type 제약조건 보정 SQL이 빠지지 않도록 했다.

## 리스크와 후속 작업

- 현재 schema 적용은 repository 초기화 시 실행된다. 장기적으로는 별도 migration runner와 schema version table을 두는 편이 안전하다.
- 운영 DB 적용 시에는 애플리케이션 동시 기동보다 migration을 먼저 실행하는 배포 순서가 더 안정적이다.
