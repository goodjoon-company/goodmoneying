# 2026-06-30 관심종목 화면과 관심목록 토글

## 변경 요약

- 운영 콘솔의 `시장 리스트` 메뉴를 `관심종목`으로 변경했다.
- `관심종목` 메뉴를 데이터 수집관리 그룹 밖의 최상단 1레벨 메뉴로 이동했다.
- 운영 화면 공통 상단에 관심 코인 개수 버튼을 추가하고, 버튼 클릭 시 관심 코인 목록 레이어 팝업(layer popup)을 표시하도록 했다.
- 관심목록은 신규 테이블이 아니라 기존 활성 수집 대상(Active Collection Target)의 사용자 표현으로 정의했다.
- 관심목록 순서는 `collection_targets.target_order`에 저장하고, `/v1/collection-targets`, `/v1/market-list`, `/v1/dashboard/summary`가 같은 순서를 사용하도록 했다.
- 후보 유니버스(Candidate Universe)는 `favoriteOrder`를 내려주며, Backfill 관리 저장 요청은 후보 표시 순서가 아니라 기존 관심목록 순서를 보존한다.
- `/v1/market-list`는 활성 50개만 반환하던 계약에서 후보 유니버스(Candidate Universe) 100개 기준 관심종목 View Model을 반환하도록 확장했다.
- 관심 별 토글은 기존 `PUT /v1/collection-targets`를 사용해 운영 상태와 Backfill 관리의 대상 종목 기준을 함께 바꾼다.
- 코인/주식 전환, 화폐 단위 분리 표시, 전일 종가 대비 등락률 헤더, 기준일시, 캔들 커버리지 기간, 1분 캔들 수 표시를 추가했다.
- KRW 현재가와 24시간 거래대금은 소수부를 제거해 표시하도록 정리했다.
- `/v1/market-list`의 캔들 커버리지 시작일은 저장된 1분봉이 없어도 수집 계획(Collection Plan)의 시작일을 내려주도록 보정했다.
- `/v1/market-list/stream` SSE(Server-Sent Events)를 추가해 관심종목 가격 정보를 `marketList` 이벤트로 push한다.
- 관심종목 화면은 `marketList` 이벤트를 받으면 React Query 캐시를 갱신해 현재가, 24시간 거래대금, 등락률, 기준일시를 최신 payload로 바꾼다.
- 관심종목 화면에서 `품질` 컬럼을 제거했다.

## 리스크와 후속 작업

- 주식 탭은 아직 수집 계약과 데이터가 없으므로 빈 상태만 제공한다.
- `/v1/market-list` 응답 행 수가 50개에서 100개로 늘어났으므로 운영 데이터가 커질 때 응답 시간 회귀를 주기적으로 확인해야 한다.
- `/v1/market-list/stream`은 현재 운영 서버의 기존 SSE 패턴을 따라 주기적으로 최신 View Model을 push한다. DB 변경 이벤트 기반 증분 push와 재연결 누락 보정은 후속 고빈도 화면에서 재검토한다.
- 기존 `qualityStatus`, `storageRowCount` 필드는 하위 호환을 위해 계약에 남아 있지만 관심종목 화면의 주요 표시 기준은 각각 제거/`oneMinuteCandleCount`로 이동했다.
