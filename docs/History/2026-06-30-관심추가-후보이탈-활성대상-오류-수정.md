## 변경 요약

- `PUT /v1/collection-targets`가 기존 활성 수집 대상(Active Collection Target)의 후보 유니버스(Candidate Universe) 이탈 상태를 신규 후보 밖 선택으로 오인해 `INVALID_COLLECTION_TARGETS`를 반환하는 문제를 수정했다.
- SQLite/PostgreSQL 저장소(Repository) 모두 신규 선택 ID만 후보 유니버스에 속하는지 검증한다.
- 이미 활성 상태인 ID는 유지 요청에서 재활성화하지 않아 `candidate_status = out_of_universe`가 `in_universe`로 잘못 덮이지 않게 했다.
- 저장소 테스트와 API 테스트에 회귀 시나리오를 추가했다.
