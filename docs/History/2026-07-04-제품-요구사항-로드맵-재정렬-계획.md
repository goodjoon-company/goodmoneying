# 제품 요구사항과 로드맵 재정렬 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. 이 저장소는 `AGENTS.md`에 따라 별도 superpowers 계획 디렉터리를 장기 산출물 위치로 쓰지 않는다. 실행 단위는 GitHub Issue로 만들고, 검증 증적은 `docs/Test/`, 오래 유지되는 변경 요약은 `docs/History/`에 남긴다.

**Goal:** 현재 구현 산출물과 `docs/01_Product.md`의 제품 요구사항, MVP(Minimum Viable Product), 로드맵(roadmap)을 재점검해 앞으로 만들 제품 방향을 단순하고 검증 가능한 기준으로 다시 세운다.

**Architecture:** 이 작업은 기능 구현이 아니라 제품 기준 재정렬 작업이다. 먼저 구현된 산출물을 사실 목록으로 고정하고, 그 다음 사용자가 원하는 제품 방향과 어긋난 지점을 분류한 뒤, `docs/01_Product.md`를 중심으로 아키텍처(Architecture), 계약(Contract), Task/Test/History 색인을 맞춘다. 미래 기능은 먼 마일스톤까지 세부 Task로 쪼개지 않고, 다음 의사결정 게이트와 가까운 실행 단위만 남긴다.

**Tech Stack:** Markdown, GitHub Issue, `rg`, `git diff`, `pytest`, `npm test`, Playwright E2E(End-to-End)

## 현재 관찰

- 현재 `docs/01_Product.md`는 최종 제품 비전을 넓게 담고 있지만, MVP가 운영 콘솔 중심으로 굳어져 있어 사용자가 원한 투자 데이터 플랫폼 방향과 거리가 생겼다.
- M1부터 M3까지 구현 산출물은 업비트 수집, 운영 상태, Backfill 관리, 관심종목, 코인 상세 레이어, 실시간 수집, SSE(Server-Sent Events), 체결 이벤트 히트맵에 집중되어 있다.
- 요구사항 표는 `GM-PROD-001`부터 `GM-PROD-024`까지 세부화되어 있으나 대부분 `Draft`이며, 사용자가 지금 다시 결정해야 할 질문과 이미 구현된 사실이 섞여 있다.
- 로드맵은 M0부터 M11까지 너무 멀리 번호화되어 있고, M4 이후 항목이 구체적 실행 계획처럼 보인다.
- `docs/Task/`는 GitHub Issue 전환 후 Milestone 요약만 남긴다는 기록이 있으나, 상세 Task 파일들이 아직 남아 있어 문서 정책과 실제 파일 구조가 어긋나 있다.
- 로컬 `main`은 `origin/main`보다 2커밋 앞서 있으므로, 제품 재정렬 착수 전 기준 브랜치와 원격 반영 여부를 먼저 결정해야 한다.

## 비범위

- 이 계획은 새 기능을 구현하지 않는다.
- 이 계획은 M4 이후 기능을 세부 Task로 분해하지 않는다.
- 이 계획은 기존 구현을 삭제하거나 되돌리지 않는다.
- 이 계획은 국내 주식, 미국 주식, LLM(Large Language Model), 봇(Bot), 실거래(Live Trading)의 상세 계약을 확정하지 않는다.

- 로드맵은 먼 미래 기능을 세부 Task처럼 쓰지 않고, 결정 게이트와 큰 제품 단계만 남긴다.
