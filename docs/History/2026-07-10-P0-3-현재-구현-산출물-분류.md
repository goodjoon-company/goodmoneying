# P0-3 현재 구현 산출물 분류

## 배경

GitHub Issue #5는 구현된 사실과 앞으로의 제품 우선순위를 분리하는 작업이다. 선행 Issue #4에서 운영 콘솔을 내부 운영 도구로 두고, 사용자가 관심 코인을 비교해 투자 후보를 좁히는 상태를 가까운 MVP 성공 기준으로 결정했다.

## 분류 결과

| 분류 | 산출물 |
|---|---|
| 계속 가져갈 것 | 업비트 상위 100개 후보와 활성 50개, Backfill 관리, 관심종목 화면, 코인 상세 레이어 |
| 운영 도구로 격하할 것 | 운영 상태 대시보드 |
| 보류할 것 | 실시간 체결 히트맵 추가 확장, 호가 원천 스냅샷 저장, M4 이후 다중 워커·고가용성 구현 |
| 삭제 후보 | 과거 `docs/Task/*-T*.md` 상세 계획 문서 10개 |

현재 런타임 기능은 삭제하지 않는다. 실시간 체결 히트맵은 현재 구현을 보존하되 투자 후보 판단 가치가 검증될 때까지 추가 기능을 만들지 않는다.

## 코드/문서 리뷰

- 요구사항 적합성: Issue #5 범위의 주요 산출물 8개와 상세 Task 처리 방침을 모두 분류했다.
- 아키텍처 적합성: 구조를 바꾸지 않고 현재 구현의 제품상 위치만 정했다. 다중화와 호가 원천 저장의 아키텍처 문구는 후속 Issue #7로 연결한다.
- 계약 변경 여부: DB, API, 메시지 계약 변경은 없다.
- 테스트 영향: 런타임 코드와 테스트 기대값을 바꾸지 않는다.
- 보안·성능·운영 안정성: 런타임과 배포 설정을 변경하지 않으므로 새로운 영향은 없다.
- 남은 위험: 실시간 체결 히트맵의 투자 판단 가치는 사용자 시나리오로 검증되지 않았고, 현재 Product의 먼 미래 로드맵은 후속 Issue #6에서 다시 작성해야 한다.
