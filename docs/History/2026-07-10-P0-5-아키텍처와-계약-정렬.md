## 아키텍처 결정

- M1~M3 구현은 완료 기준선이다.
- 현재는 관심종목과 코인 상세의 데이터 신뢰·비교 흐름을 우선한다.
- 호가 원천은 전략 실험에서 필요성이 승인될 때 저장 해상도·깊이·보존·재생·비용을 먼저 결정한다.
- 다중 워커, 메시지 큐, 고가용성, 파티셔닝·복제는 처리량·복구·저장 임계값을 넘을 때 검토한다.
- 주식, 문서·LLM, 전략, 봇, 거래는 별도 제품 결정 뒤 아키텍처와 계약을 설계한다.

## 리스크

- 확장성 임계값은 진입 조건의 종류만 정의했고 구체 수치는 아직 없다. 실제 운영 문제 또는 승인된 확장 요구가 생길 때 수치화해야 한다.
- 과거 ADR 본문에는 당시 M3·M4·M5 계획이 역사적 맥락으로 남아 있다. 각 문서는 ADR-0007 참조를 통해 현재 실행 계획과 구분한다.
- OpenAPI의 운영 콘솔 명칭은 현재 API 역할을 설명하는 계약 용어로 유지했다. 제품 우선순위를 뜻하지 않는다.
