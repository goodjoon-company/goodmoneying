# 수집 후보군 사용자 문구 표준화

Date: 2026-07-11
Related Task: GitHub Issue #11
Related PR: TBD

## 변경 요약

- Backfill 관리 화면 제목과 로딩 문구를 `후보 유니버스`에서 `수집 후보군`으로 바꿨다.
- SQLite/PostgreSQL 저장소(Repository)의 사용자 노출 오류 메시지와 PostgreSQL fixture 차단 메시지를 `수집 후보군` 기준으로 정리했다.
- 실시간 수집 워커(Realtime Collection Worker)의 활성 대상 재동기화 사유와 live backfill 스크립트 안내 문구를 같은 용어로 맞췄다.
- 관련 frontend 단위 테스트, 저장소 단위 테스트, Playwright E2E 기대값과 테스트 제목을 먼저 바꾼 뒤 구현했다.

## 영향 문서

- 표준 용어 정의는 기존 `UBIQUITOUS_LANGUAGE.md`, `docs/01_Product.md`, `docs/02_Architecture.md`, `docs/contracts/api/openapi.yaml`를 그대로 따른다.
