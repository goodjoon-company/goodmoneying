# 개발 사양 문서 재구성

Date: 2026-07-14
Related Issue: [#15](https://github.com/goodjoon-company/goodmoneying/issues/15)

## 변경 요약

- `docs/README.md`를 개발 사양 진입점으로 바꾸고 제품 → 아키텍처 → 모듈 → 계약 → 사용·검증의 읽는 순서를 명시했다.
- 제품 문서는 PRD(Product Requirements Document) 역할과 사양 요약을 앞에 두고 기존 요구사항·정책·로드맵을 단일 기준으로 유지했다.
- 아키텍처 문서는 시스템 문맥, 소프트웨어 레이어, 수집·집계·분석 데이터 흐름, 브라우저 실시간 전송을 Mermaid 다이어그램으로 설명한다.
- 업비트 수집 파이프라인, 개발·운영 사용 안내, 코인 분석 빠른 안내를 현재 세 워커와 `dev.sh` 실행 방식에 맞췄다.
- 계약 인덱스와 API·DB 안내는 소비자, 기계 검증 파일, 변경 순서를 명확히 했다.

## 보존한 경계

- API 경로, DB schema, WebSocket 스키마, 런타임 코드와 ADR 결정은 변경하지 않았다.
- 기계 검증 가능한 상세는 `docs/contracts/`에만 두고 제품·아키텍처 문서에 복제하지 않았다.
- 실행 계획은 GitHub Issue #15로, 검증 증적은 `docs/Test/2026-07-14-개발-사양-문서-재구성-검증.md`로 분리했다.

## 리스크와 후속 작업

- Mermaid CLI가 저장소에 없어 실제 렌더링 자동 검증은 하지 못했다. 문서 사이트 또는 렌더링 도구를 도입하면 CI(Continuous Integration)에서 Mermaid 렌더링 검사를 추가한다.
- 현재 작업 트리에는 코인 분석·집계 기능의 별도 미커밋 변경이 있다. 이번 문서는 그 구현의 현재 구조를 설명하지만, 해당 기능 병합 전에 기능별 검증 증적과 계약 리뷰를 별도로 유지해야 한다.
