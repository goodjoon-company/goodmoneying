# 업비트 API 테스트 페이지 인계

Date: 2026-07-14
Issue: [#14](https://github.com/goodjoon-company/goodmoneying/issues/14)

## 변경 요약

- 좌측 메뉴에 개발용 `업비트 API 테스트` 화면을 추가했다.
- 브라우저가 업비트 공개 캔들 API를 직접 호출해 거래쌍별 분·시·일·주·월 OHLCV를 조회한다.
- 최신순 응답을 시간 오름차순으로 정렬하고, 캔들·거래량·SMA 20·EMA 20·볼린저 밴드·RSI 14와 최근 10개 OHLCV 표를 표시한다.
- 입력 오류, HTTP 실패, 빈 조회 상태를 분리해 표시하고, 조회 후 입력값을 바꿔도 이미 표시된 데이터의 조건을 잘못 표기하지 않는다.
- 새 조회는 이전 요청을 취소하고 최신 요청만 화면에 반영한다.

## 경계와 리스크

- 이 화면은 개발·검증 도구다. 내부 API, DB, 수집·집계 워커, 기존 코인 분석 WebSocket 계약을 사용하거나 변경하지 않는다.
- 업비트 공개 API의 호출 제한·가용성은 외부 의존성이다. 화면은 사용자가 명시적으로 조회할 때만 요청한다.
- Ruff는 통과했다. 전체 Mypy는 이번 변경과 무관한 `aggregation_collection_worker.py`의 `repository` 속성 오류 2건 때문에 실패한다. 세부 증적은 `docs/Test/2026-07-14-업비트-API-테스트-페이지-검증.md`를 따른다.

## 검증과 다음 작업

- 웹 단위 테스트 48개, Python 테스트 159개, Playwright E2E 4개, 웹 빌드가 통과했다.
- 다음 작업은 `aggregation_collection_worker.py`의 `repository`/`_repository` 속성 불일치를 해당 집계 작업 Issue에서 정리해 전체 Mypy 녹색 상태를 회복하는 것이다.
