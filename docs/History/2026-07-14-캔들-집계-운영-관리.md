# 2026-07-14 캔들 집계 테이블과 시스템 관리

## 변경 요약

- 1분 원천봉과 일봉을 `5m/10m/30m/60m/1d/1w/1M` 분석용 집계 테이블로 멱등 저장하는 모델, DB 계약, 작업·대상 진행률을 추가했다.
- 최신 시각이 원천보다 오래된 활성 코인·주기를 자동 감지해 집계 작업을 만들고 별도 캔들 집계 워커가 처리한다.
- `시스템 관리` 메뉴와 `/v1/realtime/system-management` WebSocket을 추가해 실시간 수집·Backfill·집계의 코인별 대상, 데이터 유형, 상태, 진행률을 작은 상태 메시지로 갱신한다.
- 로컬·배포 프로필에 `candle-aggregation-worker`를 추가했다.
- 2026-07-16 완료 기준 재검토에서 시스템 관리 WebSocket과 화면에 집계 워커의 실제 heartbeat 상태·최신 시각을 추가하고, 집계 작업의 대기 대상 수를 전체·완료·실행·실패 수와 함께 표시하도록 보완했다.

## 결정과 리스크

- 결정 근거는 [ADR-0009](../ADR/ADR-0009-캔들-집계-테이블과-자동-워커.md), 계약은 [시스템 관리 WebSocket](../contracts/api/realtime-system-management-websocket.md)을 따른다.
- DB 스키마 변경은 없다. 기존 `collection_worker_heartbeats`의 `candle_aggregation` 행과 집계 대상 상태를 API 뷰 모델에 노출하는 변경이다.
- 현재 작업 환경에는 Podman이 없어 실제 PostgreSQL 컨테이너 장기 실행은 검증하지 못했다. PostgreSQL 계약은 정적 타입 검사·저장소 테스트·격리 E2E로 확인했으며, 배포 전 컨테이너 smoke test가 남아 있다.
