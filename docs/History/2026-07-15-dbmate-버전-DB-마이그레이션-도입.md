## 변경 요약

- `docs/contracts/db/migrations/`를 DB 변경 이력의 단일 기준(source of truth)으로 만들고 현재 스키마를 기준선 `20260715000100`으로 등록했다.
- `schema.sql`은 고정 PostgreSQL 17.10 환경에서 생성·정규화하고 CI가 드리프트(drift)를 검사하는 스냅샷으로 전환했다.
- API·워커 저장소 생성 시 전체 DDL(Data Definition Language)을 실행하던 동작을 제거했다.
- `dev.sh db new|migrate|status|dump|rollback`을 추가하고 앱 시작 전에 미적용 변경을 자동 실행한다.
- 기준선의 빈 `down`이 적용 이력만 제거하지 못하도록 rollback을 차단했다.
- `pg_dump`가 없는 개발 머신은 격리 Docker 설정과 고정 dbmate 이미지로 스냅샷을 생성하며 로컬 URL을 컨테이너 호스트로 안전하게 변환한다.
- prod-home 배포에 같은 릴리스 태그의 일회성 마이그레이션 서비스를 추가하고 최대 60초 DB 대기 후 성공한 경우에만 API·워커를 갱신한다.
- dbmate 기반 이미지는 버전과 다이제스트(digest)를 고정하고 배포 이미지는 전체 Git SHA 태그를 사용한다.
- README의 `dev.sh` 사용법에 마이그레이션 작성·적용·운영 경계를 추가했다.

## 운영 영향

- DB 서버에는 추가 바이너리나 SQL 파일 설치가 필요 없다.
- 앱 서버의 `app.env`는 실제 DB URL과 SSL 모드, 마이그레이션에 필요한 DDL 권한을 제공해야 한다.
- 운영 DB 변경 전 백업은 필수이며 자동 rollback 대신 순방향 수정(forward fix)을 사용한다.
- 이후 적용된 마이그레이션은 수정하지 않고 새 타임스탬프 파일로 변경한다.
