Verification: [검증 증적](../Test/2026-07-16-업비트-API-게이트웨이-계약-검증.md)

## 변경 요약

- 브라우저 직접 업비트 연결을 제거하는 별도 `apps/upbit_gateway/` 경계를 아키텍처와 ADR로 확정했다.
- 공식 v1.6.3 REST 51개와 WebSocket 15개 기능의 파라미터·제한 그룹·안전 등급 카탈로그를 추가했다.
- 실제 주문, 모든 주문 취소, 포켓 자산 이전, 출금·입금 상태 변경, 트래블룰 검증을 `blocked`로 고정하고 공식 주문 생성 테스트만 `test`로 허용했다.
- 임의 URL 대신 `endpoint_id`만 받는 OpenAPI와 연결·구독·프레임·오류 WebSocket 추적 스키마를 정의했다.
- FastAPI health·catalog와 항상 501을 반환하는 실행 경계를 추가했다. 업비트 상향 HTTP·WebSocket 호출은 포함하지 않았다.
- Python 경로와 Ruff·mypy 대상에 새 패키지를 추가하고 계약·TestClient 자동 테스트를 연결했다.

## 운영 영향

현재 게이트웨이는 별도 실행·배포 구성에 연결되지 않았고 키나 네트워크 연결을 사용하지 않는다. 따라서 기존 운영 서버·워커·DB 런타임 영향은 없다. 후속 Issue가 실행 엔진을 도입할 때 키 저장, 포켓별 제한, 429·418 복구, 로그 마스킹과 배포 모니터링을 함께 검증해야 한다.
