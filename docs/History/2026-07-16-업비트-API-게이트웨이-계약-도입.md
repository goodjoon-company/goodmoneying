# 2026-07-16 업비트 API 게이트웨이 계약 도입

Date: 2026-07-16
Status: Completed
Related Issue: [#19](https://github.com/goodjoon-company/goodmoneying/issues/19)
Related ADR: [ADR-0011](../ADR/ADR-0011-업비트-API-게이트웨이와-비파괴-테스트-경계.md)
Verification: [검증 증적](../Test/2026-07-16-업비트-API-게이트웨이-계약-검증.md)

## 변경 요약

- 브라우저 직접 업비트 연결을 제거하는 별도 `apps/upbit_gateway/` 경계를 아키텍처와 ADR로 확정했다.
- 공식 v1.6.3 REST 활성 50개·사용 중단(deprecated) 1개와 WebSocket 데이터 스트림 14개·운용 1개의 파라미터·제한 그룹·안전 등급 카탈로그를 추가했다.
- 실제 주문, 모든 주문 취소, 포켓 자산 이전, 출금·입금 상태 변경, 트래블룰 검증을 `blocked`로 고정하고 공식 주문 생성 테스트만 `test`로 허용했다.
- 임의 URL 대신 `endpoint_id`만 받는 OpenAPI와 연결·구독·프레임·오류 WebSocket 추적 스키마를 정의했다.
- FastAPI health·catalog와 차단 403·알려진 미구현 501·알 수 없음 404·요청 오류 422를 구분하는 실행 경계를 추가했다. 업비트 상향 HTTP·WebSocket 호출은 포함하지 않았다.
- Python 경로와 Ruff·mypy 대상에 새 패키지를 추가하고 계약·TestClient 자동 테스트를 연결했다.

## 코드 리뷰 보완

- `LIST_SUBSCRIPTIONS`를 데이터 스트림 목록에서 분리하고 게이트웨이 `list` 운용으로 연결했다.
- 체크인된 OpenAPI의 카탈로그 응답을 구체 스키마(schema)로 바꾸고 FastAPI 런타임과 응답 상태 코드 동등성을 자동 검증한다.
- 공식 REST `endpoint_id`·메서드·경로·파라미터 투영의 SHA-256 스냅샷(snapshot)을 추가해 계약 드리프트(drift)를 검출한다.
- 공식 요청 제한 그룹명 `default`와 `Origin` 발동 조건·적용 대상·그룹·한도를 분리해 표현했다.
- 프로젝트를 설치 가능 패키지로 구성하고 임시 작업 디렉터리(CWD)에서도 카탈로그를 읽는 회귀 테스트를 추가했다.

## 패키징·계약 의미 동등성 보완

- 실제 wheel에 계약 카탈로그 복사본을 패키지 데이터(package data)로 포함하고 `importlib.resources`로 읽는다. `docs/contracts/`는 계속 단일 기준(source of truth)이며 자동 테스트가 두 파일의 바이트 동등성을 검사한다.
- 임시 복제본에서 `uv build`를 실행하고 새 가상환경(Virtual Environment)에 wheel을 설치한 뒤, 별도 임시 작업 디렉터리(CWD)에서 카탈로그를 읽는 자동 테스트를 추가했다. 빌드 산출물은 모두 임시 디렉터리에 격리한다.
- 체크인 OpenAPI와 FastAPI 런타임의 `Health` 참조, URI 형식, REST 51·WebSocket 14·운용 1 배열 제약을 구성요소 스키마(component schema) 단위로 비교한다.
- 해시(hash)만 저장하던 공식 REST 스냅샷(snapshot)을 `endpoint_id`·메서드·경로·파라미터 전체가 보이는 명시적 투영으로 교체했다.

## 운영 영향

현재 게이트웨이는 별도 실행·배포 구성에 연결되지 않았고 키나 네트워크 연결을 사용하지 않는다. 따라서 기존 운영 서버·워커·DB 런타임 영향은 없다. 후속 Issue가 실행 엔진을 도입할 때 키 저장, 포켓별 제한, 429·418 복구, 로그 마스킹과 배포 모니터링을 함께 검증해야 한다.
