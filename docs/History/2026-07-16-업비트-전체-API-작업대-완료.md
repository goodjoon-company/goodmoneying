# 업비트 전체 API 작업대 완료

Date: 2026-07-16
Related Issues: #19~#24
Verification: `docs/Test/2026-07-16-업비트-전체-API-작업대-통합-검증.md`

## 변경 요약

- 업비트 API 테스트 아래에 Quotation API 테스트, Exchange API 테스트, WebSocket API 테스트 메뉴를 완성했다.
- 정식 카탈로그 기반 REST 51종과 WebSocket 14종을 모두 탐색하고 안전 범위에서 실행할 수 있다.
- 공통 거래쌍과 파생 마켓(Quote)·기준 자산(Base)을 세 작업대에서 공유한다.
- 입력 범위가 정해진 값은 선택 필드, 날짜는 날짜·시간 필드, 식별자·수량·가격은 직접 입력 필드로 제공한다.
- 차트·표·호가창·상태 카드·실시간 뷰와 원본 응답·API 출처 추적 팝업을 제공한다.
- 캔들 차트 가장자리 이동과 버튼으로 과거·미래 데이터를 연속 조회한다.
- 별도 업비트 API 게이트웨이를 개발·운영 스크립트, 배포 프로필, 상태 점검, 아키텍처 문서와 다이어그램에 반영했다.

## 안전 정책

- 조회와 공식 주문 생성 테스트만 업비트로 전송한다.
- 실제 주문·취소·자산 이전·입출금·Travel Rule 상태 변경 14종은 로컬에서 영구 차단한다.
- 브라우저는 키를 알지 못하며 게이트웨이가 서버 자격 증명과 요청 제한을 관리한다.
- 첨부 키를 사용한 실제 검증에서도 주문 생성 테스트의 정상 검증 오류까지만 확인했고 실제 주문은 만들지 않았다.

## 운영 인계

- 개발: `./dev.sh app start upbit-gateway`
- 안전 실연동: `PYTHONPATH=apps/upbit_gateway uv run python scripts/verify_upbit_safe_live.py --key-file <키 파일>`
- 운영: `prod-home` 앱 서버의 `upbit-gateway` 서비스, 8001 상태 점검, 웹의 같은 출처 `/upbit-gateway` 프록시를 사용한다.
- 요청 제한은 `Origin`이 있는 브라우저 직접 호출의 10초당 1회가 아니라, 서버 게이트웨이의 공식 그룹별 제한을 적용한다.

## 남은 비차단 리스크

- 웹 프로덕션 번들(bundle) 500KB 초과 경고는 후속 코드 분할(code splitting) 후보이다.
- 업비트 공식 엔드포인트(endpoint)·요청 제한·스키마(schema)는 변경될 수 있으므로 카탈로그 갱신 시 공식 문서를 다시 확인해야 한다.
- Secret Key 길이 40바이트 경고가 있어 키 교체 시 64바이트 이상을 권장한다.
