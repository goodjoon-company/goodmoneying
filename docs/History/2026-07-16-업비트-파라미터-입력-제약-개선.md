# 업비트 파라미터 입력 제약 개선 인계

Date: 2026-07-16
Related Product: GM-PROD-034
Related Issue: [#25](https://github.com/goodjoon-company/goodmoneying/issues/25)
Verification: `docs/Test/2026-07-16-업비트-파라미터-입력-제약-검증.md`

## 변경 요약

- Quotation과 Exchange가 `CatalogParameterField`와 공통 직렬화·검증 규칙을 사용하도록 중복 입력 구현을 통합했다.
- 날짜·시간 입력은 네이티브 달력·시계, 초 단위 `step=1`, KST·UTC 안내, 현재 시각(Now), 선택값 지우기를 제공한다.
- 캔들 `count` 초기값을 임의 최대값 200에서 공식 기본값 1로 바꿨다.
- 기본·최소·최대·단위·기간·최대 항목 수·동적 제한 출처를 도우미 문구와 빠른 입력 버튼으로 제공한다.
- 숫자 형식 문자열은 입력·전송에서 문자열을 보존하고 검증 시에만 숫자로 해석한다.
- 범위·정수·배열·CSV·중복·날짜 기간 오류는 첫 필드에 포커스하고 게이트웨이 호출 전에 차단한다.
- 게이트웨이는 같은 카탈로그로 타입·형식·범위·기간·항목 수를 다시 검증한다.

## 계약 변경

`docs/contracts/upbit/upbit-api-catalog.yaml`에 `default`, `step`, `unit`, `max_items`, `unique_items`, `timezone`, `range_with`, `range_max_seconds`, `dynamic_constraint_source`를 추가했다. 공식 스냅샷과 패키지 데이터 복사본을 같은 내용으로 동기화했다.

입출금 목록의 `from`·`to`를 `date-time`에서 `cursor`로 바로잡고, 호가·주문·자산 이전·입출금의 숫자 형식 문자열을 표시 의미에 맞게 보강했다. 취소 후 재주문 `new_volume`은 `remain_only` 특수값을 허용하므로 숫자 전용 입력으로 강제하지 않았다.

## 안전과 호환성

- 실제 주문·취소·자산 이전·입출금·트래블룰 검증의 `blocked` 안전 등급은 바꾸지 않았다.
- 게이트웨이 외부 경로·OpenAPI·DB·메시지 계약은 바꾸지 않았다.
- 동적 가격·수량 제한은 정적 추정값을 만들지 않고 공식 주문 가능 정보·호가 정책·출금 가능 정보 기능을 출처로 표시한다.
- 밀리초 타임스탬프는 실제 Unix 밀리초로 해석해 ISO 8601과 같은 기간 검증을 적용한다.

## 후속 참고

- 네이티브 날짜·시간 선택기의 팝오버 모양은 브라우저·운영체제에 따라 다르지만 입력 타입·초 단위·직렬화 계약은 동일하다.
- 향후 동적 제한을 자동 조회해 실값으로 채우려면 자격 증명 상태·페어 변경·캐시·호출 제한을 함께 설계해야 하며 이번 변경은 공식 출처 표시까지로 경계를 유지한다.

## 2026-07-17 시각 QA 후속 수정

- 어두운 Quotation 패널의 네이티브 달력·시계 아이콘이 배경에 묻히는 사용자 피드백을 반영했다.
- Quotation 날짜·시간 입력에는 어두운 네이티브 테마, 밝은 Exchange 패널에는 밝은 네이티브 테마를 명시했다.
- 브라우저 계산 스타일을 직접 검증하는 Playwright 회귀 테스트를 추가했으며 전체 E2E 15건이 통과했다.
- API 계약, 날짜 직렬화, 게이트웨이 검증과 안전 등급에는 변경이 없다.
