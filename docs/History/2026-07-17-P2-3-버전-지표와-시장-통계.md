# P2-3 버전 지표와 시장 통계 구현

날짜: 2026-07-17
관련 Issue: [#29](https://github.com/goodjoon-company/goodmoneying/issues/29)

## 변경 요약

- SMA20·SMA60·EMA20·볼린저 밴드20·RSI14 정의 버전과 결정론적 Decimal 계산을 추가했다.
- 원천/집계 개정과 품질 프런티어를 추적하는 추가 전용 지표 물질화·값·시장 통계·무효화 계약을 추가했다.
- 원천·집계·품질 변경을 DB 시계 임대와 세대 fencing으로 처리하고 기존 이력 업그레이드는 상품·주기당 한 작업으로 제한했다.
- 최근 계산 상태를 행별 체크포인트로 저장하고 512구간 단위 `progress_at` 재개를 추가해 긴 과거 정정을 제한된 메모리로 처리한다.
- 같은 상품·주기의 선행 프런티어와 재개 청크를 ID 순서로 끝까지 처리한 뒤 후속 프런티어를 선점하도록 멀티 워커 순서를 고정했다.
- 영향 시작보다 먼 체크포인트 또는 체크포인트가 없는 복구 예열은 큐의 일시 계산 상태와 512개 bounded 물질화를 함께 전진시켜 진행 정체 없이 모든 부모·값·현재 입력 계보를 보존했다.
- 물질화 삽입과 큐 완료를 하나의 fenced 트랜잭션으로 묶어 완료 직전 실패에도 부분 지표·통계가 남지 않게 했다.
- 저장된 결과만 읽는 범위·`asOf`·정의 버전·고정 ID 상한 cursor REST 조회를 추가했다.
- cursor에 상품·주기·조회 범위를 결합하고, WebSocket 최신 1,000개 캔들과 저장 지표의 반개방 시간 경계·정렬을 일치시켰다.
- 런타임 권한 자동 추출 대상에 지표 저장소 SQL을 포함하고 P2-3 읽기·삽입·큐 갱신 권한을 준비도 검사에 반영했다.
- 분석 WebSocket을 준비 기간 `null`/상태/정의·물질화 계보 필드로 확장하고 최신 upsert와 과거 전체 갱신을 분리했다.

## 호환성과 후속 작업

- 기존 캔들·분석 경로는 유지한다. PostgreSQL 지표 GET은 요청 시 계산하거나 쓰지 않는다.
- P2-4 이후 Data Lab은 지표 `definitionSetHash`, 시장 통계 계산 버전, 원천·품질 프런티어를 데이터셋 입력으로 고정해야 한다.
- 별도 지표 워커 프로세스는 큐 지연·처리량 측정으로 집계 워커 유휴 처리 방식의 한계가 확인될 때 분리한다.
