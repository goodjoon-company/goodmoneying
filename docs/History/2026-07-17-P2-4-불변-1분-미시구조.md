# P2-4 불변 1분 미시구조 변경 기록

날짜: 2026-07-17

## 사용자 결과

수집 이후의 1분 구간에서 호가 스프레드·상위 10단계 잔량·주문장 불균형과 매수(`BID`)·매도(`ASK`) 표시 체결 빈도·수량·불균형·체결 강도를 계산 버전과 원천 계보로 고정한다. 과거 미수집 구간과 불완전한 입력은 합성하지 않는다.

## 구현 경계

- receipt와 정규화 체결·호가 행의 직접 계보와 원자적 더티 범위 생성
- 이벤트 입력을 1분 범위로 합친 내구성 무효화·재계산
- 추가 전용 정의·물질화·통계와 부모·원천·품질 프런티어
- 연결 시작·종료 부분 분, 장애·재연결 공백과 완전 연결 구간의 불변 품질 증거
- 의미 프런티어별 직렬 A→B→A 이력, 선행 작업 대기 시 재시도 예산 보존과 임대 세대 회수
- 저장 결과 전용 REST 페이지와 분석 WebSocket 청크·최신 upsert·과거 정정 refresh
- SQLite 요청 시 계산 금지와 빈 테스트 호환 투영

## 의미 결정

- 공식 Upbit 용어대로 `BID`는 매수, `ASK`는 매도다. 공격자·메이커·테이커 의미는 부여하지 않는다.
- 호가 모아보기는 원천에 보존하지만 v1 계산은 비교 가능한 `level=0`만 사용한다.
- 분모 0, 10단계 미만, 가격 역전, 체결·캔들 대사 불일치는 임의 값 대신 계산 상태와 원천 품질로 노출한다.
- 다주기 미시구조, 주식시장과 타 거래소는 포함하지 않는다.

## 후속 연결

P2-5 데이터셋 버전은 `microstructure-v1`, 물질화 ID 상한, 원천·품질 프런티어와 결측 처리 정책을 입력에 포함해야 한다. P2-7·P2-8은 현재 분석 WebSocket 메시지를 순번·커서·REST 스냅숏 복구 envelope로 감싼다.
