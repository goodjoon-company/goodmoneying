# P2-5 불변 데이터셋 버전 변경 기록

날짜: 2026-07-17

## 사용자 결과

연구자는 조회 조건만 저장하는 대신 수락 시점의 원천·품질·시장 상태 프런티어와 정확한 캔들·지표·시장 통계·미시구조 입력을 불변 데이터셋 버전으로 고정할 수 있다. 같은 내용은 같은 내용 해시로 재사용되며 이후 늦은 수집·정정은 기존 버전을 바꾸지 않는다.

## 구현 경계

- 멱등 생성 명령과 변경 가능한 빌드 수명주기
- 수락 트랜잭션에서 `asOf`와 모든 입력 ID 상한 고정
- 원천·집계·지표·시장 통계·미시구조 typed member의 정확한 외래키와 인과 시각
- 시점별 시장 상태와 5단계 coverage 불변 snapshot
- 대리키 독립 canonical selection·manifest·상태·coverage·content hash
- 임시 스테이징, 4,096행 서버 커서 증분 해시와 집합 기반 삽입
- 별도 연결 heartbeat, 소유자·임대 세대·만료 시각 fencing, 제한 재시도와 dead-letter
- 봉인 전 child 완성, 봉인 뒤 root·모든 child의 UPDATE·DELETE·INSERT 차단
- 저장된 빌드·버전·coverage·정확 member 전용 REST와 문맥 고정 커서

## 의미 결정

- 데이터셋은 결측 채움 결과를 저장하지 않는다. `null`, `drop`, `no_trade_carry_forward_v1` 적용은 P3·P4 소비 계층의 책임이다.
- `missing_policy=fail`은 `missing`, `unavailable`, `unverified`와 관측 부족을 차단하지만 확정 `no_trade`는 허용한다.
- 내용 해시는 DB 대리키와 생성 시각을 제외하며 시장 자연키, 범위, 정책, 정의·계산 버전, 정렬된 member 내용, 시장 상태와 coverage를 포함한다.
- 일봉은 직접 1일봉을 우선하고 없는 날짜만 1분 집계를 사용하며 `daily-source-preferred-v1` 의미로 고정한다.
- 주·월 원천 폴백을 포함한 모든 조회 결과는 UTC 반개방 범위 `[from, to)` 밖의 버킷을 반환하지 않는다.

## 후속 연결

P2-6 Data Lab은 이 REST 계약을 사용해 빌드 생성·상태, 버전 목록·상세, coverage와 제한된 exact member를 표시한다. 완료 버전은 편집하지 않고 새 빌드로 복제한다. P2-7은 순번·커서가 있는 내부 스트림을, P2-8은 snapshot·cursor 재동기화를 추가했다.
