## 공식 Upbit 기준

2026-07-17 개발자센터 v1.6.3을 확인했다. Exchange 한도는 포켓 단위이며 order-test 8 req/s, private myOrder는 주문조회 권한, order-test는 주문하기 권한을 각각 요구한다. 분봉은 최대 200개, 초봉은 요청 시점 최근 3개월, 무체결 캔들은 생성되지 않는다. 실제 matching sandbox는 없고 `/v1/orders/test`는 주문을 생성하지 않는 사전 검증 API다.

## 용량·시간 위험

공식 market API snapshot은 전체 783, KRW 269개다. 2024-01-01 이후 1분 candle 상한은 약 3.59억 행, 이론 최소 179만 REST 요청과 전용 10 req/s에서 49.9시간이다. 실제 저장량은 측정 전 약 135~360GB 계획 범위이며 현재 운영 DB 17GB에 무검증 활성화할 수 없다. Issue #28에서 실측 row size, index, partition query plan, disk·backup, throttle을 먼저 통과한다.

## 배포 차단 조건

- 최신 main CI 실패
- 114 commit·322 file의 운영 격차
- Gateway secret·bridge 준비 미완료
- branch·environment protection 없음
- 운영 DB backup·restore rehearsal 없음
- aggregation·heartbeat·row delta·WebSocket·SHA health gate 없음

따라서 P0~P7과 ADR-0015 gate가 완료되기 전 `release` 승격과 운영 배포를 수행하지 않는다.

P0는 기존 자동 `release` push를 제거하고 승인 SHA 수동 실행, 외부 Action commit pinning, Administration read 자격 증명, P8 exact-SHA 잠금과 보호·CI 증명을 build 전 gate로 추가했다. profile script도 전체 40자리 SHA만 허용하며 migration 중 실행 취소를 금지했다. 현재 enable SHA가 없으므로 보호 설정이 우연히 추가돼도 배포는 열리지 않는다.

## 독립 리뷰 반영

- 주문 의도를 위험 검사 전에 영속해 `risk_rejected` 감사·멱등 기록을 보존했다.
- 내부 idempotency key와 56자 Upbit identifier의 제출 전 영속 매핑을 정의했다.
- Gateway 비재시도, 읽기 전용 수집기의 새 GET, 주문 `outcome_unknown` 대사를 분리했다.
- 호가 snapshot 내용 dedup과 모든 WebSocket receipt provenance 보존을 분리했다.
- 도메인 Markdown을 `docs/contracts/`에서 모듈 설계로 이동하고 실제 migration·OpenAPI·JSON Schema만 기계 계약으로 인정했다.
- 구 `GM-PROD-001~035`를 새 P0~P8에 전부 매핑하고 이전 refinement는 역사 상태로 표시했다.
- UI 정보 구조·화면 흐름의 단일 기준을 `docs/ui/`로 모으고 PRD는 제품 결과만 소유하게 했다.
- Linux 한글 wrap 회귀를 magic height가 아닌 공유 서브그리드와 681·680px/WCAG 회귀 테스트로 수정했다.
