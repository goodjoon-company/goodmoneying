## 변경 요약

- `displayFormat.ts`에 KST 날짜·시간, 숫자, 화폐와 자산 수량의 공통 표시 정책을 추가했다.
- 운영 상태, 시장 목록, 코인 분석, 상세, Backfill, Quotation, Exchange와 WebSocket 구조화 화면을 같은 정책으로 통일했다.
- 세 차트의 시간 축·십자선과 가격 축을 KST 24시간·후행 화폐 단위로 표시한다.
- Quotation 공통 조회 기준을 마켓 선택 → 기준 자산 → 읽기 전용 거래쌍 순서로 바꿨다.
- 전체 카탈로그 파라미터를 `한글 이름(parameter_name)`으로 표시하고 불리언 입력을 텍스트 옆 작은 체크박스로 배치했다.
- 조회용 `count`는 기능별 최대값으로 시작하고 공식 API 기본값은 도움말에 별도로 표시한다.
- 데스크톱에서 `to`와 `count`를 같은 행의 내용 맞춤 폭으로 배치하고 390px에서는 한 열로 쌓는다.

## 표시 정책

- 날짜·시간: `YYYY.MM.DD HH:mm:ss KST`, 24시간제
- 날짜: `YYYY.MM.DD`
- KRW: 소수점 없이 `1,234 ￦`
- USDT·BTC: 최대 소수점 8자리로 `1,234.56 $`, `0.12345678 ₿`
- 기타 자산: 최대 소수점 8자리와 후행 코드로 `1,234.5678 ETH`
- 통화 문맥이 없으면 단위를 추정하지 않고 구분자만 적용

## 계약과 안전 경계

- 원본 API 추적 JSON, 요청값, 응답 원문과 DB·HTTP·WebSocket 계약은 변경하지 않았다.
- 실제 주문·취소·입출금 등 상태 변경 API의 비파괴 차단 정책은 유지했다.
- 조회용 화면 초기값과 업비트 공식 API 기본값을 분리했으며 카탈로그 자체의 기본값은 바꾸지 않았다.
- 새 ADR(Architecture Decision Record)은 필요하지 않다. 이번 변경은 프런트엔드 표시·입력 배치에 한정되고 외부 호환성이나 모듈 배포 경계를 바꾸지 않는다.

## 검증과 후속 참고

- Vitest 135개, Pytest 360개, Playwright E2E 15개와 TypeScript·Vite 빌드·Ruff·MyPy를 통과했다.
- 업비트 게이트웨이는 현재 실행 중이며 PyJWT 2.13.0 import도 확인했다.
- Vite 대형 청크와 Starlette 지원 중단 예정 경고는 이번 변경 범위 밖의 기존 후속 과제다.
