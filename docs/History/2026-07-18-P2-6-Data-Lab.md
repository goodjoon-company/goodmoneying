# P2-6 Data Lab 변경 기록

날짜: 2026-07-18

## 사용자 결과

연구자는 Data Lab에서 KRW 시장과 KST 범위를 입력해 dataset build를 만들고, 기존 dataset version은 편집하지 않고 새 build로 복제할 수 있다. 게시된 version은 coverage heatmap, exact member 차트·표, A/B 비교로 확인한다.

## 구현 경계

- Command Center에 Data Lab 전용 메뉴와 runtime `REST polling` 표시를 추가했다.
- Data Lab 전용 React feature를 추가하고 build composer, build lifecycle, immutable version 목록, coverage, series exact member, A/B 비교를 분리했다.
- 브라우저 API client는 `/api/v1/dataset-builds`, `/api/v1/dataset-versions`, coverage, series, data foundation을 호출하고 운영 토큰을 직접 넣지 않는다.
- `/v1/data-foundation` 시장 응답에 `instrumentId`를 추가해 Data Lab이 시장 선택을 dataset build series로 연결할 수 있게 했다.
- E2E seeded API에 dataset version repository fixture를 추가해 build 생성과 version 탐색을 브라우저에서 검증했다.
- build, version, exact member는 `nextCursor`가 있으면 더 보기로 누적 탐색한다.
- 다중 series version은 Series Exact Members 패널에서 series를 선택해 조회한다.
- coverage heatmap은 긴 범위에서도 viewport 가로 overflow를 만들지 않도록 wrap한다.

## 의미 결정

- P2-6은 내부 WebSocket을 구현하지 않는다. build 상태 재발견은 REST polling으로 제한한다.
- exact member의 `sourceAsOf`는 현재 API에 없으므로 UI가 임의로 만들지 않는다. 표시는 `occurredAt`, `knowledgeAt`, 품질, 값, content hash에 근거한다.
- Data Lab 화면은 기존 실시간 코인 분석 화면을 재사용하지 않고 연구 데이터 전용 feature로 둔다.
- P2-6은 full-history 무한 자동 fetch 대신 운영자가 page를 확장하는 방식으로 대용량 version/member 조회 비용을 명시화한다.
