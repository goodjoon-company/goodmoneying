# P2-7 내부 WebSocket 스트림 구현

## 변경 요약

- `/v1/realtime/analysis/stream`이 기존 분석 payload를 P2 내부 실시간 envelope로 감싸도록 변경하고, `/v1/realtime/analysis`는 기존 테스트·운영 호환 alias로 유지했다.
- `internal-realtime-stream.schema.json`을 추가해 `schema_version`, `topic`, `scope`, `event_id`, `sequence`, `cursor`, `message_type`, `payload`를 기계 검증한다.
- `goodmoneying_shared.realtime_stream`에 HMAC-SHA256 기반 cursor encode/decode와 envelope builder를 추가했다.
- Web 클라이언트에 `realtimeStream.ts` tracker를 추가해 중복·역순 event를 버리고 sequence gap 이후 event 적용을 멈춘다.
- 기존 분석 reducer와 테스트 호환을 위해 top-level legacy `version/type/sentAt`은 전환기 필드로 유지한다.
- 리뷰에서 발견한 heartbeat gap 은폐, 비객체 JSON handler 예외, 운영 secret 기본값, subscribe topic/scope 미검증을 수정했다.

## 후속 완료

- P2-8에서 REST snapshot 재동기화 endpoint, cursor 만료 뒤 snapshot+cursor 자동 복구, slow consumer backpressure 신호를 완료했다.
