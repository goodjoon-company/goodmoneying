# P2-8 snapshot 복구

Date: 2026-07-18
Related Task: [P2](../Task/P2.md), GitHub Issue #29
Related PR: TBD

## 변경 요약

- `/v1/realtime/analysis/snapshot` REST endpoint를 추가해 분석 화면 상태 snapshot과 같은 topic·scope의 서명 cursor를 반환한다.
- WebSocket stream subscribe가 유효한 REST snapshot cursor를 받으면 전체 초기 snapshot을 다시 보내지 않고 `subscribed`와 heartbeat 경계부터 이어간다.
- cursor snapshot version이 현재 snapshot version과 다르면 resume을 거부한다.
- 브라우저는 `snapshot_required`, sequence gap, heartbeat-ahead gap, `slow_consumer` 뒤 REST snapshot을 자동으로 읽고 상태를 교체한 뒤 `resumeCursor`로 재구독한다.
- 서버 WebSocket 전송에 `GOODMONEYING_STREAM_SEND_TIMEOUT_SECONDS` 기반 timeout guard를 추가하고, 지연 소비자는 `stream.slow_consumer` 신호와 code `1013` 종료로 복구 경로에 태운다.
- 코드 리뷰에서 지적된 REST cursor의 snapshot 결속 문제를 보정해 `snapshotVersion=analysis-snapshot-v1:{snapshot-content-sha256}`를 cursor에 서명하고, 재구독 시 현재 snapshot hash와 다르면 최신 전체 snapshot을 다시 보내도록 했다.

## 리스크

- 현재 slow consumer 검증은 네트워크 지연 실측이 아니라 timeout guard와 envelope 계약 고정이다. 장시간 부하·지연 테스트는 P7 운영 품질 검증에서 별도 수행한다.
- 비차단 보강 권고: REST cursor 발급 뒤 snapshot 내용이 실제 변경되는 통합 테스트, 실제 지연 transport에서 1013 close를 확인하는 비동기 테스트.
