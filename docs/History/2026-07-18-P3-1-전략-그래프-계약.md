# P3-1 전략 그래프 계약

날짜: 2026-07-18
Related Task: [P3](../Task/P3.md), GitHub Issue #30

## 변경 요약

- 전략 그래프 서버 검증기를 추가해 순환, port 자료형, 시간 주기, 미래참조, parameter 범위, 결측 정책, warmup, 출력 누락을 안정 오류 코드로 차단한다.
- 그래프 의미가 node·edge 배열 순서에 흔들리지 않도록 canonical graph hash를 추가했다.
- 전략 정의 생성, 전략 버전 게시·목록·상세 조회 API와 OpenAPI 계약을 추가했다.
- PostgreSQL migration에 전략 정의·버전·그래프·parameter 테이블과 append-only trigger를 추가했다.
- 같은 전략의 동시 버전 게시를 전략 단위 advisory lock으로 직렬화하고, `strategy_graphs.graph_hash`와 `strategy_versions.graph_hash`를 복합 외래키(Foreign Key)로 묶어 그래프 해시 불일치를 DB에서 차단했다.
- 전략 명령 멱등 충돌과 cursor 문맥 충돌의 409 응답을 OpenAPI 계약에 명시했다.

## 설계 결정

- P3-1은 서버 계약과 불변 저장 경계를 먼저 고정하고, Strategy Studio UI는 후속 조각에서 이 계약을 소비한다.
- 전략 graph wire field는 도메인 설계와 맞춰 `schema_version`, `input_ports`, `output_ports`, `from_node` 형태를 사용한다.
- 게시 요청은 모든 변경 명령 공통 규칙에 맞춰 `requestId`, `idempotencyKey`, `actorId`, `requestedAt`, `reason`을 요구한다.
