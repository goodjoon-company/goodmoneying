# 2026-07-18 P4-5 백테스트 Worker

검증: [P4-5 백테스트 Worker 검증](../Test/2026-07-18-P4-5-백테스트-worker-검증.md)

## 변경 요약

- `backtest_runs`에 worker 임대(lease), generation fencing, retry wait, dead-letter 컬럼과 claim 인덱스를 추가했다.
- `PostgresBacktestStore`에 `claim_next_run`, `complete_claimed_run`, `fail_claimed_run`을 추가했다.
- 완료와 실패 쓰기는 `id`, `lease_owner`, `lease_generation`, 미만료 lease를 모두 검증한다.
- `BacktestWorker` core를 추가해 claim → executor → complete, executor 오류 → fail, lease lost → 늦은 쓰기 없음 경로를 구현했다.
- live PostgreSQL E2E에서 stale worker 완료 거부, artifact 저장, `retry_wait`, `dead_letter` 전이를 확인했다.

## 의도적으로 제외한 범위

- 백테스트 실행 생성 API와 UI
- published strategy와 sealed dataset에서 replay input을 재구성하는 materializer
- 외부 object storage artifact 저장소
- 대용량 결과 pagination과 WebSocket 진행 이벤트

## 후속 작업

- 백테스트 실행 요청 API를 추가할 때 idempotency payload와 권한·감사 정책을 별도 계약으로 고정한다.
- 외부 artifact store를 도입할 때 DB transaction과 object storage 사이의 orphan cleanup 정책을 ADR로 기록한다.
