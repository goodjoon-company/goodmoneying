# 2026-07-18 P4-7 백테스트 run 생성

## 변경 요약

- `POST /v1/backtest-runs` API를 추가했다.
- 요청은 운영 토큰(`X-Operator-Token`)을 요구하며 `202 Accepted`와 `BacktestRunSummary`를 반환한다.
- `PostgresBacktestStore.create_run()`은 published 전략 version과 sealed 데이터셋 version을 같은 transaction에서 읽고, strategy graph hash, dataset content hash, dataset 범위·정책, engine version, canonical parameters hash, initial cash, execution model, seed, materialized candle events를 `backtest-run-input-v1` payload로 고정한다.
- `backtest_runs.input_payload` JSONB를 추가해 Worker claim이 생성 시점 engine input을 재구성할 수 있게 했다.
- 새 run은 내부 DB 상태 `queued`, API 상태 `pending`, `result_hash=NULL`로 저장되어 기존 Backtest Worker claim 대상이 된다.
- 같은 idempotency key와 같은 본문은 기존 run을 재생하고, 같은 key의 다른 본문은 `BACKTEST_IDEMPOTENCY_CONFLICT`로 거부한다.
- 같은 semantic input hash가 이미 있으면 다른 idempotency key의 중복 생성으로 보고 409로 거부한다.
- Web client에 `createBacktestRun`을 추가했다.
- 체결 모델 execution 입력에서 `feeRate`와 `slippageBps`는 0 이상, `maxParticipationRate`는 0 초과 1 이하만 허용하도록 API schema와 OpenAPI 계약을 맞췄다.
