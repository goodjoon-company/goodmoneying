# 2026-07-18 P5-1 포트폴리오·봇·위험 DB 계약

## 변경 요약

- P5 paper/shadow 실행을 위한 DB migration `20260718000500_p5_portfolio_bot_risk.sql`을 추가했다.
- `portfolios`, `portfolio_policies`, `capital_allocations`로 포트폴리오와 자금 배분 정책 경계를 고정했다.
- `bot_definitions`, `bot_instances`, `bot_state_transitions`로 paper/shadow 봇 상태와 감사 전이를 고정했다.
- `order_intents`, `exchange_orders`, `order_fills`, `position_projections`로 실제 주문 없는 simulated 주문·체결·포지션 projection 경계를 고정했다.
- `risk_limits`, `risk_events`, `kill_switches`로 위험 판정 증거와 긴급 정지 감사를 고정했다.
- `bot_state_transitions`, `order_fills`, `risk_events`, `kill_switches`는 append-only trigger로 수정·삭제를 거부한다.

## P6 차단 경계

- `execution_mode`는 `paper|shadow`만 허용한다.
- `bot_instances.stage`는 `draft|backtest|paper|shadow|paused|stopped|faulted`까지만 허용한다.
- 실제 Upbit 주문 제출, private WebSocket, 주문 테스트 API, `live-ready|live` 전이는 구현하지 않았다.
