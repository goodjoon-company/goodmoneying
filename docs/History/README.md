# History

이 디렉터리는 인계가 필요한 변경 기록을 둔다. Task나 Test에 남긴 세부 진행 로그를 반복하지 않고, 나중에 다시 읽어야 할 변경 요약과 리스크를 짧게 보존한다.

## 파일 규칙

- 파일명: `YYYY-MM-DD-한글-변경-요약.md`
- 새 History 문서는 `docs/History/템플릿.md`를 복사해 작성한다.

## 기록 대상

- 구조 변경
- 계약 변경
- 운영이나 배포에 영향이 있는 변경
- 다음 작업자가 알아야 할 리스크와 후속 작업

## 정리 규칙

- 오래 살아야 하지 않는 임시 문서는 삭제하거나 History에 짧게 흡수한다.
- Product/Architecture와 중복되는 설명은 링크로 대체한다.
