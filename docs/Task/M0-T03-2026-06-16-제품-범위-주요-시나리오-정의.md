## 범위

- 포함: 제품 전체 범위, 1차 MVP(Minimum Viable Product) 범위, 제외 범위, MVP 시나리오, 전체 제품 시나리오, 요구사항, 정책, 로드맵
- 포함: LLM(Large Language Model) 분석 신호, 봇(Bot) 설계, 전략 파이프라인(Pipeline), 봇 동작 시뮬레이션(Simulation) 시나리오
- 제외: 시스템 아키텍처(Architecture) 상세, DB/API/message 계약(Contract), 구현 계획, 코드 작성

## 현재 맥락

`docs/01_Product.md`에는 제품 목적만 작성되어 있고 제품 범위, 시나리오, 요구사항, 정책, 로드맵은 비어 있었다. 사용자는 초기 MVP를 데이터 수집, 저장, 조회, 데이터 품질(Data Quality) 확인 중심으로 제한하고, 전체 제품에는 분석, 전략, 봇 관리, 시뮬레이션, 모의매매, 실거래까지 포함하기로 승인했다.

## 설계 메모

- 제품 문서는 MVP 범위와 전체 제품 시나리오를 분리해 혼동을 줄인다.
- 업비트(Upbit) 코인은 분석 실행 시점 기준 시가총액(Market Cap) 상위 50개로 제한한다.
- 호가(Orderbook)는 원천 스냅샷(Snapshot)과 요약 데이터를 병행 저장한다.
- LLM 신호는 카테고리(Category), 방향, 강도(Strength), 신뢰도(Confidence), 기준 시점, 관련 종목을 갖는 전략 입력으로 다룬다.
- 봇은 투자 가능 금액, 리스크 한도, 전략 가중치(Weight), 전략 파이프라인을 통해 최종 판단을 만든다.

## 계약 링크

- 현재 단계에서는 기계 검증 가능한 DB/API/message 계약 변경이 없다.
- 후속 데이터 수집 구현 전 `docs/contracts/`에 데이터 계약(Contract)을 추가해야 한다.
