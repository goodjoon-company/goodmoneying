# M0 요약: 제품/문서 기준 정리

## 핵심 결과

- `docs/01_Product.md`, `docs/02_Architecture.md`, `docs/contracts/`, `docs/ADR/`, `docs/Test/`, `docs/History/` 구조를 만들고 역할을 분리했다.
- 초기 산출물을 `docs/` 아래로 이동하고, 제품 범위와 주요 사용자 시나리오를 `docs/01_Product.md`에 통합했다.
- 중복 PRD(Product Requirements Document)와 병렬 설계 문서를 제거하고 Product 문서를 단일 기준(source of truth)으로 고정했다.
- `superpowers` 산출물은 repo-local 규칙에 맞춰 Product, Architecture, ADR, Test, History로 흡수하도록 정했다.

## 남은 원칙

- 새 실행 단위는 GitHub Issue로 관리한다.
- 제품 범위와 정책 변경은 `docs/01_Product.md`가 우선한다.
- 검증은 `docs/Test/`, 오래 유지되는 변경 요약은 `docs/History/`에 남긴다.
