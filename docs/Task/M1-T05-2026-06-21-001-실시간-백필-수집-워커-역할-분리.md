## 목표

실시간 수집 워커(Realtime Collection Worker)와 백필 수집 워커(Backfill Collection Worker)를 별도 실행 단위로 분리하고, CLI(Command Line Interface) 옵션으로 워커 모드를 바꾸는 구조를 제거한다.

## 요구사항 링크

## 범위

- `apps/worker/goodmoneying_worker/main.py` CLI 제거
- `realtime_collection_worker`, `backfill_collection_worker`, `runtime` 모듈 추가
- `dev.sh` app 단위를 `realtime-collection-worker`, `backfill-collection-worker`로 변경
- 로컬/운영 Compose와 prod-home healthcheck를 두 워커 기준으로 변경
- 자동화 테스트와 아키텍처 문서 갱신

## 현재 맥락

기존 `dev.sh app start worker`는 실시간 수집만 반복했다. 승인된 백필 작업이 DB에 `pending`으로 있어도 `--run-backfill-loop` 옵션을 별도로 실행하지 않으면 처리되지 않았다.

## 설계 메모

- 실시간 수집은 기존처럼 외부 실행 루프가 1분 주기로 `realtime_collection_worker`를 실행한다.
- 백필 수집은 `backfill_collection_worker`가 자체 루프를 가지고 기본 10초마다 DB의 백필 작업 상태를 확인한다.
- 백필 작업 상태의 단일 기준(Source Of Truth)은 기존 `backfill_jobs`, `backfill_job_targets` 계약을 유지한다.
- 메시지 큐(Message Queue)와 코인별 병렬 백필은 M3.5로 미룬다.
