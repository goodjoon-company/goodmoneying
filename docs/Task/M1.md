# M1 요약: 업비트 수집 운영 MVP

## 핵심 결과

- 업비트 KRW 후보 100개, 기본 활성 수집 대상 50개, 수집 계획(Collection Plan), 원천 캔들(Source Candle), 현재가 스냅샷(Ticker Snapshot), 호가 요약(Orderbook Summary) 저장 계약을 정의했다.
- SQLite/PostgreSQL 저장소, FastAPI 운영 API, fixture/live 업비트 클라이언트, worker, React 운영 화면, Playwright E2E(End-to-End) 기반을 구현했다.
- 실제 업비트 월간 백필과 다크 테마 운영 화면을 검증했다.
- prod-home CI/CD(Continuous Integration/Continuous Delivery) 배포 기반과 배포 프로필을 정리했다.
- 실시간 수집 워커와 백필 수집 워커의 책임을 분리했다.
