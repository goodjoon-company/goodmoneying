# M2 요약: 데이터 수집관리, Backfill 관리, 관심종목

## 핵심 결과

- 관심종목은 최상단 1레벨 메뉴로 두고, 데이터 수집관리 화면은 운영 상태, Backfill 관리, 코인 상세 레이어 중심으로 재구성했다.
- 수집 대상 최대 50개 조정, 백필 계획 생성, pending 작업 저장, 승인/재개/중지 흐름을 Backfill 관리 화면으로 모았다.
- 개발 환경 제어 스크립트(`dev.sh`)와 `.env` 로딩, API/web/worker 단위 실행 흐름을 정리했다.
- PostgreSQL 스키마 불일치로 인한 API 오류를 보정하고 schema 적용 경로를 안정화했다.
- `관심종목` 화면은 후보 100개 기준으로 관심 여부, 현재가, 24시간 거래대금, 전일 종가 대비 등락률, 기준일시, 캔들 커버리지 기준 시작일과 현재일, 1분 캔들 수를 표시한다.
- 운영 화면 공통 상단은 관심 코인 개수와 관심 코인 목록 레이어 팝업(layer popup)을 제공한다.
- 관심목록 순서는 사용자가 조정할 수 있고, 운영 상태, Backfill 관리, 공통 관심 코인 목록에서 같은 순서로 적용한다.
- 관심종목 가격 정보는 `/v1/market-list/stream` SSE(Server-Sent Events) 이벤트로 push 갱신하고, HTTP 조회를 보조 경로로 둔다.

## 현재 기준

- 관심목록은 기존 활성 수집 대상(Active Collection Target)의 사용자 표현이다.
- 관심 별 토글과 순서 변경은 `PUT /v1/collection-targets`를 사용하며 운영 상태와 Backfill 관리의 대상 종목 기준을 함께 바꾼다.
- 주식 탭은 아직 수집 계약과 데이터가 없으므로 빈 상태만 제공한다.

## 제품 방향상 위치

- 관심종목은 사용자가 투자 후보를 비교하고 좁히는 다음 제품 단계의 중심 화면으로 계속 가져간다.
- Backfill 관리는 과거 데이터 기반과 결측 복구를 위한 필수 데이터 준비 도구로 유지한다.
- 데이터 수집관리의 운영 기능은 관심종목보다 앞선 제품 중심 화면으로 확장하지 않는다.
