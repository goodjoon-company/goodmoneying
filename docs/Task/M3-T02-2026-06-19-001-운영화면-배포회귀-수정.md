## 범위

- Web Nginx 프록시(proxy)로 `/api` 요청에 운영 토큰을 서버 측에서 붙인다.
- 첫 화면 조회를 대시보드와 백필 작업으로 줄이고 후보/시장/상세 데이터는 지연 로딩한다.
- 거래대금 표시, 수집 범위 `~ NOW`, 품질 상세 tooltip, 운영 상태 테이블을 개선한다.
- API 계약에 `qualityDetail`, `storedRowCount`를 추가한다.

## 비범위

- 장기 최적화용 PostgreSQL materialized view 또는 별도 통계 테이블 도입
- 운영 서버 직접 재배포
- destructive rebuild 백필

## 현재 맥락

운영 web은 `http://100.68.208.102:8080/`에서 제공되고, 기존 번들은 `http://app-server01:8000`과 `local-dev-token`을 브라우저에 굽고 있었다. 운영 API의 `GOODMONEYING_OPERATOR_TOKEN`과 달라 수집 대상 저장과 백필 계획 생성이 401을 반환했다.

## 설계 메모

- 브라우저 번들에는 운영 토큰을 넣지 않는다.
- web 컨테이너의 Nginx가 `/api`를 `GOODMONEYING_API_INTERNAL_URL`로 프록시하고 `GOODMONEYING_OPERATOR_TOKEN`을 `X-Operator-Token`으로 붙인다.
- 로컬 Playwright 개발 서버만 `VITE_OPERATOR_TOKEN=local-dev-token`을 사용한다.
- 첫 화면은 `dashboard/summary`와 `backfill/jobs`만 호출한다.

## 계약 변경

- `CandidateUniverseEntry.qualityDetail` 추가
- `CollectionDataStatus.storedRowCount` 추가
- 거래대금 표시값은 통화기호와 천 단위 구분자를 포함한다.

## 실패 케이스

- 운영 API 토큰과 브라우저 번들 토큰이 다르면 쓰기 요청이 401을 반환한다.
- 첫 화면이 후보/시장/상세/캔들을 모두 조회하면 대시보드 표시가 느려진다.
- 품질 배지가 의미 설명 없이 `정상/주의`만 보여 사용자가 원인을 알 수 없다.
