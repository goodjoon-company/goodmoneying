- 운영 배포와 데이터 migration

## 현재 맥락

아키텍처 문서는 기존 활성 수집 대상이 최신 후보 유니버스 상위 100 밖으로 이탈해도 자동 제거하지 않고 후보 유니버스 이탈 상태로 표시한다고 정의한다. 그러나 저장 API는 요청에 포함된 모든 instrument id가 최신 후보 유니버스 안에 있어야 한다고 검증해, 이미 활성 상태였던 후보 밖 대상까지 `INVALID_COLLECTION_TARGETS`로 막았다.

## 설계 메모

- 신규로 선택되는 활성 수집 대상은 계속 최신 후보 유니버스 안에 있어야 한다.
- 이미 활성 상태인 대상은 후보 밖으로 이탈했더라도 사용자가 유지할 수 있다.
- 기존 활성 대상을 유지하는 저장 요청은 재활성화하지 않는다. 그래야 `candidate_status = out_of_universe` 상태를 잘못 `in_universe`로 덮지 않는다.

- 기존 활성 대상 A가 최신 후보 유니버스 밖으로 이탈한다.
