## 비범위

- 별도 관심종목(Watchlist) DB/API 도메인 추가
- 후보 유니버스 밖 신규 종목 활성화 허용
- 캔들 커버리지 lazy endpoint 분리 재설계

## 현재 맥락

기존 화면은 관심추가된 항목과 후보 항목을 한 표에 섞어 보여줬고, 검색 입력은 UI만 있었으며 정렬은 select 형태로만 표시됐다. 캔들 커버리지는 실제 저장된 캔들 구간을 읽지 않고 고정 퍼센트 세그먼트를 반환해, 백필 구간과 실시간 수집 구간 사이의 공백을 정확히 표현하지 못했다.

## 설계 메모

- 현 MVP에서는 관심추가 상태를 별도 도메인이 아니라 활성 수집 대상(Active Collection Target) 선택 상태로 표현한다.
- 검색과 정렬은 같은 후보 유니버스 응답에서 파생해 상단 관심추가 항목과 하단 후보 목록에 동일하게 적용한다.
- 캔들 커버리지는 1분봉 `source_candles`의 연속된 `candle_start_at`을 collected 구간으로 묶고, 구간 사이 공백을 missing 구간으로 표시한다.
- `offsetPercent`와 `widthPercent`는 실제 저장 구간의 첫 시각부터 마지막 캔들 종료 시각까지의 전체 기간을 기준으로 계산한다.
