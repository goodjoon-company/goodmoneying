## 범위

- Rate limit 여유 패널을 최근 1분 수집 행 수 패널로 교체한다.
- 최근 1분 수집 행 수는 실시간 수집과 백필 수집을 분리해 표시한다.
- 코인 상세의 중복 행 표시는 제거하고 24시간 변동 금액, 등락율, 거래량, 거래량 등락율을 표시한다.
- 헤더의 CSV 내보내기와 운영 변경 저장 버튼을 제거한다.
- 사이드바의 코인 상세 메뉴와 독립 화면을 제거하고, 시장 리스트 행 클릭 시 모달로만 코인 상세를 표시한다.
- 화면 숫자는 하드코딩하지 않고 API View Model에서 받은 값만 표시한다.
- 저장량 중심 문구는 Row Count 기준의 저장 행 단위로 조정한다.

## 현재 맥락

선행 M3-T04 작업에서 새 운영 콘솔 디자인 요소가 반영되었지만, 디자인 파일에는 현재 저장소가 실제로 관측하지 않는 `Rate limit 여유`와 `중복 행` 표시가 남아 있었다. 운영 UI는 합성 지표 없이 영속화(persisted) 또는 계산(computed) 가능한 값만 표시해야 한다.

## 설계 메모

- `Rate limit 여유`는 워커(Worker)가 업비트(Upbit) rate limit 헤더를 저장하지 않으므로 표시하지 않는다.
- 대체 지표로 `collection_runs`와 `target_collection_results` 기반 최근 1분 수집 행 수를 제공한다.
- 실시간 수집은 `collection_runs.run_type = incremental`, 백필 수집은 `collection_runs.run_type = backfill` 기준으로 분리한다.
- 기본 `npm run dev:web` 실행은 `VITE_API_BASE_URL` 없이 `/api`를 호출하므로, Vite 개발 서버가 `/api`를 로컬 API 서버로 프록시(proxy)해야 한다.
- 실행 중인 API 서버가 새 필드 배포 전 응답을 유지할 수 있으므로, 프론트엔드 API 클라이언트는 누락된 새 대시보드 필드를 정규화(normalize)해 첫 화면 렌더 실패를 막는다.
- 백필 수집도 SQLite 저장소(Repository)에서 `record_backfill_candles()` 시 수집 실행과 대상별 결과를 기록해 PostgreSQL 저장소와 같은 계약을 맞춘다.
- 코인 상세의 24시간 변동 금액은 현재가와 24시간 등락율로 계산한다.
- 코인 상세의 24시간 거래량과 거래량 등락율은 최근 24시간 1분봉 거래량과 직전 24시간 1분봉 거래량을 비교해 계산한다.
- 저장 단위는 용량 중심 `storageBytes`를 보조 정보로 유지하되, UI의 주요 비교값은 `storageRowCount`와 `storageRowsToday`를 사용한다.
