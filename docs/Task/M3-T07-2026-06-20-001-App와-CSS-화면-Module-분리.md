## 범위

- `App.tsx`를 Query Provider와 운영 콘솔 조립 shell로 축소한다.
- 운영 콘솔 shell, 대시보드, 수집 대상/백필, 시장 리스트, 상세 레이어, 확장성 점검, 공통 표시 UI를 화면 단위 Module로 분리한다.
- `styles.css`를 entrypoint로 축소하고 base/shell/common/data-tables/modals/dashboard/responsive 등 역할별 stylesheet로 나눈다.
- 기존 API 계약과 화면 동작은 바꾸지 않는다.

## 비범위

- REST 계약, DB 계약, 백엔드(Backend) 동작 변경
- 시각 디자인 변경
- React Query 정책과 폴링(Polling) 주기 변경
- CSS Module 라이브러리 또는 CSS-in-JS 도입

## 현재 맥락

`M3-T06`에서 데이터 접근 Adapter, fixture Adapter, workflow, 표시 helper, 운영 콘솔 상태 hook은 분리되었지만 `App.tsx`에는 화면별 JSX와 차트 Implementation이 여전히 남아 있다. `styles.css`도 여러 역할의 스타일이 한 entrypoint에 누적되어 있다.

## 설계 메모

- `App.tsx`의 Interface는 애플리케이션 Provider 조립으로 제한한다.
- 화면별 Module은 각 화면이 필요한 데이터와 이벤트만 props로 받는다.
- CSS는 런타임 import 순서를 보존하는 entrypoint와 역할별 stylesheet로 분리한다.
- 이번 분리는 shallow pass-through Module을 만들지 않고, 유지보수자가 찾는 화면 단위 Locality를 높이는 데 집중한다.
