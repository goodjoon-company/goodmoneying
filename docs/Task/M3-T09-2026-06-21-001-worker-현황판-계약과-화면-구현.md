## 현재 맥락

기존 운영 상태 첫 카드는 활성 대상, 주의/장애, 전체 실패율만 표시했다. worker 분리 이후에는 실시간 수집 워커와 백필 수집 워커의 상태를 직접 확인할 수 있는 카드가 필요하다.

## 설계 메모

- worker 동작 상태는 `collection_worker_heartbeats`의 최신 heartbeat를 기준으로 `running`, `stale`, `failed`로 계산한다.
- 실시간 수집 오류는 최근 24시간 `collection_runs` 중 `run_type='incremental'` 실패 실행을 기준으로 계산한다.
- 백필 오류는 전체 `backfill_job_targets.status='failed'`를 기준으로 계산한다.
- 현재 백필 동작 코인 수는 pending/running 백필 작업의 target 중 `running` 수와 전체 target 수를 사용한다.
- 프론트엔드는 오류 상세를 별도 호출 없이 `DashboardSummary.workerStatus`의 `recentErrors`로 표시한다.

## 계약 링크

## 계약 변경

- DB에 `collection_worker_heartbeats` 추가
- API `DashboardSummary`에 `workerStatus` 추가
- API에 `CollectionWorkerStatus`, `RealtimeWorkerStatus`, `BackfillWorkerStatus`, `CollectionWorkerError` 스키마 추가

## 실패 케이스

- heartbeat가 없으면 worker 상태를 `stale`로 표시한다.
- heartbeat가 허용 지연 시간을 넘으면 `stale`로 표시한다.
- 마지막 heartbeat가 실패이면 `failed`로 표시한다.
- 오류 상세가 없으면 레이어 팝업에 빈 상태 문구를 표시한다.
