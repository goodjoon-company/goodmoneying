# M3-T09 실시간 수집 웹소켓(WebSocket) + SSE(Server-Sent Events) 전환

## 목표

현재 REST API 폴링(Polling) 기반 실시간 수집 워커를 업비트 웹소켓(WebSocket) 스트림 수신 기반으로 전환하고, 운영 화면은 운영 서버의 SSE(Server-Sent Events) 스트림으로 대시보드 상태를 갱신한다.

## 범위

- 업비트 웹소켓(WebSocket) 구독 요청 생성
- 현재가(Ticker Snapshot), 호가 요약(Orderbook Summary), 1분 캔들(Source Candle) 웹소켓 메시지 변환
- 변환된 스트림 데이터를 기존 저장 계약에 맞춰 저장하는 버퍼(buffer) 도입
- `/v1/dashboard/summary/stream` SSE(Server-Sent Events) API 계약 추가
- 운영 화면에서 EventSource 기반 대시보드 갱신 구독

## 비범위

- 브라우저가 업비트 웹소켓에 직접 연결하는 방식
- 메시지 큐(Message Queue), 다중 워커 분산, dead-letter 처리
- 실거래 주문/자산 웹소켓 구독
- 운영 배포 환경 프로세스 매니저 재구성

## 현재 맥락

현재 실시간 수집은 `apps/worker/Dockerfile`에서 60초 sleep 루프로 REST API를 반복 호출한다. 운영 화면은 React Query로 `/v1/dashboard/summary`를 15초 주기로 HTTP 폴링한다. 사용자는 이 구조를 서버 측 업비트 웹소켓 수집과 운영 서버 SSE 화면 갱신 방식으로 변경하기로 결정했다.

## 설계 메모

- 업비트 웹소켓은 서버 측 워커가 연결한다.
- 웹소켓 수집기는 `ticker`, `orderbook`, `candle.1m`을 활성 수집 대상 market code로 한 번에 구독한다.
- 수신 메시지는 기존 `TickerSnapshot`, `OrderbookSummary`, `SourceCandle` 모델로 변환한다.
- 저장은 기존 `record_incremental_collection()`을 우선 재사용해 DB 계약 변경을 최소화한다.
- 운영 화면은 직접 업비트에 연결하지 않고 운영 API의 SSE 스트림을 구독한다.
- SSE 이벤트는 `event: dashboard`, `data: DashboardSummary` 형식을 사용한다.
