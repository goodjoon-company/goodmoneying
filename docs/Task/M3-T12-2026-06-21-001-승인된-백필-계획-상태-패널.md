## 현재 맥락

기존 화면은 승인 전 draft 계획과 승인 완료 toast성 메시지만 보여줬다. 이미 API는 `/v1/backfill/jobs`로 저장소의 백필 job 목록과 진행률을 제공하고 있었으므로, 새 계약 없이 화면 패널에서 해당 데이터를 표시한다.

## 설계 메모

- `status !== 'planned'`인 job을 승인된 백필 계획으로 본다.
- 상태 라벨은 `pending=대기 중`, `running=실행 중`, `paused=일시정지`, `stopped=중지`, `succeeded=완료`, `failed=실패`로 표시한다.
- 상태는 아이콘(icon)과 텍스트를 함께 사용해 스캔성을 높인다.
- 진행률은 0~100 범위로 clamp해 잘못된 API 값이 레이아웃을 깨지 않게 한다.

## 실패 케이스

- 사용자가 백필 계획을 승인해도 수집 대상/설정 화면에서 실행 상태를 확인할 수 없다.
- 여러 승인 job이 있을 때 pending/running/succeeded 상태를 구분할 수 없다.
- 진행률 값이 문자열로 전달되어 UI 레이아웃이 깨질 수 있다.
