# M3-T13 백필 계획 제어와 worker 지표 정정

## 목표

운영 상태 화면과 수집 대상/설정 화면에서 백필 작업(Backfill Job)의 실제 실행 단위와 상태를 혼동 없이 확인하고 제어할 수 있게 한다.

## 요구사항 링크

- `docs/contracts/api/openapi.yaml`

## 범위

- 운영 상태 worker 현황판의 `마지막 수집` 문구를 `마지막 저장 성공`으로 변경
- 백필 worker의 `동작중 코인`을 현재 실행 중인 단일 백필 계획 기준으로 계산
- 대기 백필 job 수와 대기 target 수를 보조지표와 진단 상세에 추가
- 승인된 백필 계획 목록에 대상 코인과 백필 시작/종료 일시 표시
- 승인된 백필 작업 중지와 삭제 API 및 화면 버튼 추가
- API 계약(OpenAPI)에 백필 job 대상/기간 필드와 삭제 endpoint 반영

## 비범위

- 백필 동시성 확대
- 여러 코인을 한 번에 가져오는 업비트 API 호출 방식 변경
- 삭제 후 재수집(Destructive Rebuild)

## 설계 메모

- 업비트 1분봉 백필은 코인별 요청으로 실행되며, worker는 하나의 백필 job 안에서 대상 코인을 순차 처리한다.
- `runningTargetCount/totalTargetCount`는 전체 pending/running job 합계가 아니라 현재 worker가 처리 중이거나 다음으로 처리할 단일 job 기준이다.
- 대기 중인 나머지 pending job/target은 `queuedJobCount/queuedTargetCount`로 분리한다.
- 실행 중인 job은 바로 삭제하지 않고 `stop` 후 삭제한다.
