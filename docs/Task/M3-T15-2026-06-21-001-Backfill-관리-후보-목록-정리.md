# M3-T15 Backfill 관리 후보 목록 정리

Status: Done
Owner: Codex
Created: 2026-06-21
Updated: 2026-06-21

## 목적

수집 대상/설정 메뉴를 Backfill 관리 화면으로 재정의하고, 후보 유니버스 목록의 컬럼을 실제 운영 판단에 필요한 정보 중심으로 정리한다.

## 요구사항

- 좌측 메뉴의 `수집 대상/설정` 이름을 `Backfill 관리`로 변경한다.
- 활성 컬럼의 체크박스 옆 `수집` 텍스트를 제거하되 접근성 라벨은 유지한다.
- 거래대금 컬럼은 `24시간 거래대금`으로 표시한다.
- 후보 목록의 품질 컬럼과 품질순 정렬은 제거한다.
- 수집 범위는 수집된 데이터 기준의 `수집 시작일`, `수집 최종일` 컬럼으로 분리한다.
- 실시간 수집 대상은 `실시간` 라벨로 표시한다.

## 계약 변경

- `CandidateUniverseEntry` API 응답에 `collectedStartAt`, `collectedEndAt`, `isRealtimeTarget` 필드를 추가한다.
- 기존 `collectionRangeDisplay`는 하위 호환을 위해 유지하지만 Backfill 관리 화면에서는 새 분리 필드를 렌더링한다.

## 실행 계획

- [x] RED: App, E2E, API, OpenAPI 계약 테스트에 새 메뉴명/컬럼/필드 기대값 추가
- [x] GREEN: API schema와 서비스 응답에 수집 시작/최종일 및 실시간 여부 추가
- [x] GREEN: Backfill 관리 화면 테이블 컬럼과 정렬 옵션 정리
- [x] 제품(Product), 아키텍처(Architecture), API 계약(Contract) 문서 갱신
- [x] 자동화 검증 실행

## 완료 기준

- Backfill 관리 메뉴에서 후보 목록이 활성, 후보, 24시간 거래대금, 수집 시작일, 수집 최종일만 표시한다.
- 품질 컬럼과 품질순 정렬이 보이지 않는다.
- 실시간 수집 대상은 실시간 라벨을 표시한다.
- API 계약과 구현이 동일한 필드를 제공한다.
