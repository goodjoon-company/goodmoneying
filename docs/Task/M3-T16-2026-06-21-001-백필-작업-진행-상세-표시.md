# M3-T16 백필 작업 진행 상세 표시

Status: Done
Created: 2026-06-21
Updated: 2026-06-21

## 목표

Backfill 관리 화면의 백필 작업 목록에서 실행 중인 작업의 동적인 진행 수치를 10초 주기 갱신으로 확인할 수 있게 한다.

## 요구사항 링크

- 사용자 요청: 백필 계획 목록에 총 대상 코인 수, 현재 실행 대상 순번, 현재 대상의 백필 row 수, 결측 구간 처리 수, 예상 요청 수를 표시한다.
- 사용자 요청: Backfill 관리의 작업 목록은 10초마다 갱신한다.
- 사용자 요청: 중지된 백필 작업은 작성일이 최근 1개월 이내인 것만 표시한다.
- 사용자 요청: 백필 작업 카드의 `결측 구간` 표기는 `결측 구간 처리`로 의미를 명확히 한다.
- 사용자 요청: 전체 진행률은 완료 대상뿐 아니라 현재 대상의 예상 결측 구간 처리 비율도 반영한다.
- 사용자 요청: hover 부가정보가 있는 항목에는 정보 아이콘을 표시한다.
- 사용자 요청: 운영 상태의 코인별 수집 상태는 전체 수집 코인을 표시하고 24H 거래대금 기본 정렬과 헤더 클릭 정렬을 제공한다.
- API 계약(Contract): `docs/contracts/api/openapi.yaml`
- DB 계약(Contract): `docs/contracts/db/schema.sql`

## 범위

- `/v1/backfill/jobs` 응답에 작업별 진행 상세 필드 추가
- `backfill_job_targets`에 결측 구간 처리 수, 예상 결측 구간 수, 현재 대상 row 수 기록
- 백필 워커(Backfill Worker)가 결측 구간 처리마다 진행 카운터를 기록
- Backfill 관리 화면 카드에 진행 상세 수치 표시
- Backfill 관리 화면 카드의 진행률에 현재 대상의 결측 구간 처리 비율 반영
- 운영 상태의 코인별 수집 상태 전체 목록 표시 및 헤더 정렬
- Backfill 관리 화면의 10초 갱신 유지 검증
- `stopped` 작업 목록은 작성일 기준 최근 30일 이내만 노출

## 비범위

- 백필 병렬 처리
- 결측 구간 병합 최적화
- 업비트(Upbit) rate limit 헤더 영속화

## 현재 맥락

기존 진행률은 완료 대상 수 기준이라 50개 중 5개 완료 상태에서는 한 대상 내부에서 수천 개 결측 구간을 처리해도 10%로 보였다. 작업 12의 `KRW-WLD`처럼 결측 구간이 잘게 흩어진 경우 UI 변화가 거의 없어 워커가 멈춘 것처럼 보였다.

## 설계 메모

- 작업 진행률 `progressPercent`는 완료 대상 수와 현재 대상의 `processed_missing_range_count / estimated_missing_range_count`를 합산해 계산한다.
- 현재 실행 대상의 세부 진행은 별도 필드로도 추가해 카드에서 숫자로 노출한다.
- 결측 구간 처리 수는 워커가 `fetch_minute_candles()` 한 구간을 마칠 때마다 증가한다.
- 현재 대상 백필 row 수는 이번 대상 처리 중 `record_backfill_candles()`가 기록한 누적 행 수로 표시한다.
- 중지 작업의 최근 1개월은 운영 단순성을 위해 30일로 계산한다.
- 코인별 수집 상태는 API가 준 전체 수집 대상 배열을 화면에서 정렬하며, 기본 정렬은 24H 거래대금 내림차순이다.

## 계약 변경

- `BackfillJob` 응답에 `estimatedRequestCount`, `totalTargetCount`, `completedTargetCount`, `runningTargetIndex`, `currentTarget`, `currentTargetBackfillRowCount`, `processedMissingRangeCount`, `estimatedMissingRangeCount` 추가
- `backfill_job_targets`에 `processed_missing_range_count`, `estimated_missing_range_count`, `rows_written_count` 추가

## 실패 케이스

- 실행 중 대상 내부에서 진행 중인데 카드 수치가 변하지 않는다.
- 중지된 오래된 작업이 계속 작업 목록을 오염시킨다.
- 기존 백필 시작/중지/재개 응답이 새 필드를 누락한다.
