# M3 요약: 운영 콘솔 고도화와 realtime/backfill 운영

## 핵심 결과

- 운영 화면을 디자인 기준에 맞춰 대시보드, Backfill 관리, 관심종목, 코인 상세 레이어로 정리하고 화면 Module과 CSS를 분리했다.
- 합성 지표를 제거하고 실제 저장 데이터 기반 커버리지, 결측, 저장량, worker 상태, 감사 로그, 운영 추이를 표시하도록 바꿨다.
- worker 현황판, Realtime worker 24시간 수집 row, SSE(Server-Sent Events) 대시보드 갱신, 업비트 WebSocket 실시간 수집 경로를 구현했다.
- Backfill 작업 진행 상세, 결측 구간 기반 진행률, 실패 작업 재개, 중지/재개/삭제 제어, 배치 저장과 heartbeat 갱신을 정비했다.
- 실시간 체결 이벤트(Trade Event)를 저장하고 체결 빈도 히트맵으로 전환했다.
- `확장성 점검` 화면은 메뉴에서 제거하고, M4 확장성 검토는 문서/ADR 기준으로 남겼다.

## 현재 기준

- 저장량 집계는 원천 시장 데이터 3종(source_candles, ticker_snapshots, orderbook_summaries)을 기준으로 한다.
- `target_collection_results`는 저장량이 아니라 운영 결과 로그로 다룬다.
- 브라우저 갱신은 SSE, 서버 수집은 live 프로필에서 업비트 WebSocket을 사용한다.

## 제품 방향상 위치

- 코인 상세 레이어는 관심 대상 분석의 진입점으로 계속 가져가되 운영 품질 확인보다 투자 분석 맥락을 강화한다.
- 실시간 체결 히트맵은 투자 후보 판단 가치를 검증할 때까지 현재 구현만 보존하고 추가 확장을 보류한다.
- 호가 원천 스냅샷 저장은 시작하지 않고 호가 요약만 유지한다.
