# P1 요약: 시스템 트레이딩 데이터 기반

GitHub Issue: [#28](https://github.com/goodjoon-company/goodmoneying/issues/28)

## 핵심 결과

- 업비트 공식 시장 전체와 시점 상태 이력을 저장하고 모든 KRW 시장에 2024-01-01 UTC 기본 정책을 자동 적용한다.
- 시장별 시작 시각, 데이터 유형, 1분 기준 주기, 보존 기간, 우선순위, 연속 수집과 활성·일시정지·제외를 API와 Coverage & Quality 화면에서 관리한다.
- 캔들 백필은 PostgreSQL 멱등 작업·우선순위·임대·지수 백오프·재시도·데드레터로 조정하고, 오래된 작업자의 늦은 쓰기를 임대 펜싱(lease fencing)으로 거부한다.
- 체결·호가·티커는 DB의 희망 구독과 세대를 기준으로 WebSocket 연결을 회전하며 실제 연결 성공 뒤 읽었던 정확한 세대만 적용 완료로 기록한다.
- 웹소켓 프레임별 원본 수신 영수증과 동일 경제 상태의 호가 스냅숏을 분리하고, 전체 30레벨과 기존 10레벨 요약을 원자적으로 저장한다.
- 정책의 보존 기간은 원본 receipt·snapshot·level 정리 경로에 연결하고 파생 요약과 manifest는 보존한다.
- 원천 행은 사건·수신·저장·지식 시각과 체크섬 fetch manifest를 참조하고, coverage는 `available|no_trade|missing|unavailable|unverified`만 사용한다.
- 빈 응답으로 무거래를 합성하지 않으며 재시도 예산 소진은 미확인 coverage와 품질 이벤트까지 실패로 전이한다.

## 운영 기준

- DB 변경 단일 기준은 `docs/contracts/db/migrations/`이고 생성 스냅숏은 `docs/contracts/db/schema.sql`이다.
- 운영 내구성은 PostgreSQL만 권위가 있으며 SQLite는 과거 단위·API 테스트 호환 어댑터다.
- 내부 저장과 API wire는 UTC, UI 표시는 KST다.
- 기본 정책 생성은 곧바로 실제 3.5억 행 전체를 운영 DB에 강제 적재한다는 뜻이 아니다. 작업 우선순위·요청 수 제한·디스크·백업 게이트를 지키며 재개 가능한 큐로 처리한다.

## 검증과 후속

실제 명령·수량·브라우저 증적은 [`P1 데이터 기반 검증`](../Test/2026-07-17-시스템-트레이딩-플랫폼-P1-데이터-기반-검증.md)에 둔다. P2는 이 원천·품질 계약 위에서 집계·지표·데이터셋 버전·Data Lab·복구 가능한 내부 WebSocket을 구현한다.

P1의 네 구현 단위는 테스트 주도 개발(TDD)과 과업별 독립 리뷰·재리뷰를 거쳤고 최종 리뷰에서 Critical·Important·Minor 발견사항이 없다. 로컬 전체 회귀와 실제 PostgreSQL·Chromium E2E는 통과했으며, 원격 CI와 최종 SHA는 push 뒤 검증 문서와 Issue #28에 기록한다.
