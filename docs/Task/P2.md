# P2 요약: 연구 데이터 계층과 Data Lab

GitHub Issue: [#29](https://github.com/goodjoon-company/goodmoneying/issues/29)

## 현재 완료 결과

- P2-1은 캔들 개정 원장과 집계 계보를 추가해 원천 수정과 파생 결과의 입력 프런티어(frontier)를 추적한다.
- P2-2는 원천·품질 변경을 내구성 무효화 큐로 전파하고 불변 증분 집계를 같은 트랜잭션 경계에서 갱신한다.
- P2-3은 버전 지표와 시장 통계를 추가 전용 물질화(materialization)로 저장하고 안정 cursor REST 조회를 제공한다.
- P2-4는 1분 미시구조 통계를 체결·호가 receipt와 연결하고 원천 품질과 계산 상태를 분리한다.
- P2-5는 dataset build 수명주기와 게시 후 불변 dataset version, coverage, exact member REST 계약을 구현한다.
- P2-6은 Data Lab 화면에서 build 생성·복제, build 목록 REST polling, version 목록, coverage heatmap, series 선택, exact member 표·차트, A/B 비교를 제공한다.

## 현재 제외·대기

- P2-7 내부 WebSocket stream과 P2-8 snapshot·delta 재동기화는 아직 구현하지 않았다.
- 주식시장 기능은 현재 목표의 비범위다.
- Data Lab은 브라우저에 운영 토큰을 보관하지 않고 동일 출처 프록시가 주입하는 경계를 유지한다.
- build, version, exact member의 다음 page는 cursor 기반 더 보기로 명시 탐색한다.

## 검증과 후속

P2-6의 실제 명령과 브라우저 증적은 [`P2-6 Data Lab 검증`](../Test/2026-07-18-P2-6-Data-Lab-검증.md)에 둔다. P2 완료 선언은 P2-7·P2-8까지 구현하고 전체 회귀·E2E·문서 동기화가 끝난 뒤 별도로 수행한다.
