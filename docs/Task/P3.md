# P3 요약: 형식 안전 전략 DAG와 불변 전략 버전

GitHub Issue: [#30](https://github.com/goodjoon-company/goodmoneying/issues/30)

## 현재 진행 결과

- P3-1은 전략 그래프(Strategy Graph)의 서버 검증기와 canonical graph hash를 추가한다.
- P3-1은 `/v1/strategy-graphs/validate`, `/v1/strategies`, `/v1/strategies/{strategyId}/versions`, `/v1/strategy-versions/{strategyVersionId}` REST 계약을 추가한다.
- P3-1은 `strategy_definitions`, `strategy_versions`, `strategy_graphs`, `strategy_parameters` DB 계약과 append-only trigger로 불변 전략 버전 경계를 고정한다.
- P3-1은 같은 전략의 동시 버전 게시 순번을 직렬화하고, 전략 그래프 해시가 게시 버전 해시와 DB에서 불일치하지 않도록 복합 외래키(Foreign Key)를 추가했다.
- P3-1은 PostgreSQL migration E2E에서 `versions=15`, live PostgreSQL `121 passed`, schema snapshot 동일을 확인했다.
- P3-1은 전체 로컬 회귀에서 `mypy`, `ruff`, 전체 `pytest`, `npm test`, 웹 빌드, Playwright E2E 통과를 확인했다.
- P3-1은 원격 CI `29636915475`에서 commit `ec37af5fcf3fc05a080e7a459a88c97a430f5479` success를 확인했다.
- P3-1은 기존 SQLite 집계 heartbeat 교차 테스트의 CI 스케줄링 흔들림을 안정화한 뒤, 원격 CI `29637456180`에서 commit `e28c6ecbb868ce33bffe28ba1c29aba01144c1d9` success를 확인했다.
- P3-2는 Operations Console에 Strategy Studio 메뉴와 화면 진입점을 추가한다.
- P3-2는 전략 그래프(Strategy Graph)를 포인터 뷰와 텍스트 대안으로 함께 표시하고, 키보드 대체 편집기로 출력 신호와 edge 오류 예제를 조작하게 한다.
- P3-2는 `/v1/strategy-graphs/validate` 서버 검증 오류를 안정 코드·node·edge 위치 텍스트와 `role="alert"`로 표시한다.
- P3-2는 검증 통과 graph만 `/v1/strategies/{strategyId}/versions`로 불변 전략 version에 게시하고, 게시 결과를 읽기 전용으로 표시한다.
- P3-2는 seeded E2E API에 in-memory 전략 저장소 fixture를 추가해 브라우저 E2E에서 실제 REST 검증·게시 흐름을 검증한다.
- P3-2는 표적 Vitest, P3 계약 pytest, 웹 빌드, Playwright Strategy Studio E2E, `ruff`, `mypy`, `npm test`, 전체 `pytest`, 전체 Playwright E2E, Docker compose build 통과를 확인했다.

## 현재 제외·대기

- P3-2 원격 CI는 브랜치 푸시 뒤 기록한다.
- 백테스트 실행, 체결 모델, 봇·주문·위험 연결은 P4 이후 범위다.
- 주식시장 기능과 타 거래소 실제 연동은 현재 범위가 아니다.
