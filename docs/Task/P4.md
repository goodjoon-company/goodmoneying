추적: GitHub Issue #31
시작일: 2026-07-18

## 목표

P4는 같은 데이터셋·전략·엔진 버전·파라미터·seed가 같은 결과를 반환하는 백테스트 체계를 만든다. 연구, 백테스트, paper·shadow·live-ready 단계는 같은 전략 평가 의미와 주문 체결 의미를 공유해야 한다.

## P4-1 완료 범위

- `backtest-core-v1` 순수 백테스트 엔진 추가
- dataset content hash, strategy graph hash, parameter hash, deterministic seed 기반 입력 hash 고정
- `(knowledge_at, source_priority, stable_sequence)` 사건 정렬
- wall clock과 직접 난수 사용 금지
- Decimal 기반 수수료·슬리피지(slippage)·지연(latency)·부분 체결(partial fill) 계산
- `orderbook_absent_uses_candle_close`, `partial_fill_by_candle_volume_participation` 가정 기록
- golden replay 신호 동등성 검증

## P4-2 완료 범위

- `backtest_runs`, `backtest_trades`, `backtest_equity_points`, `backtest_metrics`, `backtest_artifacts` DB 계약 추가
- `input_hash`, `result_hash`, dataset content hash, strategy graph hash 영속화
- published 전략 버전과 sealed 데이터셋 버전만 run 입력으로 허용
- terminal run과 결과 행 append-only 봉인
- `PostgresBacktestStore` completed run 저장·조회·멱등 재생 구현
- dbmate migration E2E와 live PostgreSQL Backtest Store E2E 통과

## 남은 P4 범위

- Backtest Worker 임대·재시도·artifact 저장
- REST·WebSocket 진행 이벤트
- Backtest Lab UI와 자동 E2E
- walk-forward, sensitivity, bootstrap 성과 artifact
