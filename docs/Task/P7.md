## P7-1 품질 증적 매니페스트 완료 목표

- `docs/contracts/quality/p7-quality-evidence.yaml`을 P7 품질 게이트(Quality Gate)의 단일 기준으로 둔다.
- readiness mode는 필요한 gate, 실행 명령, 증적 위치, 명시적 임시 예외를 검증한다.
- release mode는 모든 gate가 `passed`이고 `docs/Test/` 증적이 존재할 때만 통과한다.
- CI는 P7 readiness gate를 실행해 이후 P7 조각의 누락을 조기에 차단한다.
- 제품 코드의 미해결 `TODO`, `FIXME`, 임시 목(mock), 문서화되지 않은 수동 절차를 스캔한다.

## P7-1 안전 경계

- readiness mode의 허용 경로는 P7 후속 조각에서 제거할 부채 목록이며 완료 판정이 아니다.
- 이 단계는 실거래, 외부 부하 발생, 운영 DB 복구 실행을 수행하지 않는다.
- release mode는 후속 P7 조각이 실제 측정과 복구 리허설 증적을 남긴 뒤 사용한다.
