# P7 상용 운영 품질 증명

상태: 진행 중
GitHub Issue: #34

## 목표

성능(Performance), 접근성(Accessibility), 보안(Security), 부하(Load), 장시간(Soak),
장애 주입(Chaos), 백업·복구(Backup/Restore), 미해결 산출물 정리를 추측이 아닌 자동화
증적으로 증명한다.

## 범위 밖

- 주식시장 기능과 타 거래소 실제 연동
- 출금 기능·권한
- 실제 자금을 움직이는 주문 또는 AI/CI의 live 활성화
- 운영 서버 직접 수동 수정

## P7-1 품질 증적 매니페스트 완료 목표

- `docs/contracts/quality/p7-quality-evidence.yaml`을 P7 품질 게이트(Quality Gate)의 단일 기준으로 둔다.
- readiness mode는 필요한 gate, 실행 명령, 증적 위치, 명시적 임시 예외를 검증한다.
- release mode는 모든 gate가 `passed`이고 `docs/Test/` 증적이 존재할 때만 통과한다.
- CI는 P7 readiness gate를 실행해 이후 P7 조각의 누락을 조기에 차단한다.
- 제품 코드의 미해결 `TODO`, `FIXME`, 임시 목(mock), 문서화되지 않은 수동 절차를 스캔한다.

## P7-1 안전 경계

- readiness mode의 허용 경로는 P7 후속 조각에서 제거할 부채 목록이며 완료 판정이 아니다.
- 이 단계는 실거래, 외부 부하 발생, 운영 DB 복구 실행을 수행하지 않는다.
- release mode는 후속 P7 조각이 실제 측정과 복구 리허설 증적을 남긴 뒤 사용한다.

## 후속 범위

## P7-2 성능 gate 완료 목표

- Web Vitals, 첫 유용 셸, 실시간 event 반영을 Playwright E2E로 자동 측정한다.
- `package.json` script와 `docs/contracts/quality/p7-quality-evidence.yaml`의 performance gate 명령을 일치시킨다.
- 통과 증적은 gate별 `docs/Test/2026-07-25-P7-*-검증.md`에 남긴다.

## 후속 범위

## P7-3 접근성·viewport gate 완료 목표

- WCAG 2.2 AA proxy와 7개 viewport overflow 검증을 Playwright E2E로 자동화한다.
- `package.json` script와 품질 매니페스트의 accessibility gate 명령을 일치시킨다.

## 후속 범위

## P7-4 보안 gate 완료 목표

- dependency, image, secret, auth/input 보안 gate를 자동화한다.
- `npm audit --audit-level=high` 기준 high·critical 취약점 0건을 요구한다.
- 런타임 이미지의 root 실행을 금지하고 web 이미지는 내부 비특권 포트 8080을 사용한다.
- 운영 키·토큰 평문 노출 scan을 P7 gate에 편입한다.
- 인증·입력 경계 회귀 테스트를 P7 보안 gate로 고정한다.

## 후속 범위

## P7-5 resilience gate 완료 목표

- load, soak, chaos probe를 local profile로 자동화한다.
- 운영 서버와 외부 Upbit API에 부하를 주지 않고 seeded SQLite repository와 FastAPI TestClient로 반복 요청·단기 지속 실행·일시 장애 주입을 검증한다.
- probe 결과를 `docs/Test/`와 품질 매니페스트에 연결한다.
