## 주요 확인

- 대시보드 API 계약에 `metricPrinciples`, `collectionActivity`, `storageBreakdown`, `operationsTrend`, `missingRangeTop`, `auditLogSummary`가 포함된다.
- `rateLimitRemainingPercent`, `duplicateRows24h`는 totals 또는 KPI 카드로 표시하지 않고, 미노출 원칙으로만 표시한다.
- 코인별 수집 상태 행은 거래대금, 등락률, 최신성, 커버리지, 저장량을 백엔드 View Model로 렌더링한다.
- 백필 승인 패널은 실제 `collection_target_changes`와 `backfill_jobs` 기반 감사 로그 요약을 표시한다.
- Chromium E2E에서 히트맵, 저장량 breakdown, 운영 추이, 결측 상위, 백필 생성/승인, 상세 품질 이력, 확장성 점검 화면이 모두 표시된다.

## 잔여 리스크

- Rate limit 여유율과 중복 저장 시도 수는 관측 저장소가 없으므로 의도적으로 미노출 상태다.
- 확장성 점검은 M3.5 의사결정 게이트 화면이며 실제 CPU/메모리/TPS/QPS 모니터링이 아니다.
