## 주요 확인

- `DashboardSummary.totals`는 `storageRowsToday`, `realtimeRowsLastMinute`, `backfillRowsLastMinute`를 제공한다.
- 대시보드와 시장 리스트는 `storageRowCount`를 주요 저장 단위로 표시한다.
- `InstrumentDetail`은 24시간 가격 변동과 거래량 변동을 백엔드 계산값으로 제공한다.
- SQLite와 PostgreSQL 저장소 모두 신규 Row Count 계약을 구현한다.
- 백필 저장도 대상별 수집 결과를 기록해 최근 1분 백필 수집 행 수에 포함된다.
- Vite 개발 서버는 기본 `/api` 요청을 로컬 API 서버로 프록시한다.
- 프론트엔드 API 클라이언트는 구버전 대시보드 응답을 첫 화면 렌더 가능한 형태로 정규화한다.

## 잔여 리스크

- 24시간 거래량 등락율은 저장된 1분봉 기준이다. 해당 기간 캔들이 없으면 0으로 계산된다.
- Rate limit 백분율과 중복 저장 시도 수는 별도 관측 저장소가 생기기 전까지 의도적으로 표시하지 않는다.
