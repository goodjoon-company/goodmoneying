## 런타임 확인

- `./dev.sh app restart all`로 API, web, realtime collection worker, backfill collection worker를 재시작했다.
- 실행 DB의 `collection_runs_data_type_ck`와 `target_collection_results_data_type_ck` 모두 `trade_event`를 허용하는 상태로 변경됐다.
- `trade_events_count`는 재시작 후 `1124`에서 `1816`으로 증가했다.
- 최근 5분 `trade_event` collection run은 `succeeded=171`로 확인했다.
- `/v1/dashboard/summary` 기준 `workerStatus.realtime.status=running`, `statusDetail=최근 heartbeat 정상`으로 회복됐다.
- 히트맵 전체 집계는 `totalTradeCount=2256`, `nonzeroCells=56`으로 체결 빈도 반영을 확인했다.
- PostgreSQL 최근 3분 로그에서 `violates check constraint`, `deadlock`, `ERROR` 재발은 확인되지 않았다.

## 남은 리스크

- backfill worker는 `./dev.sh app restart all` 이후 running 상태로 재시작됐다. 현재 poll 결과는 `rows=0`이며 대기 lock은 없다.
- worker heartbeat의 `last_error_at`은 과거 오류 시각을 보존하지만, 현재 상태 판단은 `running`과 최신 successful heartbeat 기준으로 정상 회복된다.
