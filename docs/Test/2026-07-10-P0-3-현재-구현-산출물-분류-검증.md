## E2E(End-to-End)와 QA 판단

이번 작업은 런타임 코드, API/DB 계약, 사용자 화면 동작을 변경하지 않고 기존 구현의 제품상 위치와 문서 보존 정책만 정리한다. 따라서 새 자동화 종단 간 테스트(E2E Test)와 브라우저 QA는 실행하지 않았다. 각 런타임 산출물의 기존 E2E·브라우저 증적은 삭제하지 않고 `docs/Test/`에 유지했다.

## 리뷰 결과

- 요구사항 적합성: 주요 산출물 8개를 네 분류 중 하나로 정하고 이유와 다음 처리를 기록했다.
- 아키텍처 적합성: 구현 구조는 변경하지 않았고, 보류 결정과 충돌하는 기존 아키텍처 문구는 Issue #7 후속 위험으로 남겼다.
- 계약 적합성: API, DB, 메시지 계약 변경이 없다.
- 추적성: 상세 Task 제거 후 Milestone 요약, Test, History, ADR 보존 위치를 매핑했고 깨진 Task 경로 참조를 제거했다.
- 보안·성능·운영 안정성: 실행 코드와 배포 설정 미변경으로 새로운 영향이 없다.
- 남은 위험: 실시간 체결 히트맵의 투자 판단 가치는 미검증이며, 먼 미래 로드맵과 요구사항 상태는 Issue #6에서 재작성해야 한다.
