Date: 2026-07-10
Related Task: GitHub Issue #6
Environment: macOS 로컬 저장소, `codex/p0-4-post-mvp-correction`, `codex/p0-4-ci-test-stability`, GitHub Actions

## 검증 대상

- `docs/01_Product.md`가 제품 목표와 가까운 단계를 먼저 설명하는지 확인한다.
- M1~M3가 완료된 초기 제품 기준선이고 현재 범위가 초기 구현 이후 제품화(Post-MVP Productization)로 구분됐는지 확인한다.
- 운영 도구와 사용자 가치 화면의 경계가 명시됐는지 확인한다.
- `GM-PROD-001`부터 `GM-PROD-024`까지 현재 상태가 재분류됐는지 확인한다.
- M0~M11 장기 실행 계획이 `P0`, `P1`, `P2`, `P3`, `Later` 단계와 결정 게이트로 축소됐는지 확인한다.
- 기존 수집 품질 산식과 임계값이 의미 변경 없이 보존됐는지 확인한다.
- 런타임 코드와 API·DB 계약이 변경되지 않았는지 확인한다.
- 전체 CI에서만 시간 초과된 관심종목 복합 회귀 테스트가 안정적으로 완료되는지 확인한다.

## 코드·문서 리뷰

- 요구사항 적합성: Issue #6의 제품 원칙, 완료된 초기 기준선, 현재 제품화 범위, 요구사항 상태, 가까운 로드맵을 모두 반영했다.
- 사용자 피드백 반영: M1~M3 완료 이후의 현재 단계를 MVP 구현으로 표현한 오류를 제거했다.
- 아키텍처 적합성: 현재 아키텍처나 모듈 책임을 변경하지 않았다. 제품 기준과 남은 아키텍처 문구의 정렬은 Issue #7로 유지한다.
- 계약 변경 여부: OpenAPI와 DB 스키마 변경이 없으며 메시지 계약도 변경하지 않았다.
- 기존 의미 보존: 수집 품질 산식·임계값과 구현된 호가 요약·화면 경계·Backfill 실행 규칙은 구조 단순화 과정에서 삭제하지 않고 제품 기준에 유지했다.
- 테스트 영향: 제품 동작과 기대값은 바꾸지 않았다. CI에서 연속 두 번 5초 제한을 넘긴 복합 사용자 흐름 테스트 하나에만 10초 제한을 적용했다.
- 보안·성능·운영 안정성: 실행 경로를 바꾸지 않는 문서 전용 변경이라 신규 영향이 없다.
- 추적성: Issue #6, Product, 이 Test, `docs/History/2026-07-10-P0-4-제품-기준-문서-재작성.md`를 연결했다.

## 종단 간 테스트와 QA

이번 작업은 런타임 코드, API·DB·메시지 계약, 사용자 화면 동작을 변경하지 않는다. 관심종목부터 Backfill 저장까지 연결하는 기존 자동화 회귀 테스트와 전체 웹 테스트를 재실행했다. 새 기능이 아니므로 별도 종단 간 테스트(E2E Test), 브라우저 QA, 디자인 QA는 실행하지 않았으며 현재 기능의 기존 자동화·브라우저 증적은 `docs/Test/`에 보존돼 있다.
