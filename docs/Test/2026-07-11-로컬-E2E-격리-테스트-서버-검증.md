# 로컬 E2E 격리 테스트 서버 검증

Status: Complete
Date: 2026-07-11
Related Task: [GitHub Issue #10](https://github.com/goodjoon-company/goodmoneying/issues/10)
Environment: macOS 로컬 저장소, `codex/issue-10-isolated-e2e`

## 검증 목표

- 루트 `.env`와 외부·운영 데이터베이스(DB, Database) 없이 `npm run e2e`가 단독 실행되는지 확인한다.
- 수집 후보군 100개와 활성 수집 대상 50개를 명시적인 테스트 저장소 주입으로만 준비하는지 확인한다.
- 런타임 fixture·demo repository 경로를 다시 허용하지 않았는지 확인한다.
- 사용자 지정 운영 토큰이 API와 웹에 안전하게 같은 값으로 전달되는지 확인한다.
- E2E 종료 뒤 API·웹 서버 프로세스가 남지 않는지 확인한다.
- CI가 Chromium 설치와 격리 E2E를 필수 품질 게이트로 실행하는지 확인한다.

## 구현 경계

- `tests/e2e/seeded_api.py`가 SQLite 저장소를 만들고 `FixtureUpbitClient`를 `seed_repository`에 직접 주입한다.
- 테스트 전용 프록시는 FastAPI의 동기 endpoint와 SSE가 같은 SQLite 연결을 동시에 사용하지 않도록 repository 공개 호출 단위를 직렬화한다.
- `playwright.config.ts`는 `.env`를 source하지 않고 API·웹 서버별 환경변수를 셸 문자열이 아닌 `env` 속성으로 전달한다.
- API·웹 호스트와 포트도 사용자 지정 URL에서 파싱한 뒤 따옴표로 감싼 환경 변수로 전달해 셸 명령 문자열에 직접 보간하지 않는다.
- `GOODMONEYING_DATABASE_URL`은 Playwright가 관리하는 테스트 API에서 빈 값으로 고정한다.
- `GOODMONEYING_DEMO_DATA=1` 런타임 경로는 복원하지 않았다.
- 배포 환경 검증용 `E2E_SKIP_WEBSERVER=1` 동작과 외부 URL 계약은 유지했다.
- 로컬 E2E가 성공하면 Node.js 종료 검증기가 API·웹 포트 연결 불가를 확인한다. 배포 환경은 외부 서버를 종료하면 안 되므로 검증기를 명시적으로 건너뛴다.

## 계약과 문서 영향

- 제품 요구사항과 사용자 화면 동작은 변경하지 않았다.
- OpenAPI·DB·메시지 계약은 변경하지 않았다.
- 배포 E2E는 기존 `E2E_SKIP_WEBSERVER=1` 경로를 그대로 사용한다.
- 아키텍처 단일 기준(source of truth)에 운영 런타임, 로컬 격리 E2E, 배포 환경 E2E의 시험 데이터 경계를 구분해 기록했다.
- 모듈 경계와 공개 제품 계약을 바꾸지 않는 되돌리기 쉬운 테스트 하네스 변경이므로 새 ADR(Architecture Decision Record)은 필요하지 않다고 판정했다.
- README와 M1 사용 설명서에 로컬 격리 E2E 동작과 배포 환경 검증 경계를 기록했다.

## 코드 리뷰

- 요구사항 적합성: `npm run e2e` 단일 명령, 외부 DB 비사용, 명시적 fixture 주입, 서버 정리, CI 실행을 모두 충족한다.
- 아키텍처 적합성: 테스트 지원 코드는 `tests/e2e/`에만 두고 제품 API·repository에 테스트 전용 메서드를 추가하지 않았다.
- 보안 적합성: 사용자 입력 가능 토큰·호스트·포트를 셸 명령에 직접 보간하지 않고 프로세스 환경으로 전달한다.
- 테스트 적합성: 실제 FastAPI·React·Playwright 흐름을 검증하며 내부 mock 행위를 assertion하지 않는다.
- 운영 적합성: 운영 DB와 배포 URL을 사용하지 않고 전용 포트의 프로세스를 종료한다.
- 프론트엔드 디자인 QA: 화면 코드·스타일·반응형 동작을 변경하지 않아 별도 디자인 리뷰는 미적용했다.
- 독립 코드 리뷰: 최초 리뷰의 중요(Important) 발견사항인 URL 명령 인자 보간과 아키텍처 문서 불일치를 수정하고 서버 종료 자동 검증 요구도 반영했다. 재리뷰 결과 Critical·Important·Normal·Minor 추가 발견사항 0건, 병합 가능(Ready to merge) 판정을 받았다.

## 결론

Issue #10의 로컬 E2E 격리 하네스 복구 범위는 통과했다. 로컬과 CI가 같은 `npm run e2e` 진입점을 사용하고, 런타임 fixture 경로와 외부 DB 의존 없이 브라우저 시나리오를 검증한다.
