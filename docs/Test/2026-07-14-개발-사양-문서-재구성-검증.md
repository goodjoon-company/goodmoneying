## Mermaid 수동 검토

- `docs/02_Architecture.md`에 시스템 문맥, 소프트웨어 아키텍처, 수집·집계·분석 흐름, 브라우저 실시간 전송의 Mermaid 블록 4개가 있다.
- `docs/02_Architecture/upbit-collection-pipeline.md`에 모듈 경계 Mermaid 블록 1개가 있다.
- `docs/README.md`에 문서 변경 흐름 Mermaid 블록 1개가 있다.
- 저장소에는 Mermaid CLI(Command Line Interface)가 설치되어 있지 않아 실제 SVG 렌더링 자동 검증은 수행하지 못했다. 블록 경계와 현재 코드 경로·계약 경로를 수동 대조했다.

## 문서 리뷰

- 제품 사양은 제품 목표·범위·요구사항·로드맵을 유지하고 구현 상세를 아키텍처와 계약으로 연결한다.
- 아키텍처 사양은 현재 구현된 단일 API, 세 워커, PostgreSQL과 업비트 경계만 그리며 미래 후보를 현재 구성으로 표현하지 않는다.
- 계약 사양은 OpenAPI·JSON Schema·SQL DDL을 복제하지 않고 소비자와 변경·검증 순서를 안내한다.
- 사용 안내는 `dev.sh` 기반 실행, 데이터 준비, 분석·시스템 관리, 문제 확인의 실제 순서를 따른다.
