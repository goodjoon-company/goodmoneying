## E2E와 화면 QA

`업비트 API 테스트 화면에서 공개 캔들 차트와 보조지표를 조회한다` 시나리오는 5분봉 응답 20개를 주입해 다음을 확인했다.

- 메뉴 진입, 5분봉 선택, 캔들 조회
- 캔들 차트와 최신 OHLCV·SMA 20·EMA 20·RSI 14 렌더링
- 390px viewport에서 조회 조건과 차트를 표시하고 `.app-shell` 가로 넘침이 없는지 확인

## 코드 리뷰 후 조치

독립 코드 리뷰에서 발견한 P1·P2 항목을 모두 수정하고 회귀 테스트를 추가했다.

- 조회 완료 시점의 거래쌍·주기를 결과 상태로 고정해, 입력을 다시 바꾸기만 해도 기존 데이터를 새 조건으로 오표기하지 않는다.
- 새 요청 전에 이전 `AbortController`를 취소하고, 완료된 최신 요청만 결과를 반영한다.
- 빈 배열은 최초 진입 안내와 구분해 `해당 조건에는 업비트 캔들 데이터가 없습니다.`로 표시한다.
- 차트에 최신 10개 OHLCV를 읽을 수 있는 표 대안을 추가했다.

`design-review` 자동 절차는 기존 미커밋 작업 트리에서 개별 커밋을 요구해 중단했다. 대체로 Playwright 데스크톱·390px QA와 44px 모바일 제어 영역을 적용했다.

## 전체 정적 검사 상태

`uv run mypy apps/api apps/worker packages/shared tests`는 이번 화면과 무관한 미커밋 집계 워커 파일 `apps/worker/goodmoneying_worker/aggregation_collection_worker.py`에서 다음 2건으로 실패했다.

- 30행: `CandleAggregationWorker`에 `repository` 속성이 없음
- 36행: `CandleAggregationWorker`에 `repository` 속성이 없음

해당 파일은 이번 Issue 범위가 아니므로 수정하지 않았다. Ruff와 Python 전체 테스트는 통과했다.

## 코드 리뷰

- 요구사항·P2·Issue #14: 거래쌍, 업비트 지원 분 단위 전체(1·3·5·10·15·30·60·240분), 일·주·월, OHLCV와 보조지표를 제공한다.
- 아키텍처·계약: 브라우저의 공개 읽기 전용 호출로 한정해 내부 API·DB·수집 워커·WebSocket 계약을 변경하지 않는다.
- 보안·운영: API 키를 사용하지 않고 거래쌍·개수를 요청 전에 제한하며, HTTP 실패와 비정상 응답을 화면 오류로 표시한다.
- 발견·조치: 초기 리뷰에서 누락된 3·10·30분 단위와 모바일 44px 제어 영역을 수정했다. 남은 이번 변경 범위의 차단 또는 후속 발견사항은 없다.
