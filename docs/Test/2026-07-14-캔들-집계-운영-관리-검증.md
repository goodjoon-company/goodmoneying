## 수동 확인 경로

Podman이 설치된 환경에서 `./dev.sh infra start` 후 API·web·실시간 수집·백필·캔들 집계 워커를 시작한다. 브라우저의 `시스템 관리` 메뉴에서 WebSocket 연결, 실시간 수집 대상의 데이터 유형, Backfill 대상, 집계 대상별 행 수와 전체 진행률을 확인한다.

현재 작업 환경은 `podman-not-found`여서 로컬 PostgreSQL 장기 실행 검증은 하지 못했다. 격리 SQLite 기반 Playwright E2E가 API/WebSocket/UI 경로를 대신 검증했다.
