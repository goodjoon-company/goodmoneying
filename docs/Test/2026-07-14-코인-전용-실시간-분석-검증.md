### 설계·계약 판단

- 서버는 유효한 각 구독마다 `analysis.session`을 첫 프레임으로 보내고 이후 메시지를 순서대로 전송한다. 후속 차트·지표·시장 프레임에는 상품 ID가 없으므로 프레임별 상품 추측 대신 이 승인 경계를 사용한다.
- 클라이언트 요청 ID와 서버 응답 필드를 새로 추가하지 않고 기존 `analysis.session` 순서를 사용해 JSON Schema 호환성을 유지한다.
- 새 구독을 전송하는 순간 이전 승인 세대를 무효화한다. 승인이 겹치면 대기 세대를 FIFO로 소비하므로, 이전 세션 승인을 늦게 받아도 최신 세대로 잘못 대응하지 않는다.
- 시각 마크업·스타일, API 인증, 외부 업비트 경계는 변경하지 않는다.
