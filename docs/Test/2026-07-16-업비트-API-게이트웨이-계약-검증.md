# 2026-07-16 업비트 API 게이트웨이 계약 검증

Date: 2026-07-16
Result: Pass
Related Issue: [#19](https://github.com/goodjoon-company/goodmoneying/issues/19)
Related ADR: [ADR-0011](../ADR/ADR-0011-업비트-API-게이트웨이와-비파괴-테스트-경계.md)

## 검증 범위

- 업비트 개발자 센터 v1.6.3 `llms.txt`와 개별 공식 마크다운(markdown) 기준 REST 활성 50개·사용 중단(deprecated) 1개, WebSocket 데이터 스트림 14개·운용 1개
- REST 메서드·경로·파라미터 타입·필수 여부·요청 제한 그룹·`read|test|blocked` 안전 등급
- 실제 주문, 모든 취소, 자산 이전, 입출금 생성·취소, 트래블룰 검증 14개 `blocked`
- 주문 생성 테스트 1개 `test`, 그 밖의 조회 기능 `read`
- 게이트웨이 OpenAPI와 WebSocket JSON Schema, health·catalog·403·404·422·501 로컬 판정 골격
- 업비트 상향 호출 코드와 키·인증 정보 부재

## 테스트 주도 개발 증적

첫 RED에서 계약 파일과 게이트웨이 모듈을 만들기 전에 다음 명령을 실행했다.

```bash
uv run pytest -q tests/contracts/test_upbit_gateway_contract.py tests/upbit_gateway/test_app.py
```

결과는 `8 failed`였다. 세 계약 파일의 `FileNotFoundError`와 `goodmoneying_upbit_gateway`의 `ModuleNotFoundError`로 실패해 기대한 누락을 검증했다. 최소 카탈로그·계약·FastAPI 골격 구현 후 같은 명령은 `8 passed`였다.

첫 GREEN 리뷰에서 WebSocket 파라미터가 이름만 있고 타입·필수 여부가 없는 누락을 발견해 실패 테스트를 추가했다. 단일 테스트는 문자열을 타입 계약으로 읽을 수 없어 `1 failed`였고, 타입·필수·기본값 계약을 추가한 뒤 전체 관련 테스트는 `9 passed`가 됐다.

후속 코드 리뷰 지적을 재현하는 테스트를 먼저 추가했다. `LIST_SUBSCRIPTIONS` 분리, OpenAPI 런타임 동등성, endpoint_id 상태 구분, 작업 디렉터리(CWD) 독립 가져오기, 공식 스냅샷(snapshot), 요청 제한 명칭·구조 검증에서 `9 failed, 5 passed`를 확인했다. 구현 후 동일 범위는 `14 passed`가 됐다. 구체 응답 스키마에 포함된 공식 출처 `source_url`을 요청 URL 허용으로 오탐한 테스트 1건은 `GatewayRequest.properties`만 검사하도록 범위를 바로잡았다.

패키징·계약 의미 동등성 후속 리뷰는 테스트를 먼저 추가한 뒤 다음 명령에서 `4 failed, 8 passed`를 확인했다.

```bash
uv run pytest -q tests/contracts/test_upbit_gateway_contract.py tests/upbit_gateway/test_wheel.py
```

실패 원인은 해시(hash) 전용 스냅샷에 명시 투영이 없고, wheel에 카탈로그가 없으며, 설치본이 저장소 상대 경로를 찾고, FastAPI `/health`가 체크인 `Health` 스키마를 참조하지 않는 것이었다. 패키지 리소스·명시 투영·런타임 모델 제약을 구현한 뒤 같은 범위는 `12 passed`, 전체 게이트웨이 표적 범위는 `17 passed`가 됐다.

## 코드·문서 리뷰

Issue #19, 제품 P2.2·GM-PROD-030~033, ADR-0011, 게이트웨이 모듈 설계와 세 기계 계약을 기준으로 요구사항·아키텍처·계약·보안·테스트를 검토했다. 첫 리뷰에서 WebSocket 파라미터 타입 계약, Ruff import·상수 형식, mypy의 jsonschema 타입 선언 문제를 발견해 수정했다. 후속 리뷰에서 데이터 스트림·운용 혼합, OpenAPI 런타임 불일치, endpoint_id 상태 혼합, 작업 디렉터리(CWD) 결합, 공식 계약 드리프트 검출 부재, 요청 제한 명칭·구조 모호성을 발견해 수정했다. 재리뷰에서 wheel의 저장소 경로 의존, OpenAPI 구성요소 의미 차이, 사람이 검토할 수 없는 해시 전용 스냅샷을 발견해 패키지 리소스·구성요소 비교·명시 투영으로 보완했다. 재검증 기준 치명적(Critical)·중요(Important) 발견사항은 0건이다. 리뷰 서브에이전트는 동시 실행 한도 때문에 만들지 못해 동일 체크리스트로 직접 리뷰했다.

프론트엔드 변경은 없으므로 디자인·브라우저 QA는 적용하지 않았다. 이 이슈의 E2E 범위는 계획대로 FastAPI TestClient를 통한 health·catalog·상향 호출 차단 자동 검증이다.

## 남은 경계

- 공식 문서 드리프트(drift)는 후속 실행 기능 착수 시 다시 확인해야 한다.
- JWT·허용 목록 실행·요청 제한·마스킹·실제 WebSocket 연결은 Issue #20 이후 범위다.
- Starlette TestClient가 `httpx` 사용 중단 예정 경고 1건을 출력하며 현재 테스트 결과에는 영향이 없다.
