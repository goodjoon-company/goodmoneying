# 2026-07-16 업비트 REST 실행 엔진 검증

Date: 2026-07-16
Result: Pass
Related Issue: [#20](https://github.com/goodjoon-company/goodmoneying/issues/20)
Related ADR: [ADR-0011](../ADR/ADR-0011-업비트-API-게이트웨이와-비파괴-테스트-경계.md)

## 검증 대상

- HS512 JWT, 매 요청 UUID nonce, 입력 순서와 배열 키 반복을 보존한 SHA512 쿼리 해시(query hash)
- 저장소 밖 환경 변수 또는 절대 경로 읽기 전용 파일 자격 증명
- 공식 REST 그룹별 요청 제한(rate limit), `Origin` 1회/10초, `Remaining-Req`, 429·418 냉각(cooldown)
- `read`·`test` 실행과 `blocked`의 자격 증명·제한기·네트워크 이전 차단
- 고정 공식 URL과 명시적 루프백(loopback) 테스트 URL 허용 목록
- 키·JWT·Authorization·쿼리 해시(query hash)의 추적 봉투(Trace Envelope) 마스킹
- `dev.sh`의 `upbit-gateway` 8001 start·stop·restart·status·logs·readiness와 단독 실행 시 DB 비의존성
- 게이트웨이 Dockerfile, 로컬 Compose, 환경 샘플, CI 타입 검사·이미지 빌드

## 테스트 주도 개발 증적

인증, 제한기, 안전 정책, 클라이언트, 추적, 실행기, 개발 스크립트, 컨테이너 계약, 프로세스 E2E 순서로 실패 테스트를 먼저 작성했다. 각 첫 실행은 모듈 누락(`ModuleNotFoundError`), 심볼 누락(`ImportError`), 계약 누락(`KeyError`), 예상 상태 불일치로 실패했고 최소 구현 뒤 같은 범위를 통과시켰다.

[공식 인증 문서](https://docs.upbit.com/kr/reference/auth)의 Python 예시가 `unquote(urlencode(..., doseq=True))`를 사용하는 것을 재확인해 공백의 `+` 표현과 입력 순서를 계약 테스트로 고정했다. [공식 요청 수 제한 문서](https://docs.upbit.com/kr/reference/rate-limits)를 기준으로 그룹 한도, `Remaining-Req`, 429·418, `Origin` 제한을 재검토했다. 실제 키 파일은 읽거나 복사하지 않았고 모든 인증 테스트는 64바이트 가짜 비밀 키만 사용했다.

## E2E 안전 증적

가짜 상류 호출 목록을 별도 조회해 브라우저 `Origin`이 전달되지 않았고 `POST /v1/orders` 호출이 0회임을 확인했다. 취소·자산 이전·입출금·트래블룰 검증을 포함한 모든 상태 변경 endpoint는 카탈로그 계약상 `blocked`이고 실행기 안전 검사가 HTTP 클라이언트보다 먼저 수행된다. 응답 문자열에는 가짜 접근 키·비밀 키가 없었다.

## 코드·보안·설계 리뷰

Issue #20, 제품 P2.2·GM-PROD-032~033, 게이트웨이 아키텍처, OpenAPI, 카탈로그, ADR-0011을 기준으로 직접 리뷰했다. 첫 리뷰에서 한 그룹의 제한 대기가 다른 그룹까지 잠그는 전역 잠금, 호스트 이름·경로가 붙은 테스트 URL 허용, 임의 응답 필드에 반사된 쿼리 해시(query hash), 네 가지 대안 필수 파라미터 조합의 비기계적 표현, 연결 예외 원문 보존, Compose 자격 증명 전달 누락을 발견했다.

각 항목에 실패 테스트를 추가한 뒤 그룹 잠금 해제 후 대기, 루프백 IP 리터럴과 경로 없는 URL만 허용, 계산된 쿼리 해시의 값 기반 마스킹, `any_of_required` 계약, 원인 사슬 없는 고정 연결 오류, 셸 환경 자격 증명 전달로 수정했다. 재리뷰와 전체 검증 기준 치명적(Critical)·중요(Important) 발견사항은 0건이다.

## 미검증 항목

- 실제 Upbit 계정·키·네트워크 호출은 안전 정책에 따라 의도적으로 검증하지 않았다.
- WebSocket 중계와 프론트엔드 기능별 시각화는 후속 Issue 범위다.

## 결론

가짜 자격 증명과 가짜 상류만 사용한 자동화 검증에서 REST 실행·인증·제한·마스킹·차단·개발 런타임 경계가 요구사항, 아키텍처, OpenAPI, 카탈로그와 일치한다.
