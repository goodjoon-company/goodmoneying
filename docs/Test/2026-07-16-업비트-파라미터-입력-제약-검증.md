## 공식 기준 재확인

2026-07-16 업비트 개발자 센터 최신 문서에서 캔들·체결·호가·주문 목록·자산 이전·입출금·주문 취소 파라미터를 다시 확인했다. 특히 다음 누락을 함께 수정했다.

- 호가 REST `count`는 WebSocket 지원값 목록이 아니라 기본 30·범위 1~30이다.
- 일괄 주문 취소 `count`는 기본 20·최대 300이고 페어 목록은 최대 20개다.
- ID 주문 목록 조회는 최대 100개, ID 주문 목록 취소는 최대 20개다.
- 입출금 목록 `from`·`to`는 날짜가 아닌 UUID 페이지네이션 커서다.
- 자산 이전·입출금 `amount`, 주문 `price`·`volume`, 호가 `level`은 숫자 형식 문자열이다.
- 포켓 이전 기간은 ISO 8601 또는 밀리초 타임스탬프를 허용하고 최대 7일이다.

## E2E 시나리오

- 분 캔들 화면에서 `count` 공식 기본값 1과 `기본 1 · 최소 1 · 최대 200 · 단위 개` 안내를 확인했다.
- `현재 시각(Now)` 클릭 시 날짜·시·분·초가 채워지고 `count=201`은 게이트웨이 요청 증가 없이 화면에서 차단됐다.
- Exchange 체결 대기 주문에서 `limit=100` 기본값과 최대 100 안내를 확인했다.
- 위험 작업은 기존 정책대로 미리보기만 제공하고 업비트 상향 호출을 만들지 않았다.
- 390px에서 가로 넘침이 없고 키보드 탭 이동이 유지됐다.

## 수동 시각 확인

데스크톱 1440×1000에서 Quotation 요청 패널을 확인했다. 날짜·시간 입력, KST 안내, 현재 시각(Now), 숫자 제약과 빠른 입력 버튼이 한 열에 겹침 없이 표시됐다. 네이티브 날짜·시간 입력이 브라우저의 달력·시계 선택기를 제공하며 필수 필드에는 불필요한 지우기 버튼이 나타나지 않는다.

## 2026-07-17 날짜 선택 아이콘 대비 후속 검증

- 사용자 시각 QA에서 어두운 Quotation 입력의 네이티브 달력 아이콘이 검은색으로 렌더링되어 배경과 구분되지 않는 문제를 재현했다.
- 브라우저 계산 스타일(Computed Style)을 확인한 결과 `datetime-local` 입력의 `color-scheme`이 `normal`이었다. 입력 글자색과 달리 네이티브 아이콘은 사용자 에이전트(User Agent) 기본 밝은 테마 비트맵을 사용한 것이 원인이었다.
- RED: `npx playwright test tests/e2e/m1-operations.spec.ts --grep "날짜 선택 아이콘"`은 Quotation의 기대값 `dark`와 실제값 `normal` 차이로 1건 실패했다.
- GREEN: Quotation 날짜·시간 입력에는 `color-scheme: dark`, 밝은 Exchange 패널에는 `color-scheme: light`를 적용한 뒤 같은 테스트가 1건 통과했다.
- 데스크톱 1440×1000 시각 확인에서 어두운 입력 우측 달력 아이콘이 흰색으로 선명하게 표시됐다. 전체 Playwright 15건과 기존 390px 반응형 시나리오도 통과했다.
