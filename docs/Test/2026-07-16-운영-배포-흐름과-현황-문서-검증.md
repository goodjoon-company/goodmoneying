## 현재 확인 한계

- 점검 장비에서 Mac Mini M4의 mDNS SSH 주소가 시간 초과되어 PostgreSQL 컨테이너의 `pg_isready`와 runner 상태를 직접 확인하지 못했다.
- API 컨테이너에서 PostgreSQL TCP 연결과 DB heartbeat·적재 쿼리가 성공했으므로 애플리케이션의 DB 사용 경로는 동작한다.
- 현재 운영 `release-1db7d40`에는 Upbit Gateway와 캔들 집계 워커가 없어 해당 구성요소의 운영 상태를 확인할 수 없다.
- 자동화된 운영 E2E는 새 배포를 실행하지 않았으므로 이번 문서화 작업에서 재실행하지 않았다. 마지막 성공 배포 실행의 E2E 성공 이력을 배포 이력으로 사용했다.
- 현재 작업 브랜치 `9b8623c`는 원격 `main`보다 43커밋 뒤이므로 로컬 31개 회귀 테스트 통과를 최신 `main` CI 성공으로 간주하지 않는다. 최신 `main` 상태는 GitHub Actions 실행 `29451955296`의 실패 결과를 기준으로 기록했다.
