## 수동 확인

- Playwright로 1440px와 390px에서 CoinAnalysis의 11개 단위 버튼, 가로 넘침 없음,
  `계산 · 품질 · 완전성` 메타데이터를 확인했다.
- API 날짜와 지표 날짜는 RFC 3339 UTC `Z` wire format으로 일치하고 UI에서만 KST로 표시함을 확인했다.

## 독립 리뷰 후 회귀 보강

- 독립 리뷰에서 부분 물질화(materialization) 결과가 최신 원천 폴백(fallback)을 가리는 문제와,
  조회 범위에 직접 일봉이 하나라도 있으면 다른 날짜의 분봉을 버리는 문제,
  비정렬 조회 시작 시각보다 앞선 주·월 버킷을 반환하는 문제를 확인했다.
- 라이브 PostgreSQL RED 테스트로 두 누락을 각각 재현한 뒤 다음 계약으로 수정했다.
  - 같은 버킷은 물질화 결과를 우선하고, 아직 물질화되지 않은 버킷은 원천에서 계산해 병합한다.
  - 직접 일봉은 같은 UTC 날짜의 분봉만 대체하고, 직접 일봉이 없는 날짜는 분봉으로 보충한다.
  - `1d`, `1w`, `1M`은 같은 날짜별 혼합 일봉을 사용한다.
  - 원천에서 만든 버킷도 REST `from`·cursor·`to` 경계 안에 있는 결과만 반환한다.
- 기존 커버리지 테스트가 다른 테스트의 선행 실행에 의존하던 문제도 필요한 구간을 직접 생성하도록 고쳤다.

  웹 단위 140건, 브라우저 E2E 17건을 다시 통과했다.
