## 검증 범위

- 원천 개정·커버리지 품질 전이와 무효화/작업의 동일 트랜잭션 생성
- 정확한 파생 버킷, 인접 병합, 작업당 최대 512개 출력 버킷
- 범위 제한 원천·커버리지 읽기와 고정 원천 개정/품질 문맥
- 추가 전용 집계 개정, 입력·커버리지·결과 해시, 현재/시점 기준 투영
- 임대, fencing, 재시도, dead-letter, 안전 재시작과 운영 API/UI 가시성
