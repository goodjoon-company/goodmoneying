# P2-3 버전 지표와 시장 통계 검증

날짜: 2026-07-17

## 검증 범위

- SMA20·SMA60·EMA20·볼린저 밴드20·RSI14의 정의 버전, Decimal 결정론, 준비 기간과 결측 초기화
- 독립 시장 통계의 1구간 수익률, 20구간 실현 변동성, 거래량·거래대금과 완전성 상태
- 원천/집계/품질 프런티어, 부모 물질화·값 계보, `A → B → A` 시점 재현
- 내구성 무효화, DB 시계 임대·세대 fencing, 집계 우선 공정 처리와 업그레이드 bounded 초기 시드
- 512구간 체크포인트 청크 재개, 정정 `sourceAsOf` 전파, 대기열 `A → B → A` 프런티어 보존
- 저장 전용 REST 조회, `asOf`·정의 버전·고정 ID 상한 cursor, WebSocket 최신 upsert/과거 전체 갱신 분기

PostgreSQL 검증은 기존 P2 이력이 있는 업그레이드 DB에서 상품·주기당 한 초기 무효화만 생성한 뒤 워커와 실제 REST 지표 1행을 확인한다. 또한 원천·품질 `A → B → A`의 세 불변 물질화와 중간 B `asOf` 재현, 512개를 넘는 정정의 청크 재개·이후 프런티어·부모 계보 유지, `pageSize=1` 첫 페이지 이후 같은 `asOf` 이하로 늦게 생성된 물질화가 기존 cursor의 ID 상한을 넘어서 다음 페이지에 섞이지 않음, 요청 시 GET이 물질화 행을 추가하지 않음, 최소권한 런타임 역할의 트리거 실행을 포함한다.

또한 520개 초기 청크 처리 중 후속 append를 두 워커가 동시에 시도해 선행 프런티어 완료 전 claim이 거부되는지 확인했다. 인접 체크포인트가 없는 601개 구간 복구는 실제 재생한 청크를 동일 프런티어로 물질화하여 지표·값·시장 통계·현재 입력의 601단계 부모 계보를 보존하며, 인접 append·정정은 영향 범위만 물질화함을 검증했다.

WebSocket 회귀 검증은 준비 기간의 `null` 값과 `warming_up` 상태가 일치하는지 확인한다. 최신 append·마지막 교체는 최신 캔들과 시각이 같을 때만 단일 upsert하고, 워커 지연의 오래된 지표는 upsert하지 않으며, 과거 지점 변경만 `revisionRefresh` 전체 청크를 보내는 분류를 검증한다.
