## 레이아웃 회귀 RED·GREEN

GitHub run `29519912264`는 `tests/e2e/m1-operations.spec.ts:86`에서 종료 시각과 count field y 좌표 차이가 기대 1px 이하, 실제 14px로 실패했다. 원인은 Linux 한글 대체 font에서 폭 8rem count label이 두 줄로 wrapping되지만 input baseline을 grid가 예약하지 않은 것이다.

- RED 1: E2E에서 count label 폭을 6rem으로 강제한 뒤 같은 test 실행, y 차이 15px로 실패
- 코드 리뷰: 고정 `min-height: 2.9em`은 WCAG text spacing과 세 줄 wrap을 보장하지 못하므로 폐기
- RED 2: 생산 CSS에 서브그리드(subgrid)가 있고 magic height가 없는지 정적 테스트 추가 후 실패
- GREEN: `to|count` field와 label이 5개 공유 grid row를 사용하고 680px 이하에서 단일 열로 복귀
- 브라우저 회귀: 681px에서 3줄 상당 header·`line-height: 1.5`·letter/word spacing에도 input y 차이 1px 이하, 680px에서 같은 폭의 수직 배치 확인
- 전체 회귀: Playwright 16 passed, Web 136 passed, build 성공

## 배포 실패 폐쇄 검증

- 자동 `release` push trigger를 제거하고 수동 `workflow_dispatch`에 승인 40자리 SHA를 요구한다.
- checkout·setup-python·setup-node·setup-uv는 tag가 아닌 실제 commit SHA allowlist로 검증한다. setup-uv annotated tag object는 peeled commit `37802adc94f370d6bfd71619e3f0bf239e1f3b78`로 정정했고, 첫 원격 CI의 Node.js 20 사용 중단 경고 뒤 checkout은 공식 v5 commit `93cb6efe18208431cddfb8368fd83d5badbf9bfd`로 갱신했다.
- P8 exact-SHA 잠금, 보호된 `main`·`release`, `prod` 승인자·브랜치 제한, 두 branch와 실행 SHA 동일성, CI `verify` 성공을 image build 전에 검사한다.
- branch protection 조회는 기본 `GITHUB_TOKEN`이 아니라 Administration read 권한의 `GOODMONEYING_DEPLOY_GITHUB_TOKEN` 환경 secret을 요구한다.
- 배포 중 migration을 후속 실행이 취소하지 않도록 `cancel-in-progress: false`를 검증한다.
- 실제 원격 `release` SHA로 enable 변수 없이 사전점검: exit 1, `P8 배포 잠금이 해제되지 않았습니다` 확인.
- profile 배포 script는 `release-` 뒤 정확히 40자리 commit SHA만 허용한다.
