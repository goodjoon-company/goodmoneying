# 시스템 트레이딩 플랫폼 P1 데이터 기반 검증

GitHub Issue: [#28](https://github.com/goodjoon-company/goodmoneying/issues/28)
검증 환경: macOS, Docker PostgreSQL 17.10, Python 3.14, Node.js 24, Chromium

## 적색(RED) 증적

- P1 계약 테스트는 최초 실행에서 시장·정책·coverage migration과 `/v1/data-foundation`이 없어 실패했다.
- 실제 PostgreSQL coverage 테스트는 초기 상태 수와 중첩 범위 처리에서 실패했고 nullable target을 `FOR UPDATE` 하던 쿼리 오류를 드러냈다.
- 시장 동기화 worker 계약은 compose·`dev.sh` 서비스가 없어 실패했다.
- Coverage & Quality 브라우저 테스트는 정책 편집 버튼과 시장별 설정이 없어 실패했다.
- 재시도 E2E는 첫 실패 뒤 기대 `retry_wait`, 실제 `failed`로 실패했다. 이후 같은 instance 중복 claim, 임대 회수 뒤 stale worker 쓰기, 부분 진행 유실, dead-letter와 coverage 불일치를 회귀 테스트로 추가했다.
- 전체 E2E의 과거 `저장 KST` 기대는 내부 UTC 계약과 충돌해 `저장 UTC`로 교체했다.

## 실제 PostgreSQL 검증 범위

- 모든 KRW 기본 정책과 신규 시장 자동 편입, 명시적 제외·일시정지 보존
- 시장 상태 이력 비중첩, 대상·job·coverage 멱등 제약
- 정책별 시작일·데이터 유형·보존·우선순위 저장과 동일 요청 무변경
- 정확한 구독 `(target_spec_id, generation)` 적용 완료
- 캔들·체결·호가·티커 실제 행의 market·사건·수신·저장·지식 시각과 checksum manifest
- 연결별 원본 수신 영수증, 동일 경제 상태 호가 스냅숏 중복 제거, 전체 30레벨과 10레벨 파생 요약의 원자 저장
- 같은 연결·프레임 순번의 동일 payload 멱등 처리와 다른 payload 충돌 rollback, 다른 연결의 동일 payload receipt 2행·snapshot 1행
- 원본 receipt·snapshot의 보존 기간 strict `<` 경계, 다중 정책 무기한 우선·최댓값, level cascade와 summary·manifest 보존
- 백필 페이지의 실제 endpoint·`to` cursor·query·요청/응답 시각·HTTP 상태·원본 JSON과 저장 batch 간 manifest 공유
- 응답 없는 전송 오류(Transport Error)의 실제 요청 cursor, 실패 감지 시각, `NULL` 상태·payload, 오류 유형·메시지 manifest
- 빈 응답의 무거래 합성 금지, 연속 캔들 batch coverage 압축
- 418 응답 차단 시간 보존, 차단 정보가 없으면 저장소 300초 기본값, 429는 1초 기본값, 재시도 대기 전 claim 금지, 예산 소진 dead-letter
- 살아 있는 lease 중복 claim 금지, 만료 회수 attempt 증가, stale owner 진행 쓰기 거부
- 부분 진행 시각·처리 구간·행 수 보존, 최종 작업 범위의 미분류 공백·`unverified → missing` 품질 이벤트와 다른 상태 보존
- 거래지원 중단·카탈로그 누락 뒤 `unavailable`, 재등장 뒤 열린 끝의 `unverified` 전이와 중복 없는 품질 이벤트

## 공식 업비트 기준

2026-07-17 최신 개발자센터의 요청 수 제한 문서를 다시 확인했다. Quotation candle은 IP 기준 초당 최대 10회이며 429는 다음 초 경계, 418은 응답에 안내된 차단 시간 이후 재시도한다. 반복 위반 시 차단 시간이 증가하므로 418 기간이 없을 때 즉시 재시도하지 않는다.

## 리뷰 결과

- 독립 리뷰에서 live lease 재선점, stale writer, 부분 진행 유실, job-target 불일치, 새 상태의 OpenAPI·UI 누락, 실패 coverage·품질 이벤트, 418 기간 전달을 발견해 반영했다.
- 과업별 독립 재리뷰에서 시장 상태 원인, 자동·수동 백필 경쟁, 페이지별 원본 증적, 네트워크 무응답 증적, 5상태 공통 전이, 작업 범위 한정 실패, 원본 호가 receipt/snapshot 분리를 검토했고 Critical·Important·Minor 0건으로 승인됐다.
- migration은 확장 전용이며 down 절에 table drop·row delete가 없다.
- SQLite는 운영 내구성 구현이 아니라 테스트 호환 어댑터임을 Architecture·ADR·History에 명시했다.

## 미실행과 안전 경계

- 실제 운영 DB의 2024년 이후 KRW 전체 백필은 P7 용량·백업·처리율 gate와 P8 승인 SHA 배포 전 실행하지 않았다.
- 실제 주문·출금·실거래 활성화는 실행하지 않았다.
- 원격 CI run과 최종 SHA는 push 뒤 이 문서의 후속 closeout에 기록한다.
