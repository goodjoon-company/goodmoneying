# 전체 표시 형식과 업비트 작업대 UI 검증

Date: 2026-07-17
Related Product: GM-PROD-035
Related Issue: [#26](https://github.com/goodjoon-company/goodmoneying/issues/26)
Related Design: `docs/refinements/requests/2026-07-17-전체-표시-형식과-업비트-작업대-UI-개선.md`

## 검증 범위

- 전체 사용자 UI와 차트의 KST `YYYY.MM.DD HH:mm:ss KST` 24시간 표시
- 화폐·자산 수량의 3자리 구분자, KRW 정수 표시, 최대 소수점 8자리와 후행 단위
- Quotation 공통 조회 기준의 마켓 선택·기준 자산·읽기 전용 거래쌍
- 전체 카탈로그 파라미터의 `한글 이름(parameter_name)` 레이블
- 조회용 `count` 최대 초기값과 공식 API 기본값의 별도 안내
- 데스크톱 `to`·`count` 한 행 배치, 390px 반응형 배치와 인라인 불리언 입력
- 원본 추적 JSON·게이트웨이 요청값·안전 등급 비변경

## 테스트 주도 개발(TDD) 증적

- 공통 표시 정책 모듈이 없는 상태에서 포맷터 단위 테스트가 실패하는 것을 확인한 뒤 KST·화폐·자산 포맷터를 구현했다.
- 차트 옵션 캡처 테스트에 KST 시간 포맷터와 호가 자산 가격 포맷터 기대값을 먼저 추가한 뒤 세 차트에 적용했다.
- 한글 파라미터 레이블, 읽기 전용 거래쌍과 조회용 최대 `count` 기대값을 컴포넌트 테스트에 먼저 추가했다.
- 통화 문맥이 빈 값인 경계 테스트는 `1,234.5 `의 불필요한 뒤 공백으로 실패하는 것을 확인한 뒤 단위가 없으면 숫자만 반환하도록 수정했다.
- 목록 파라미터의 빈 항목과 상호 배타 입력 조합이 통과하는 실패 테스트를 먼저 추가한 뒤 카탈로그 계약과 게이트웨이 실행 전 검증을 보완했다.
- 불리언 입력 레이블의 클릭 영역이 44px보다 작은 것을 Playwright로 재현한 뒤 체크박스 크기는 유지하면서 레이블 클릭 영역만 44px로 확대했다.

## E2E 시나리오

- Quotation과 Exchange에서 불리언 체크박스가 20px 이하이고 레이블 클릭 영역이 44px 이상이며 마우스로 조작되는지 확인했다.
- 공통 마켓 `KRW`와 기준 자산 `ETH`가 읽기 전용 `KRW-ETH` 거래쌍으로 파생되는지 확인했다.
- 분 캔들 조회용 `count`가 최대값 200으로 시작하고 `화면 초기 200 · API 기본 1` 안내가 표시되는지 확인했다.
- 데스크톱에서 `to`와 `count`가 같은 행에 있고 `count`가 더 좁으며, 390px에서 가로 넘침이 없는지 확인했다.
- 현재가·호가·Exchange 잔고·WebSocket 현재가가 `150,000,000 ￦`, `0.2 ETH`, `120,000 ￦`처럼 의미에 맞는 후행 단위를 사용하는지 확인했다.
- 불리언 입력이 Space 키로 토글되고 키보드 탭 이동과 원본 추적 초점 순환이 유지되는지 확인했다.
- raw 프레임의 표시 시각이 `2026.07.16 09:00:00 KST`이고 원본 JSON 문자열은 변환되지 않는지 확인했다.
- 실제 주문 기능의 게이트웨이·상향 호출은 기존 비파괴 정책대로 0건임을 확인했다.
- 상호 배타 주문 상태를 함께 입력하면 설명 가능한 오류를 표시하고 게이트웨이·상향 호출이 모두 0건인지 확인했다.

## 코드 리뷰 결과

- 요구사항과 GM-PROD-035, 승인 설계, 업비트 게이트웨이 설계, 카탈로그 계약과 ADR-0011의 비파괴 경계를 대조했다.
- 공통 표시 정책은 `apps/web/src/displayFormat.ts`에 집중되고 운영·시장·분석·Backfill·Quotation·Exchange·WebSocket 구조화 화면이 이를 소비한다.
- 원본 추적 JSON, DB·HTTP·WebSocket 계약과 `read`·`test`·`blocked` 안전 등급에는 변경이 없다.
- 독립 리뷰에서 지적된 목록 입력의 빈 항목, 주문 목록 식별자와 주문 상태의 상호 배타 조건, WebSocket 한글 레이블, 체결 UTC 날짜·시각의 KST 표시, 불리언 클릭 영역을 모두 보완했다.
- [주문 목록 조회](https://docs.upbit.com/kr/reference/list-orders-by-ids), [다수 주문 취소](https://docs.upbit.com/kr/reference/cancel-orders-by-ids), [대기 주문 조회](https://docs.upbit.com/kr/reference/list-open-orders), [종료 주문 조회](https://docs.upbit.com/kr/reference/list-closed-orders)의 최신 공식 조건과 일치하는지 확인했다.
- [개별 주문 조회](https://docs.upbit.com/kr/reference/get-order)와 [개별 주문 취소](https://docs.upbit.com/kr/reference/cancel-order)는 두 식별자를 함께 전달할 수 있고 UUID가 우선한다는 공식 동작을 유지했다.

## 런타임 확인

- `.venv`에서 `import jwt`와 PyJWT 2.13.0 로드를 확인했다.
- `./dev.sh status`에서 API, 웹, 업비트 게이트웨이, 세 워커가 모두 실행 중임을 확인했다.

## 알려진 경고

- Vite의 500 kB 초과 청크 경고는 기존 번들 분할 후속 과제이며 빌드를 실패시키지 않는다.
- Pytest의 Starlette `httpx` 지원 중단 예정 경고는 기존 의존성 경고다.
- Playwright 시험 서버 종료 중 WebSocket `ECONNRESET` 로그가 발생할 수 있으나 15개 테스트와 서버 종료 검증은 모두 통과했다.
