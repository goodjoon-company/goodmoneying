## 리뷰 보강 검증

- 같은 전략의 동시 버전 게시가 `MAX(version)+1` 경쟁 조건(race condition)을 만들지 않도록 전략 단위 advisory lock을 추가하고, live PostgreSQL 동시 게시 테스트로 `version=[1,2]`를 확인했다.
- `strategy_graphs.graph_hash`가 `strategy_versions.graph_hash`와 DB 레벨에서 불일치할 수 없도록 `(strategy_version_id, graph_hash)` 복합 외래키(Foreign Key)를 추가하고, live PostgreSQL에서 불일치 삽입이 `ForeignKeyViolation`으로 차단됨을 확인했다.
- OpenAPI의 `StrategyGraphPort.timeframe`은 Pydantic 모델과 서버 검증기처럼 선택 필드(optional field)로 정렬했다.
- API가 반환할 수 있는 전략 멱등 충돌·cursor 문맥 충돌 409 응답을 OpenAPI 경로에 명시하고 계약 테스트로 고정했다.
