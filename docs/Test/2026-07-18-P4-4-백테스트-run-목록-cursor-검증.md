## GREEN

### Python 계약·API·저장소

명령:

```bash
uv run pytest tests/shared/test_backtest_store.py tests/api/test_backtest_runs_api.py -q
```

결과:

```text
8 passed, 1 warning in 0.43s
```

확인:

- 첫 페이지 후 새 run이 삽입돼도 다음 페이지가 첫 페이지의 `ceiling` 이하에서 이어진다.
- 잘못된 cursor는 `BacktestCursorMismatchError`로 수렴한다.
- 공개 SHA-256 checksum을 재계산한 변조 cursor와 bool 정수 payload cursor를 거부한다.
- FastAPI는 cursor 문맥 불일치를 `BACKTEST_CURSOR_CONTEXT_MISMATCH` 409로 반환한다.
- 목록 API는 read-only이며 mutation count가 증가하지 않는다.

### Web API·UI

명령:

```bash
npm --workspace apps/web run test -- src/api.test.ts src/features/backtestLab/BacktestLab.test.tsx
```

결과:

```text
Test Files  2 passed (2)
Tests  16 passed (16)
```

확인:

- `loadBacktestRuns({ pageSize: 25, cursor })`가 `/api/v1/backtest-runs?pageSize=25&cursor=...`를 호출한다.
- Backtest Lab은 목록을 먼저 읽고 첫 run을 자동 선택한다.
- `nextCursor`가 있으면 “다음 run 목록 불러오기”로 다음 페이지를 누적 로드한다.
- 선택한 run 상세만 `loadBacktestRun()`으로 조회한다.
- 실행 생성 버튼은 노출하지 않는다.

## 코드 리뷰

- 1차 리뷰 Important: 공개 checksum cursor는 변조 방지가 아니고 Backtest Lab이 `nextCursor`를 소비하지 않았다.
- 조치: cursor digest를 HMAC-SHA-256으로 변경하고 공개 checksum 재계산 변조 테스트를 추가했다.
- 조치: Backtest Lab을 `useInfiniteQuery`로 전환하고 다음 페이지 누적 로드 테스트를 추가했다.
- re-review Important: bool payload 테스트가 HMAC 검증 전에 실패해 타입 검증 증적이 아니었다.
- 조치: 테스트에서 유효 HMAC으로 bool payload를 재서명해 `type(...) is not int` 검증을 실제로 통과하도록 수정했다.
