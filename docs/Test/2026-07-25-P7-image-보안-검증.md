# P7 image 보안 검증

- 일시: 2026-07-25 KST
- 대상 gate: `security.images`
- 명령: `npm run p7:image-security`

## 증적

- 런타임 Dockerfile: `apps/api/Dockerfile`, `apps/worker/Dockerfile`, `apps/upbit_gateway/Dockerfile`, `apps/web/Dockerfile`
- 모든 런타임 이미지에 비root `USER` 지시문을 선언했다.
- web 런타임 컨테이너는 비특권 포트 8080을 노출한다.
- web 이미지에서 `GOODMONEYING_OPERATOR_TOKEN` 기본 ENV를 제거했다.
- 운영 외부 URL은 유지하고 내부 컨테이너 포트만 `8080`으로 바꿨다.
- web 컨테이너 standalone 런타임 확인: `GOODMONEYING_API_INTERNAL_URL=http://127.0.0.1:1`, `GOODMONEYING_UPBIT_GATEWAY_INTERNAL_URL=http://127.0.0.1:1`로 upstream DNS 의존을 제거한 뒤 `uid=101(nginx) gid=101(nginx)`, `http://127.0.0.1:18080/` HTTP 200
- Nginx 비root 기동을 위해 `/run/nginx.pid`와 `/run` 소유권을 `nginx` 사용자에게 열었다.

## 결과

통과. 런타임 이미지의 root 실행과 이미지 내부 운영자 토큰 기본값을 제거했다.
