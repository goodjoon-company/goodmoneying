## 사용자 가치와 범위

개발자는 저장소에 수집된 데이터와 분리해 업비트 공개 시세 API의 특정 KRW 거래쌍을 즉시 조회하고, 분·일·주·월 봉의 OHLCV와 계산된 보조지표를 한 화면에서 검증한다.

- 화면은 `업비트 API 테스트` 개발용 메뉴로 제공한다.
- 거래쌍, 캔들 주기, 조회 개수(최대 200)를 입력해 명시적으로 조회한다.
- 차트에는 캔들·거래량·SMA 20·EMA 20·볼린저 밴드(20, 2)를 표시하고, 최신 RSI 14를 함께 표시한다.
- 호출 대상은 인증 없는 `https://api.upbit.com/v1/candles/*` 공개 REST API다.

## 비범위

- 내부 API, DB, 저장·집계 워커, 기존 분석 WebSocket 계약 변경
- 체결 이력, 주문, 투자 추천 또는 자동매매
- 사용자 정의 지표·수식·차트 레이아웃 저장

## 완료 기준(Acceptance Criteria)

1. 사용자는 `KRW-BTC` 등 거래쌍과 분·일·주·월 봉을 선택해 캔들을 조회할 수 있다.
2. 응답은 시간 오름차순으로 렌더링되고, 캔들·거래량·보조지표·최신 OHLCV 값이 표시된다.
3. 잘못된 거래쌍, HTTP 오류, 빈 응답을 명확한 오류 또는 빈 상태로 표시한다.
4. 업비트 요청 URL·응답 정규화·실패 처리는 단위 테스트로, 화면 조회는 Playwright 종단 간 테스트(E2E Test)로 검증한다.

## 설계·계약 판단

업비트 공개 API를 브라우저에서 읽기 전용으로 호출하는 테스트 도구다. 2026-07-14 응답의 `Access-Control-Allow-Origin: *`를 확인했다. 내부 API·DB·이벤트·WebSocket 계약과 모듈 책임은 변경하지 않으므로 계약 문서와 ADR 갱신은 필요 없다.

## 테스트와 QA 전략

- RED/GREEN: 분봉 URL 생성, 최신순 API 응답의 오름차순 정렬, HTTP 실패 처리
- UI: 조회 버튼으로 차트·보조지표·OHLCV 요약을 표시
- E2E: 업비트 네트워크 요청을 고정 응답으로 가로채 분봉 선택과 결과 렌더링을 검증
- QA: 390px 및 데스크톱 viewport에서 제어 영역과 차트 컨테이너를 확인
