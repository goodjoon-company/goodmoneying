## 사용자 가치와 범위

개발자는 저장소에 수집된 데이터와 분리해 업비트 공개 시세 API를 즉시 요청하고, 거래쌍 목록과 특정 거래쌍의 분·일·주·월 봉을 한 화면에서 검증한다. 캔들 차트를 이동하거나 축소해 현재 적재 범위의 빈 영역에 가까워지면 필요한 페이지를 API에서 추가로 읽어 같은 차트에 이어 그린다.

- 화면 상단은 API별 요청 조건을 설정하고 실행하는 패널, 중앙은 현재 요청 결과를 다양한 형식으로 표시하는 영역으로 나눈다.
- 요청 패널은 `거래쌍 목록`과 `캔들` 탭을 제공한다.
- 거래쌍 목록은 `GET https://api.upbit.com/v1/market/all`을 호출하며, 유일한 선택 옵션인 상세 정보 포함 여부(`is_details`)를 체크박스로 제공한다.
- 캔들 요청은 목록 API 응답에서 선택한 거래쌍, 캔들 주기, 조회 개수(최대 200), 종료 시각(`to`)을 사용한다. 일봉에는 원화 환산(`converting_price_unit=KRW`) 선택을 추가한다.
- 거래쌍 목록을 아직 읽지 않았으면 거래쌍 선택기와 캔들 요청을 비활성화하고, 목록 조회 성공 후 기존 텍스트 입력을 대체하는 선택기를 채운다.
- 거래쌍 결과는 검색 가능한 표와 원본 JSON으로, 캔들 결과는 차트·거래량·보조지표·OHLCV 표·원본 JSON으로 표시한다.
- 차트에는 캔들·거래량·SMA 20·EMA 20·볼린저 밴드(20, 2)를 표시하고, 최신 RSI 14를 함께 표시한다.
- 호출 대상은 인증 없는 업비트 공개 REST API(`https://api.upbit.com/v1/market/all`, `https://api.upbit.com/v1/candles/*`)다.

## 비범위

- 내부 API, DB, 저장·집계 워커, 기존 분석 WebSocket 계약 변경
- 체결 이력, 현재가, 호가, 주문, 투자 추천 또는 자동매매
- 사용자 정의 지표·수식·차트 레이아웃 저장
- 내부 데이터베이스에 업비트 API 응답을 저장하거나 내부 API로 중계하는 동작

## 완료 기준(Acceptance Criteria)

1. 사용자는 거래쌍 목록 API를 조회하고 `is_details` 옵션을 선택할 수 있다.
2. 거래쌍 목록 조회가 성공하면 그 응답으로 채워진 선택기에서 `KRW-BTC` 등 거래쌍을 골라 캔들을 요청할 수 있다.
3. 캔들 요청은 분·일·주·월 봉과 `to`, 일봉 원화 환산 옵션을 지원하며, API가 지원하지 않는 옵션은 요청에 포함하지 않는다.
4. 거래쌍 응답은 표와 원본 JSON으로, 캔들 응답은 시간 오름차순 차트·거래량·보조지표·최신 OHLCV·표·원본 JSON으로 표시한다.
5. 사용자가 차트를 좌우로 이동하거나 확대·축소해 현재 적재 범위의 빈 영역 가까이 가면 해당 방향의 캔들을 최대 200개씩 추가로 요청하고, 시각 키로 중복을 제거해 차트에 병합한다.
6. 추가 요청은 방향별로 동시에 하나만 수행하고, 빈 응답은 해당 방향의 끝으로 기록한다. 추가 요청 실패는 이미 표시한 캔들을 지우지 않고 차트 영역에 오류를 표시한다.
7. 잘못된 요청 조건, HTTP 오류, 비정상 응답, 최초 빈 응답을 명확한 오류 또는 빈 상태로 표시한다.
8. 업비트 요청 URL·응답 정규화·페이지 병합·방향별 종료 처리는 단위 테스트로, 목록 선택→캔들 조회→차트 추가 요청은 Playwright 종단 간 테스트(E2E Test)로 검증한다.

## 설계·계약 판단

업비트 공개 API를 브라우저에서 읽기 전용으로 호출하는 테스트 도구다. 내부 API·DB·이벤트·WebSocket 계약과 모듈 책임은 변경하지 않으므로 계약 문서와 ADR 갱신은 필요 없다.

업비트 공식 문서(2026-07-16 확인)에 따르면 거래쌍 목록 API의 선택 쿼리는 `is_details`이고, 캔들 API는 `to` 이전 최대 200개를 반환한다. 브라우저의 교차 출처(Cross-Origin) 요청에는 Origin 헤더 정책상 시세 REST API 전체가 **10초당 1회**로 제한된다. 따라서 목록·초기 캔들·추가 페이지를 포함한 모든 요청은 전역 10초 간격으로 예약하고, 대기 중인 요청은 화면에 남은 시간을 표시한다. 차트는 가시 논리 범위(Logical Range)가 적재 데이터의 양 끝에 가까워질 때만 방향별 단일 요청을 예약한다. 빠른 차트 이벤트는 짧게 묶고, 진행 중 요청·이미 끝난 방향·동일 커서는 다시 요청하지 않는다. 체결이 없던 시간에는 API도 캔들을 만들지 않으므로 시간 간격을 누락으로 채우지 않는다.

초기 요청에 `to`가 없다면 오른쪽 끝은 최신 API 응답이므로 과거 방향을 우선 보충한다. `to`를 지정한 경우에는 가장 최신 캔들 이후의 다음 페이지 종료 시각을 계산해 앞으로도 보충한다. 어떤 페이지든 시각 키로 합치고 시간 오름차순으로 다시 정렬한다. 추가 요청 실패는 기존 결과 상태와 분리해 표시한다.

## 테스트와 QA 전략

- RED/GREEN: 거래쌍 `is_details` URL·응답 정규화, 캔들 `to`·원화 환산 URL, 최신순 응답의 오름차순 정렬, 페이지 병합·중복 제거·방향별 종료, HTTP 실패 처리를 먼저 검증한다.
- UI: 거래쌍 조회 후 선택기 활성화, 옵션별 표·원본 JSON, 캔들 차트·보조지표·OHLCV 요약, 추가 조회 진행·오류 상태를 표시한다.
- E2E: 업비트 네트워크 요청을 고정 응답으로 가로채 목록 조회→거래쌍 선택→캔들 조회→차트 이동 뒤 추가 페이지 요청과 병합 렌더링을 검증한다.
- QA: 390px 및 데스크톱 viewport에서 탭·요청 제어 영역·결과 표·차트 컨테이너와 가로 넘침을 확인한다.
