## 사용자 가치와 범위

1분 원천봉(Source Candle)은 재처리 가능한 단일 기준(source of truth)으로 보존한다. 시스템은 활성 관심 코인에 대해 5분·10분·30분·1시간·1일·1주·1개월 집계 봉(Materialized Rollup)을 별도 테이블에 유지한다. 분석 화면은 집계가 최신일 때 집계 봉을 우선 조회해 반복 집계 비용을 없앤다.

새 `시스템 관리` 화면은 다음을 실시간으로 표시한다.

- Backfill 수집 워커의 생존·실행 상태와 현재 대상 코인·`source_candle` 데이터 유형
- 실시간 수집 워커의 생존·실행 상태와 현재 활성 코인·`ticker_snapshot`, `orderbook_summary`, `source_candle`, `trade_event` 데이터 유형
- 집계 워커의 생존·실행 상태, 현재 집계 코인·캔들 단위, 전체 집계 대상 수·완료 수·진행률

## 기능 설계

- 집계 워커(Aggregation Worker)는 활성 수집 대상의 원천봉 최신 시각과 집계 워터마크(Watermark)를 비교한다.
- 뒤처진 단위가 있으면 자동 집계 작업을 만들거나 기존 작업을 재개한다. 수동 전체 재계산은 이번 범위에 넣지 않는다.
- 작업은 코인·단위별 대상(Target)으로 나뉘며, 완료·실행·대기·실패 상태와 집계한 봉 수를 기록한다.
- 5분·10분·30분·1시간·1일은 1분 원천봉에서, 1주·1개월은 완성된 일봉 집계에서 만든다. 새 1분봉 또는 Backfill 저장 뒤의 보정은 다음 집계 작업에서 멱등 upsert로 반영한다.
- 분석 조회는 최신 집계가 있으면 사용한다. 집계가 진행 중이거나 누락됐을 때만 기존 원천봉 즉시 파생을 안전한 보조 경로로 쓴다.
- 시스템 관리 화면은 기존 운영 대시보드의 실시간 상태 스트림을 재사용하되, 집계 상태는 별도 뷰 모델로 표시한다. 기존 운영·Backfill·관심목록·코인 분석 화면은 유지한다.

## 비범위

- 사용자 정의 집계 단위, 임의 SQL 실행, 집계 작업 수동 취소·재시작 버튼
- 호가 원천 스냅샷의 집계·재생
- 주식·뉴스·전략 데이터 집계

## 완료 기준(Acceptance Criteria)

1. 1분 원천봉이 있는 활성 코인은 지원 단위별 집계 봉을 DB에서 조회할 수 있다.
2. 원천봉 워터마크보다 오래된 집계는 자동 작업으로 전환되고, 작업은 재실행해도 중복 봉을 만들지 않는다.
3. 시스템 관리 화면은 실시간·Backfill·집계 워커마다 상태, 현재 코인, 데이터 유형 또는 집계 단위, 최신 시각을 표시한다.
4. 집계 작업이 실행 중이면 전체 대상 수, 완료·실행·대기·실패 수, 백분율 진행률을 표시한다.
5. API·DB 계약, SQLite·PostgreSQL 구현, 개발·배포 워커 명령, React 화면, 자동 단위·통합·E2E 검증이 함께 갱신된다.

## 테스트와 QA 전략

- RED/GREEN: 집계 봉 upsert·재실행 멱등성, 오래된 워터마크의 자동 작업 생성, 작업 진행률, 분석 조회의 집계 우선·원천 보조 경로
- 통합: 시스템 관리 API가 현재 worker와 작업 대상을 반환하는지 확인
- E2E: 시스템 관리 메뉴에서 세 worker와 집계 진행률을 확인하고, 기존 분석 화면을 다시 연다.
- QA: 데스크톱·모바일에서 긴 코인명·다수 단위·진행률 0/100% 상태를 확인한다.
