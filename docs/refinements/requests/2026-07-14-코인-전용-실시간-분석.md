# 코인 전용 관심목록 실시간 분석 상세화

Date: 2026-07-14
Source Request: 사용자 요청
Related Product: `docs/01_Product.md`의 P1, P2 및 GM-PROD-025~027
Related Issue: [#12](https://github.com/goodjoon-company/goodmoneying/issues/12)

## 사용자 가치

사용자는 전체 업비트 KRW 코인 중 자신이 계속 살펴볼 코인을 먼저 정하고, 선택한 한 코인의 가격 흐름과 실시간 시장 상태를 한 화면에서 읽는다. 이 화면은 매수·매도를 추천하지 않으며, 저장된 시장 데이터를 해석하기 위한 분석 작업대다.

## 핵심 시나리오

1. 사용자는 관심종목 화면에서 수집 후보군의 코인을 관심목록에 넣는다.
2. 사용자는 새 `코인 분석` 메뉴를 열어 관심 코인을 고른다.
3. 첫 진입에서 최근 1년 일봉, 거래량, SMA·EMA·볼린저 밴드·RSI, 현재가와 호가 요약을 확인한다.
4. 사용자는 월·주·일·시·30분·10분·5분·1분 봉과 기간을 바꾸고, 최대 3년 범위에서 저장된 차트를 읽는다.
5. WebSocket이 재연결되면 같은 관심 코인·시간 단위·기간을 다시 구독하고 현재가, 호가, 체결 요약과 새 캔들을 각각 갱신한다.

## 기능 설계

- 분석 구독은 관심목록의 거래 상품만 허용한다. 목록 밖 대상은 명시적 오류 메시지를 받는다.
- 최초 구독은 `analysis.session`, `analysis.instrument`, `analysis.chart`, `analysis.indicators`, `analysis.market`의 작은 메시지 묶음으로 응답한다. 차트와 지표는 500개 이하 청크(chunk)로 분할하고, 새 봉 지표는 `analysis.indicator.upsert` 한 건으로 보정한다.
- 이후 갱신은 새 또는 보정된 단일 캔들, 현재가·호가 요약, 최근 체결 요약, 다시 계산된 지표로 분리한다.
- 원천 1분봉은 5분·10분·30분·시봉으로, 원천 일봉은 주봉·월봉으로 서버에서 파생한다. 1분·5분·10분·30분·시봉의 장기 범위는 최근 1,000개 캔들만 반환하며, 각 메시지는 500개 이하로 분할하고 화면에 제한 사유를 표시한다.
- 기술 지표는 서버가 계산한 고정 파라미터 SMA 20/60, EMA 20, 볼린저 밴드 20·2, RSI 14만 제공한다. 사용자 정의 수식, 추천 신호, 주문 기능은 제공하지 않는다.

## 비범위

- 주식 데이터·화면·주식 관심목록
- 원천 호가 전체 깊이 저장, 과거 호가 재생
- 투자 추천, 자동 매매, 사용자 정의 지표
- 기존 SSE 운영 화면 경로의 제거 또는 호환성 변경

## Acceptance Criteria

- 새 메뉴에서 관심 코인만 선택할 수 있고, 관심목록이 비어 있으면 관심종목 화면으로 가는 명확한 안내를 표시한다.
- 기본 1년 일봉과 거래량이 같은 차트 영역에서 표시된다.
- 시간 단위와 기간을 바꾸면 기존 차트를 덮어쓴 뒤 새 차트를 별도 WebSocket 메시지로 반영한다.
- 현재가, 호가 요약, 체결 요약, 지표가 독립 메시지로 렌더링되며 한 메시지에 전체 화면 상태를 반복하지 않는다.
- 데스크톱과 375px, 768px, 1440px 화면에서 가로 스크롤·텍스트 넘침 없이 사용 가능하다.
- 기존 운영 상태, Backfill 관리, 관심종목, 코인 상세 E2E가 유지된다.

## 테스트·QA 전략

- API 통합 테스트: 관심목록 검증, 월·주 파생 캔들, WebSocket 초기 메시지와 구독 변경, 메시지 스키마를 검증한다.
- 프론트엔드 테스트: 메뉴 진입, 기본 1년 일봉, 시간 단위 제어, WebSocket 메시지별 상태 병합을 검증한다.
- Playwright E2E: 관심목록에서 분석 메뉴로 이동해 차트·거래량·지표·호가·체결 화면과 WebSocket 갱신을 확인한다.
- 디자인 리뷰: 분석 화면의 정보 위계, 반응형, 접근성, 텍스트·숫자 밀도를 확인한다.

## 결정 기록

- 사용자 요청으로 코인만 현재 범위로 고정한다. 주식은 기존 장기 후보 기록만 유지하며 구현하지 않는다.
- 전체 화면 상태를 반복 전송하지 않고 메시지를 기능별로 분리한다.
- WebSocket은 분석 화면의 브라우저 전송에만 추가한다. 수집 워커의 업비트 WebSocket과 기존 운영 화면 SSE는 유지한다.
