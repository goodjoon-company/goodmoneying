## 목표와 사용자 가치

사용자는 goodmoneying의 어느 화면에서도 날짜·시간·화폐·자산 수량을 같은 순서와 단위로 읽는다. 업비트 API 작업대에서는 마켓과 기준 자산을 선택해 유효한 거래쌍을 만들고, 한글 파라미터 이름·적절한 입력 크기·일관된 체크박스로 요청을 빠르게 구성한다.

## 승인된 결정

1. 적용 범위는 운영 상태, 시장 목록, 관심 코인 분석, Backfill, 업비트 REST·Exchange·WebSocket 작업대를 포함한 전체 사용자 UI다.
2. 모든 사용자용 날짜·시간은 KST, 24시간제, `YYYY.MM.DD HH:mm:ss KST` 형식으로 표시한다. 날짜만 표시하는 경우 `YYYY.MM.DD`를 사용한다.
3. 원본 API 추적 JSON, 요청값, 응답 원문과 저장 계약은 변환하지 않는다.
4. 금액과 자산 수량은 항상 3자리 구분자(Thousands Separator)를 사용한다.
5. KRW는 소수점을 표시하지 않는다. 그 밖의 금액과 자산 수량은 최대 소수점 8자리까지 표시하고 불필요한 뒤쪽 `0`을 제거한다.
6. 단위는 값 뒤에 한 칸 띄워 표시한다: `1,234 ￦`, `1,234.56 $`, `0.12345678 ₿`, `1,234.5678 ETH`.
7. KRW·USDT·BTC는 각각 `￦`·`$`·`₿`를 사용하고 그 밖의 자산은 통화 코드(Currency Code)를 사용한다.
8. 공통 조회 기준은 마켓 선택 → 기준 자산 입력 → 읽기 전용 거래쌍 순서다.
9. 조회용 `count`만 기능별 최대값으로 화면 초기화한다. 일괄 취소 같은 상태 변경 기능의 `count`는 공식 기본값을 유지한다.
10. 모든 현재 카탈로그 파라미터는 `한글 이름(parameter_name)` 형식으로 표시한다.

## 표시 아키텍처

### 공통 표시 정책 모듈

프런트엔드 공통 표시 모듈은 다음 순수 함수(Pure Function)를 제공한다.

- `formatKstDateTime(value)`: KST `YYYY.MM.DD HH:mm:ss KST`
- `formatKstDate(value)`: KST `YYYY.MM.DD`
- `formatMoney(value, currency)`: 화폐 소수점·구분자·단위 정책 적용
- `formatAssetAmount(value, asset)`: 최대 8자리·구분자·자산 코드 후행 표시
- `formatNumber(value, options)`: 화폐가 아닌 일반 숫자의 구분자와 명시된 정밀도 적용

기존 `operationsDisplay.ts`와 컴포넌트별 `toLocaleString`·`Intl.DateTimeFormat` 중 사용자용 날짜·화폐 표시는 공통 모듈을 사용하도록 이동한다. 운영 집계·비율·저장량처럼 별도 의미를 가진 포맷은 기존 도메인 표시 모듈에 유지한다.

### 표시 문맥 판정

- 가격·거래금액 필드는 페어의 호가 자산(Quote Asset)을 화폐 단위로 사용한다.
- 잔고·수량·거래량 필드는 응답의 `currency` 또는 페어의 기준 자산(Base Asset)을 사용한다.
- 통화 문맥을 확인할 수 없는 일반 숫자는 구분자만 적용하고 단위를 추정하지 않는다.
- 유효하지 않은 날짜·숫자는 `NaN`이나 잘못된 단위를 만들지 않고 화면 성격에 따라 `-` 또는 기존 문자열을 표시한다.
- 원본 추적 대화상자의 JSON은 표시 포맷터를 거치지 않는다.

### 차트

`UpbitCandleChart`는 선택한 호가 자산을 입력으로 받는다. Lightweight Charts의 `localization.timeFormatter`, 시간 축 `tickMarkFormatter`, 캔들·지표 가격 포맷터를 공통 표시 정책에 연결한다.

- 시간 축과 십자선 시간 레이블: `YYYY.MM.DD HH:mm:ss KST`
- 가격 축과 십자선 가격 레이블: 호가 자산 기준 화폐 형식
- 캔들·거래량의 원본 숫자와 페이지네이션 시간은 변경하지 않는다.

## 업비트 작업대 UI 설계

### 공통 조회 기준

1. 마켓: `KRW`, `BTC`, `USDT` 선택 목록
2. 기준 자산(Base): 현재 텍스트 입력을 유지하고 대문자로 정규화
3. 거래쌍: `{마켓}-{기준 자산}` 읽기 전용 자동 표시

마켓 또는 기준 자산이 바뀌면 하나의 `WorkbenchContext`에서 거래쌍을 다시 계산한다. 사용자는 거래쌍을 직접 수정할 수 없으므로 세 값이 불일치하는 중간 상태를 만들 수 없다.

### 파라미터 레이블

공통 파라미터 표현 모듈은 현재 REST·Exchange·WebSocket 카탈로그 파라미터 전체에 한글 이름을 제공한다. 화면은 `상세 정보 포함(is_details)`, `조회 종료 시각(to)`, `조회 개수(count)`, `페이지 크기(limit)`, `시작 시각(start_time)`처럼 표시한다.

- 카탈로그의 영문 `name`은 요청 직렬화와 추적성을 위해 괄호 안에 보존한다.
- 새 카탈로그 파라미터에 한글 이름이 없으면 계약 테스트가 실패해 누락을 조기에 발견한다.
- API 원본 테이블의 응답 필드명은 원본 추적 의미를 보존하며 이번 레이블 정책의 대상이 아니다.

### 입력 배치와 초기값

- 데스크톱에서 날짜·시간 `to`와 조회용 `count`를 한 행에 배치한다.
- `to` 입력은 약 `22rem`, `count` 입력은 약 `8rem`으로 제한하고 남는 공간을 불필요하게 차지하지 않는다.
- 390px 이하에서는 두 필드를 한 열로 쌓고 최소 터치 높이를 유지한다.
- 조회용 `count`는 최대값으로 화면 초기화한다: 캔들 200, 체결 500, REST 호가 30.
- 도움말은 `화면 초기 200 · API 기본 1 · 최소 1 · 최대 200`처럼 화면 초기값과 공식 API 기본값을 구분한다.
- 조회가 아닌 `count`는 카탈로그의 공식 기본값을 그대로 사용한다.

### 불리언 입력

불리언(Boolean) 파라미터는 큰 독립 입력 박스를 사용하지 않는다. 한글·영문 레이블 바로 옆에 약 18px 체크박스를 배치하며 전체 레이블 클릭과 키보드 Space로 토글할 수 있다. 필수·선택 안내는 같은 행 끝의 작은 보조 텍스트로 유지한다.

## 오류 처리와 접근성

- 읽기 전용 거래쌍은 `readOnly`와 명확한 레이블로 보조 기술(Assistive Technology)에 노출한다.
- 한글 레이블과 원본 파라미터 이름을 하나의 접근 가능한 이름(Accessible Name)으로 제공한다.
- 체크박스는 시각 크기를 줄여도 최소 클릭 영역과 포커스 표시를 유지한다.
- 시간·화폐 포맷 실패가 요청 실행이나 화면 전체 렌더링 실패로 확산되지 않게 안전한 대체값을 사용한다.
- 원본 데이터와 표시값이 다를 때 원본 추적 대화상자에서 실제 값을 확인할 수 있다.

- 원본 JSON의 키·숫자·날짜 문자열 변환
- 외부 날짜 선택기·폼·국제화(i18n) 라이브러리 도입
- KRW·BTC·USDT 이외 자산의 비공식 기호 추정
- 상태 변경 API의 안전 등급 또는 상향 실행 정책 변경

## 인수 조건(Acceptance Criteria)

1. 전체 사용자 UI의 날짜·시간은 KST `YYYY.MM.DD HH:mm:ss KST` 24시간 형식이며 날짜만 있는 곳은 `YYYY.MM.DD`다.
2. 차트 시간 축과 십자선 레이블도 같은 날짜 순서·시간대·24시간 형식을 사용한다.
3. 모든 화폐 금액은 3자리 구분자를 가지며 KRW는 소수점이 없다.
4. KRW·USDT·BTC 금액은 값 뒤에 각각 `￦`·`$`·`₿`를 한 칸 띄워 표시한다.
5. 그 밖의 금액·자산 수량은 최대 소수점 8자리와 뒤쪽 통화 코드를 사용한다.
6. 마켓은 `KRW/BTC/USDT` 목록에서 선택하고 기준 자산 입력 뒤에 읽기 전용 거래쌍이 표시된다.
7. 데스크톱에서 `to`와 조회용 `count`가 한 행에 있으며 필드 폭이 내용에 맞고, 390px에서는 겹침 없이 세로로 배치된다.
8. 조회용 `count`는 최대값으로 시작하고 공식 API 기본값은 별도 안내된다. 상태 변경 기능의 `count`는 공식 기본값을 유지한다.
9. 현재 카탈로그의 모든 파라미터는 한글 이름과 원본 영문 이름을 함께 표시한다.
10. `is_details`를 포함한 불리언 입력은 텍스트 옆 체크박스로 표시되고 마우스·키보드로 조작할 수 있다.
11. 원본 API 추적 JSON과 게이트웨이 요청값은 표시 정책 때문에 변경되지 않는다.
12. 데스크톱·390px 브라우저 E2E와 전체 프런트엔드 회귀 테스트·빌드가 통과한다.
