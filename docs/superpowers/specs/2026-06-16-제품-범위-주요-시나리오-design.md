## 배경

goodmoneying은 주식과 코인 투자 자동화를 목표로 하는 개인용 투자 데이터 플랫폼이다. 사용자 요구는 단순 전략 구현보다 넓다. 장기간의 과거 데이터(Back Data), 실시간 호가(Orderbook), 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시를 수집하고, 대규모 언어 모델(LLM, Large Language Model) 분석 결과까지 전략 입력으로 활용할 수 있어야 한다.

## 결정한 방향

제품 전체 범위는 시장 데이터 기반, LLM 분석 신호, 전략 설계, 봇(Bot) 설계, 봇 동작 시뮬레이션(Simulation), 모의매매(Paper Trading), 실제 자동매매(Live Trading)까지 포함한다.

1차 최소 기능 제품(MVP, Minimum Viable Product)은 데이터 수집, 저장, 조회, 데이터 품질(Data Quality) 확인, LLM 분석 신호 생성까지로 제한한다. 자동매매와 모의매매는 초기 범위에서 제외한다.

## 접근안 비교

### 선택안: 통제된 카테고리 사전과 LLM 신호 추출

- 매출증대, 원가절감, 경쟁력상승, 공급계약, 비용증가, 규제리스크, 실적악화, 수요감소 같은 주가 영향 카테고리(Category)를 제품이 관리한다.
- LLM은 뉴스, 공시, 리포트에서 카테고리, 긍정/부정 방향, 강도(Strength), 신뢰도(Confidence), 기준 시점, 관련 종목을 추출한다.
- 신호는 가격 차트의 시계열(Time Series)에 정렬되어 이후 가격, 수익률, 변동성, 거래량과 비교 분석된다.
- 전략은 유사 카테고리의 신호와 강도를 입력값으로 사용할 수 있다.

이 방식을 선택한 이유는 LLM 결과를 사람이 읽는 요약에 그치지 않고, 검증 가능한 전략 입력 계약(Contract)에 가깝게 다룰 수 있기 때문이다.

### 대안: 자유 태그와 임베딩 중심

자유 태그(Tag)와 임베딩(Embedding)은 새로운 패턴 발견에 유리하지만, 카테고리가 흔들리면 전략 조건으로 재사용하기 어렵다. 초기 제품에는 재현성(Reproducibility)이 더 중요하므로 선택하지 않았다.

### 대안: 정량 데이터 우선, LLM은 보조 설명

가격, 거래량, 수급, 공매도, 재무지표를 전략 입력의 중심으로 두고 LLM을 요약에만 사용하는 방식은 안정적이다. 그러나 뉴스, 공시, 리포트의 긍정/부정 신호와 강도를 전략에 반영한다는 요구를 충족하지 못해 선택하지 않았다.

## 1차 MVP 범위

- 국내 주식, 해외 주식, 업비트(Upbit) 코인 수집 대상을 구성한다.
- 업비트 코인은 분석 실행 시점 기준 거래소 내 시가총액(Market Cap) 상위 50개로 제한한다.
- 가격/거래량, 시가총액, 수급, 공매도, 재무지표, 리포트, 뉴스, 공시, 호가(Orderbook)를 주기적으로 수집한다.
- 원천 데이터와 전략용 요약 데이터를 저장하고 조회할 수 있게 한다.
- 결측, 중복, 지연, 원천 신뢰도 같은 데이터 품질(Data Quality)을 확인할 수 있게 한다.
- 리포트, 뉴스, 공시를 LLM으로 분석해 사람이 읽는 요약과 구조화 신호(Signal)를 저장한다.
- 가격, 거래량, 수급, 공매도, 호가 요약, LLM 신호를 같은 시계열(Time Series) 기준으로 정렬한다.

## 전체 제품 시나리오

1. 시장 스캐닝(Scanning): 전체 시장에서 조건에 맞는 후보 종목과 코인을 발굴한다.
2. 관심 대상 심층 분석: 특정 종목이나 코인의 장기 시계열, 수급, 공매도, 재무지표, 호가, LLM 신호를 함께 분석한다.
3. 신호와 가격 영향 분석: 특정 카테고리의 긍정/부정 신호와 강도가 이후 가격, 수익률, 변동성, 거래량에 미친 영향을 분석한다.
4. 전략 설계와 백테스트(Backtest): 구조화 데이터와 LLM 신호를 조합해 전략을 만들고 과거 데이터로 검증한다.
5. 봇 설계: 봇별 투자 가능 금액, 시장, 종목/코인, 리스크 한도, 실행 주기, 주문 방식, 전략 가중치(Weight)를 설정한다.
6. 전략 파이프라인(Pipeline): 여러 전략을 순차 또는 병렬로 조합해 최종 매수/매도/보류 판단을 만든다.
7. 봇 동작 시뮬레이션(Simulation): 실제 주문 없이 봇의 판단, 포지션(Position), 주문 후보, 체결 가정, 손익, 판단 근거를 검증한다.
8. 모의매매(Paper Trading)와 실거래(Live Trading): 검증된 봇을 단계적으로 운영 환경에 연결한다.

## 정책과 제약

- MVP는 데이터 기반 구축에 집중하며 실제 주문 실행은 제외한다.
- 호가(Orderbook) 데이터가 없는 과거 구간에서는 호가 기반 전략 검증을 완전하게 지원하지 않는다.
- LLM 분석 결과는 투자 판단의 보조 입력이며 사실 데이터와 동일한 확정 정보로 취급하지 않는다.
- LLM 신호 카테고리(Category)는 버전 관리해야 한다.
- 원천 데이터는 재처리와 감사(Audit)를 위해 가능한 한 보존한다.
- 제품은 개인용 단일 사용자 플랫폼으로 시작한다.
- 투자 수익 보장이나 법적 의미의 매수/매도 추천은 제품 목표가 아니다.
