# 코인 분석·시스템 관리 빠른 안내

이 문서는 관심 코인을 고른 뒤 실시간 차트 분석과 시스템 관리 화면에 접속하는 빠른 절차다. 전체 설치·실행·장애 확인 기준은 [개발·운영 사용 안내](docs/사용설명서-M1-업비트-수집-운영-mvp.md)를 따른다. 현재 범위는 업비트 KRW 코인만이며, 주식 분석이나 주문 기능은 제공하지 않는다.

## 1. 최초 준비

```bash
uv sync
npm install
cp .env.sample .env
```

`.env`에서 `GOODMONEYING_LIVE_UPBIT=1`을 유지한다. 이 값이 없으면 실제 업비트 수집 워커가 캔들을 요청하지 않는다.

## 2. 실행과 접속

```bash
./dev.sh infra start
./dev.sh app start api
./dev.sh app start web
./dev.sh app start realtime-collection-worker
./dev.sh app start backfill-collection-worker
./dev.sh app start candle-aggregation-worker
./dev.sh status
```

브라우저에서 [http://127.0.0.1:5173](http://127.0.0.1:5173)을 연다. API 상태는 [http://127.0.0.1:8000/health](http://127.0.0.1:8000/health)에서 확인한다.

## 3. 관심 코인과 차트 데이터 준비

1. `Backfill 관리`에서 전체 업비트 KRW 후보 중 분석할 코인을 관심목록에 저장한다.
2. 같은 화면에서 필요한 과거 기간으로 백필(Backfill) 계획을 만들고 `백필 시작`을 누른다. 1년 일봉을 보려면 최소 1년 범위를 수집한다.
3. `코인 분석` 메뉴에서 관심 코인을 고른다. 기본값은 1년 일봉이다.

일봉 원천 데이터가 없더라도 저장된 1분봉이 있으면 서버가 일봉을 파생한다. 저장된 기간보다 긴 기간을 선택하면 차트는 저장된 구간만 표시하고, 1분·5분·10분·30분·시봉은 가장 최근 1,000개 캔들로 제한된다.

## 4. 시스템 관리와 집계 테이블

`시스템 관리` 메뉴는 별도 WebSocket으로 다음 상태를 1초 단위로 갱신한다.

- 실시간 수집: 활성 관심 코인과 `원천 캔들·시세·호가` 수집 유형
- Backfill 수집: 현재 실행 또는 대기 중인 코인과 `원천 캔들` 수집 유형
- 자동 집계 테이블: 1분 원천봉에서 생성하는 `5분·10분·30분·시·일·주·월` 단위별 대상, 저장 행 수, 전체 진행률

집계 워커는 활성 코인의 원천봉보다 집계 테이블이 오래된 것을 발견하면 작업을 자동 생성한다.

```bash
./dev.sh app start candle-aggregation-worker
./dev.sh logs candle-aggregation-worker
```

## 5. 화면 사용법

- 시간 단위: 월봉, 주봉, 일봉, 시봉, 30분, 10분, 5분, 1분
- 기간: 1일, 1주, 1개월, 3개월, 1년, 3년
- 차트 아래: 거래량
- 차트 위: SMA 20/60, EMA 20, 볼린저 밴드, RSI 14
- 오른쪽/하단 요약: 현재가, 최우선 호가, 10호가 잔량·불균형, 최근 체결량

분석 화면은 `WebSocket 실시간` 상태가 보이면 연결된 것이다. 연결이 끊기면 같은 코인·시간 단위·기간으로 자동 재연결한다.

## 6. 문제 확인

```bash
./dev.sh status
./dev.sh logs api
./dev.sh logs realtime-collection-worker
./dev.sh logs backfill-collection-worker
./dev.sh logs candle-aggregation-worker
```

차트가 비어 있으면 먼저 선택 기간에 저장된 캔들이 있는지 `Backfill 관리`에서 확인한다. API 또는 WebSocket 연결이 실패하면 API와 web 프로세스를 각각 재시작한다.

```bash
./dev.sh app restart api
./dev.sh app restart web
```
